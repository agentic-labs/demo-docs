## High-level description
This file defines TypeScript types and interfaces for various integration authentication-related data transfer objects (DTOs) used in the backend of an application. It includes types for different integration platforms such as Vercel, GitHub, AWS, Qovery, and others, providing a structured way to handle authentication and data retrieval for these integrations.

## Code Structure
The file consists of multiple type definitions, each representing a specific DTO for different integration authentication operations. These types often extend or include parts of a common `TProjectPermission` type, suggesting a consistent permission model across different integrations.

## Symbols

### `TGetIntegrationAuthDTO`
#### Description
Defines the structure for retrieving integration authentication details.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| id | string | The unique identifier for the integration auth |
| (other properties) | Omit&lt;TProjectPermission, "projectId"&gt; | Project permission properties excluding projectId |

### `TOauthExchangeDTO`
#### Description
Defines the structure for exchanging OAuth codes for access tokens.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| integration | string | The name of the integration |
| code | string | The OAuth code to be exchanged |
| url | string (optional) | The callback URL |
| (other properties) | TProjectPermission | Project permission properties |

### `TSaveIntegrationAccessTokenDTO`
#### Description
Defines the structure for saving integration access tokens.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| integration | string | The name of the integration |
| accessId | string (optional) | The access ID |
| accessToken | string (optional) | The access token |
| url | string (optional) | The integration URL |
| namespace | string (optional) | The namespace |
| refreshToken | string (optional) | The refresh token |
| awsAssumeIamRoleArn | string (optional) | The AWS IAM role ARN to assume |
| (other properties) | TProjectPermission | Project permission properties |

### `TDeleteIntegrationAuthsDTO`
#### Description
Defines the structure for deleting integration authentications.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| integration | string | The name of the integration |
| projectId | string | The project ID |
| (other properties) | TProjectPermission | Project permission properties |

(Note: Similar structures are defined for various other integration-specific DTOs, such as `TIntegrationAuthAppsDTO`, `TIntegrationAuthTeamsDTO`, `TIntegrationAuthVercelBranchesDTO`, etc. Each of these follows a similar pattern, including specific properties relevant to the integration and inheriting from `TProjectPermission`.)

### `TVercelBranches`, `TChecklyGroups`, `TQoveryProjects`, `TQoveryEnvironments`, `TBitbucketWorkspace`, `TNorthflankSecretGroup`, `THerokuPipelineCoupling`, `TTeamCityBuildConfig`
#### Description
These types define the structure of data returned by various integration APIs.

### `TIntegrationsWithEnvironment`
#### Description
Extends the `TIntegrations` type with an optional `environment` property.

## Dependencies
The file imports types from other parts of the application:
| Dependency | Purpose |
|:-----------|:--------|
| @app/db/schemas | Imports `TIntegrations` type |
| @app/lib/types | Imports `TProjectPermission` type |

This file serves as a central location for defining the structure of data used in integration authentication processes across the application. It provides a clear interface for handling various integration-specific operations while maintaining a consistent approach to project permissions.
