## High-level description
This TypeScript file defines a set of types related to secret import operations in a project management system. These types are used to structure data transfer objects (DTOs) for various operations such as creating, updating, deleting, and retrieving secret imports, as well as managing secret import replications.

## Symbols

### `TCreateSecretImportDTO`
#### Description
Defines the structure for creating a new secret import.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| environment | string | The environment for the secret import |
| path | string | The path for the secret import |
| data | object | Contains environment and path information |
| isReplication | boolean (optional) | Indicates if this is a replication |
| TProjectPermission | type | Project permission details (inherited) |

## Notes
1. All DTOs include the `TProjectPermission` type, suggesting that project permissions are required for all secret import operations.
2. The `environment` and `path` fields are common across all DTOs, indicating their importance in identifying and managing secret imports.
3. The `TCreateSecretImportDTO` includes an optional `isReplication` field, which may be used to differentiate between original imports and replications.
4. The `TUpdateSecretImportDTO` allows for partial updates of the `environment`, `path`, and `position` fields.
5. The file focuses on type definitions and does not include any implementation details or business logic.
