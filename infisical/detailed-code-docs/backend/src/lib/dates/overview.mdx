## High-level description
This code provides utility functions for working with dates and times, specifically for converting between different time units and applying random jitter to delays.

## Symbols

### `daysToMillisecond`
#### Description
This function converts a given number of days to milliseconds.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| days | number | The number of days to convert. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
|  | number | The equivalent number of milliseconds. |

#### Internal Logic
The function performs a simple multiplication of the input `days` by the number of milliseconds in a day (24 hours * 60 minutes * 60 seconds * 1000 milliseconds).

### `secondsToMillis`
#### Description
This function converts a given number of seconds to milliseconds.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| seconds | number | The number of seconds to convert. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
|  | number | The equivalent number of milliseconds. |

#### Internal Logic
The function multiplies the input `seconds` by 1000, as there are 1000 milliseconds in one second.

### `applyJitter`
#### Description
This function applies a random jitter to a given delay in milliseconds.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| delayMs | number | The base delay in milliseconds. |
| jitterMs | number | The maximum amount of jitter to apply in milliseconds (positive or negative). |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
|  | number | The adjusted delay in milliseconds after applying the jitter. |

#### Internal Logic
1. The function generates a random jitter value between `-jitterMs` and `jitterMs` using `Math.random()`.
2. It adds this jitter value to the original `delayMs` to produce the final, jittered delay.
