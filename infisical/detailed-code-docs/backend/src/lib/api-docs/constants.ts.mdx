## High-level description
The `constants.ts` file defines a set of constants that represent the parameters required for various API endpoints related to managing groups, identities, authentication methods, organizations, projects, secrets, audit logs, dynamic secrets, secret tags, and PKI (Public Key Infrastructure). These constants are used for documentation purposes and to provide clear descriptions of the expected inputs for each API call.

## Code Structure
The code consists of a series of exported constant objects, each representing a specific API category (e.g., `GROUPS`, `IDENTITIES`, `UNIVERSAL_AUTH`). Within each category, there are nested objects representing individual API endpoints (e.g., `CREATE`, `UPDATE`, `DELETE`). Each endpoint object contains key-value pairs where the key is the parameter name and the value is a description of the parameter.
