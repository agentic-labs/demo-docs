## High-level description
This file defines a set of custom error classes that extend the base `Error` class. These custom errors are used throughout the application to represent specific error conditions, such as database errors, internal server errors, authorization errors, and more. Each error class provides a standardized way to handle and communicate these errors within the application.

## Code Structure
The code consists of a series of class definitions, each representing a specific type of error. Each class extends the base `Error` class and includes properties for storing the error name, an optional error object, and an optional custom message.

## Symbols

#### Description
Represents an error that occurred during a database operation.

|:-----|:-----|:------------|
| `{ message?: string; name?: string; error: unknown }` | object | An object containing optional message and name properties, and a required error object. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| `DatabaseError` | `DatabaseError` | An instance of the `DatabaseError` class. |

#### Internal Logic
The constructor sets the error message to either the provided message or a default message, sets the error name to either the provided name or a default name, and stores the provided error object.

|:-----|:-----|:------------|

It also sets the schemas property to an array containing the SCIM error schema URI.
