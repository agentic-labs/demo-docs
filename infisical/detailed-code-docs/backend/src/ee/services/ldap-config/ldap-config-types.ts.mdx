## High-level description
This file defines TypeScript types and interfaces related to LDAP (Lightweight Directory Access Protocol) configuration and authentication for an enterprise application. It includes types for LDAP configuration, data transfer objects (DTOs) for various LDAP-related operations, and structures for LDAP login and group mapping.

## Symbols

### `TLDAPConfig`
#### Description
Represents the structure of an LDAP configuration.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| id | string | Unique identifier for the LDAP configuration |
| organization | string | Organization identifier |
| isActive | boolean | Indicates if the LDAP configuration is active |
| url | string | LDAP server URL |
| bindDN | string | Distinguished Name (DN) used to bind to the LDAP server |
| bindPass | string | Password for the bind DN |
| uniqueUserAttribute | string | Attribute used to uniquely identify users |
| searchBase | string | Base DN for user searches |
| groupSearchBase | string | Base DN for group searches |
| groupSearchFilter | string | Filter used for group searches |
| caCert | string | CA certificate for secure LDAP connections |

### `TCreateLdapCfgDTO`
#### Description
Data Transfer Object for creating a new LDAP configuration.

#### Inputs
Extends `TOrgPermission` and includes:
| Name | Type | Description |
|:-----|:-----|:------------|
| orgId | string | Organization ID |
| isActive | boolean | Indicates if the configuration should be active |
| url | string | LDAP server URL |
| bindDN | string | Bind DN |
| bindPass | string | Bind password |
| uniqueUserAttribute | string | Unique user attribute |
| searchBase | string | Search base |
| searchFilter | string | Search filter |
| groupSearchBase | string | Group search base |
| groupSearchFilter | string | Group search filter |
| caCert | string | CA certificate |

### `TUpdateLdapCfgDTO`
#### Description
Data Transfer Object for updating an existing LDAP configuration.

#### Inputs
Extends `TOrgPermission` and includes:
| Name | Type | Description |
|:-----|:-----|:------------|
| orgId | string | Organization ID |
| isActive? | boolean | Optional: Update active status |
| url? | string | Optional: Update LDAP server URL |
| bindDN? | string | Optional: Update bind DN |
| bindPass? | string | Optional: Update bind password |
| uniqueUserAttribute? | string | Optional: Update unique user attribute |
| searchBase? | string | Optional: Update search base |
| searchFilter? | string | Optional: Update search filter |
| groupSearchBase? | string | Optional: Update group search base |
| groupSearchFilter? | string | Optional: Update group search filter |
| caCert? | string | Optional: Update CA certificate |

### `TGetLdapCfgDTO`
#### Description
Data Transfer Object for retrieving an LDAP configuration.

#### Inputs
Extends `TOrgPermission` and includes:
| Name | Type | Description |
|:-----|:-----|:------------|
| orgId | string | Organization ID |

### `TLdapLoginDTO`
#### Description
Data Transfer Object for LDAP login.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ldapConfigId | string | LDAP configuration ID |
| externalId | string | External user ID |
| username | string | Username |
| firstName | string | User's first name |
| lastName | string | User's last name |
| email | string | User's email |
| orgId | string | Organization ID |
| groups? | { dn: string; cn: string; }[] | Optional: User's LDAP groups |
| relayState? | string | Optional: Relay state for SAML |

### `TGetLdapGroupMapsDTO`, `TCreateLdapGroupMapDTO`, `TDeleteLdapGroupMapDTO`, `TTestLdapConnectionDTO`
#### Description
Data Transfer Objects for various LDAP group mapping operations and testing LDAP connection.

#### Inputs
All extend `TOrgPermission` and include specific fields related to their operations.

## Dependencies
The file imports `TOrgPermission` from "@app/lib/types", which is likely used for organization-level permission checks.

This file serves as a central location for defining the structure of LDAP-related data and operations in the application, ensuring type safety and consistency across the LDAP integration features.
