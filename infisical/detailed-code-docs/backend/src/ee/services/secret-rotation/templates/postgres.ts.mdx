## High-level description
This code defines a template for PostgreSQL database configuration in the context of secret rotation. It specifies the structure and requirements for inputs and outputs related to PostgreSQL database connections, including user credentials and connection details.

## Symbols

### POSTGRES_TEMPLATE
#### Description
`POSTGRES_TEMPLATE` is a constant object that defines the structure and requirements for PostgreSQL database configuration in the context of secret rotation. It specifies the type of provider, client, input properties, and output properties.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| POSTGRES_TEMPLATE | Object | A template object for PostgreSQL configuration |

#### Outputs
The `POSTGRES_TEMPLATE` object itself is the output.

#### Internal Logic
The template defines:
1. The type of provider function (`DB`)
2. The client type (`Pg` for PostgreSQL)
3. Input properties with their types, defaults, and descriptions
4. Required input properties
5. Output properties

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| TDbProviderClients | Enum for database provider clients |
| TProviderFunctionTypes | Enum for provider function types |

## Configuration
The `POSTGRES_TEMPLATE` object defines the following configuration options:

| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| admin_username | string | - | Admin username for the PostgreSQL database |
| admin_password | string | - | Admin password for the PostgreSQL database |
| host | string | - | Hostname of the PostgreSQL server |
| database | string | - | Name of the database to connect to |
| port | integer | 5432 | Port number for the PostgreSQL server |
| username1 | string | "infisical-pg-user1" | First user that must be created in the database |
| username2 | string | "infisical-pg-user2" | Second user that must be created in the database |
| ca | string | - | SSL certificate for database authentication (optional) |

Note: The `admin_username`, `admin_password`, `host`, `database`, `username1`, `username2`, and `port` are required fields.

## API/Interface Reference
The `POSTGRES_TEMPLATE` object defines the following output interface:

| Property | Type | Description |
|:---------|:-----|:------------|
| db_username | string | The username for database access |
| db_password | string | The password for database access |

This template can be used to generate or validate configuration objects for PostgreSQL database connections in the context of secret rotation.
