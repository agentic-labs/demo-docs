## High-level description
This code defines a Zod schema for dynamic secrets and related TypeScript types. It's generated automatically from a database schema and is not intended to be manually edited.

## Code Structure
The code defines a Zod schema `DynamicSecretsSchema` and three TypeScript types: `TDynamicSecrets`, `TDynamicSecretsInsert`, and `TDynamicSecretsUpdate`.

## Symbols

### `DynamicSecretsSchema`
#### Description
A Zod schema that defines the structure and validation rules for dynamic secrets.

#### Internal Logic
The schema includes various fields with specific data types and validation rules:
- `id`: UUID string
- `name`: string
- `version`: number
- `type`: string
- `defaultTTL`: string
- `maxTTL`: optional nullable string
- `inputIV`: string
- `inputCiphertext`: string
- `inputTag`: string
- `algorithm`: string with default "aes-256-gcm"
- `keyEncoding`: string with default "utf8"
- `folderId`: UUID string
- `status`: optional nullable string
- `statusDetails`: optional nullable string
- `createdAt`: date
- `updatedAt`: date

### `TDynamicSecrets`
#### Description
A TypeScript type inferred from the `DynamicSecretsSchema`.

### `TDynamicSecretsInsert`
#### Description
A TypeScript type for inserting dynamic secrets, omitting immutable database keys.

### `TDynamicSecretsUpdate`
#### Description
A TypeScript type for updating dynamic secrets, making all fields optional and omitting immutable database keys.
