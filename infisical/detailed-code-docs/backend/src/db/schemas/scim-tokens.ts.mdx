## High-level description
The code defines a Zod schema, `ScimTokensSchema`, for validating and type-checking data associated with SCIM tokens. It also defines related types for inserting and updating SCIM token data.

## References
- `TImmutableDBKeys` from "./models"

## Symbols
### `ScimTokensSchema`
#### Description
This symbol defines a Zod schema for validating SCIM token data. It ensures that the data conforms to the expected structure and types.

#### Inputs
N/A - This is a Zod schema definition, not a function.

#### Outputs
N/A - This is a Zod schema definition, not a function.

#### Internal Logic
The schema defines the following fields:
- `id`: A required string representing the SCIM token ID.
- `ttlDays`: A required number (coerced from input) representing the time-to-live in days, defaulting to 365.
- `description`: A required string representing the token description.
- `orgId`: A required UUID string representing the organization ID.
- `createdAt`: A required date representing the creation timestamp.
- `updatedAt`: A required date representing the last update timestamp.

### `TScimTokens`
#### Description
A type alias representing the inferred type of the `ScimTokensSchema`. It represents a fully validated SCIM token object.

### `TScimTokensInsert`
#### Description
A type alias representing the input type for inserting a new SCIM token. It omits the `TImmutableDBKeys` from the `ScimTokensSchema` input type, as those keys are typically auto-generated by the database.

### `TScimTokensUpdate`
#### Description
A type alias representing the input type for updating an existing SCIM token. It's a partial type of `TScimTokensInsert`, allowing for updates to specific fields.
