## High-level description
This code defines a Zod schema, `KmsKeysSchema`, for validating KMS key data. It also defines related types for inserting and updating KMS key data.

## References
- `TImmutableDBKeys` from "./models"

## Symbols

### `KmsKeysSchema`
#### Description
This symbol defines a Zod schema for validating KMS key data. It specifies the expected data types and optional/required properties for a KMS key object.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| KmsKeysSchema | z.ZodObject | A Zod object schema for validating KMS key data. |

#### Internal Logic
The schema defines the following properties for a KMS key:
- `id`: A UUID string.
- `description`: An optional string.
- `isDisabled`: An optional boolean, defaulting to false.
- `isReserved`: An optional boolean, defaulting to true.
- `orgId`: A UUID string.
- `slug`: A string.
- `createdAt`: A date object.
- `updatedAt`: A date object.
