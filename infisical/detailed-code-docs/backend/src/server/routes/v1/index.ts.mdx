## High-level description
This code registers all API routes for the application's v1 API version. It uses FastifyZodProvider to register different route handlers under specific prefixes, effectively organizing the API structure.

## Code Structure
The code consists of a single function `registerV1Routes` that imports and registers various route handler functions. Each route handler is responsible for a specific API resource or functionality, like users, organizations, projects, etc.

## Symbols
### `registerV1Routes`
#### Description
This function registers all API routes for the application's v1 API version using FastifyZodProvider. It groups related routes under common prefixes, creating a structured API hierarchy.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| server | FastifyZodProvider | An instance of FastifyZodProvider, which is used to register the routes. |

#### Outputs
This function does not return any value.

#### Side Effects
This function registers all the API routes for the application.

#### Internal Logic
The function imports route handler functions for different API resources and registers them with the FastifyZodProvider instance using the `register` method. It organizes related routes under specific prefixes, like `/sso`, `/auth`, `/password`, etc., creating a logical structure for the API. For example, all authentication-related routes are grouped under the `/auth` prefix.
