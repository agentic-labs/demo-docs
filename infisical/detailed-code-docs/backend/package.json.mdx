## High-level description
This `package.json` file defines the configuration, dependencies, and scripts for the backend of an application. It includes various build, development, and deployment scripts, as well as a comprehensive list of dependencies and development dependencies.

## Code Structure
The file is structured as a standard `package.json`, containing metadata about the project, scripts for various tasks, and lists of dependencies.

## Symbols

### `scripts`
#### Description
Defines various npm scripts for building, testing, and running the application.

#### Internal Logic
- `binary:build`: Cleans, builds frontend and backend, and prepares for binary packaging.
- `binary:package`: Packages the application for the host platform.
- `dev`: Runs the application in development mode with watch and pretty logging.
- `build`: Builds the application using tsup.
- `start`: Runs the built application.
- `test:e2e`: Runs end-to-end tests.
- `migration:*`: Various database migration scripts.
- `seed:*`: Database seeding scripts.

### `dependencies`
#### Description
Lists the production dependencies required by the application.

#### Internal Logic
Includes various libraries for:
- AWS services integration
- Authentication and authorization
- Database operations
- Encryption and security
- API development (Fastify)
- Logging and monitoring

### `devDependencies`
#### Description
Lists the development dependencies used for building, testing, and development processes.

#### Internal Logic
Includes tools for:
- TypeScript compilation and linting
- Testing (Vitest)
- Code formatting (ESLint, Prettier)
- Build processes (Babel, tsup)
