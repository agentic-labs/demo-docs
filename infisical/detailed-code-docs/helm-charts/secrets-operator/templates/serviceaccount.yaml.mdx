## High-level description
This YAML file defines a Kubernetes ServiceAccount resource for the secrets-operator controller manager. It sets up the necessary metadata, labels, and annotations for the ServiceAccount, which is typically used for authentication and authorization purposes within a Kubernetes cluster.

## Symbols

### ServiceAccount Resource
#### Description
This YAML defines a Kubernetes ServiceAccount resource, which is used to provide an identity for processes running in a Pod.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| .Values.controllerManager.serviceAccount.annotations | map | Annotations to be added to the ServiceAccount |

#### Internal Logic
1. Sets the `apiVersion` and `kind` for a Kubernetes ServiceAccount.
2. Defines metadata for the ServiceAccount:
   - Sets the name using a template function `secrets-operator.fullname`.
   - Adds predefined labels for Kubernetes app management.
   - Includes additional labels using the `secrets-operator.labels` template function.
   - Adds annotations from the Helm values file.
