## High-level description
This file is a Helm Chart definition for the Infisical standalone application. It specifies the chart metadata, version information, and dependencies required for deploying a full Infisical application using Kubernetes.

## Code Structure
The Chart.yaml file is structured with metadata fields at the top, followed by version information and a list of dependencies.

## Symbols

### Chart Metadata
#### Description
Defines the basic information about the Helm chart.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| apiVersion | string | The API version of the Helm chart (v2) |
| name | string | The name of the Helm chart (infisical-standalone) |
| description | string | A brief description of the chart's purpose |
| type | string | The type of chart (application) |

### Version Information
#### Description
Specifies the version numbers for the chart and the application it deploys.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| version | string | The version of the Helm chart (1.0.8) |
| appVersion | string | The version of the application being deployed (1.0.1) |

### Dependencies
#### Description
Lists the required dependencies for the Infisical standalone application.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| name | string | The name of the dependency |
| version | string | The version of the dependency |
| repository | string | The repository URL for the dependency |
| condition | string | The condition under which the dependency is required |

#### Internal Logic
The dependencies section defines three main components:
1. ingress-nginx: For handling ingress traffic (version 4.0.13)
2. postgresql: For database services (version 14.1.3)
3. redis: For caching and message broker services (version 18.14.0)

Each dependency is conditionally included based on the specified condition.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| ingress-nginx | Provides ingress controller for routing external traffic |
| postgresql | Provides database services for the application |
| redis | Provides caching and message broker services |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| ingress.nginx.enabled | boolean | N/A | Enables or disables the ingress-nginx dependency |
| postgresql.enabled | boolean | N/A | Enables or disables the postgresql dependency |
| redis.enabled | boolean | N/A | Enables or disables the redis dependency |

Note: The actual default values are not specified in this file and would typically be defined in a separate values.yaml file.
