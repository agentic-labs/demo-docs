## High-level description
The `frontend/src/views/Settings/OrgSettingsPage` directory contains components and logic for rendering and managing the organization settings page in the application. It provides a comprehensive interface for users to configure various aspects of their organization, including general settings, authentication, encryption, audit logs, and incident contacts.

## What does it do?
This directory implements a complex settings page with multiple functionalities:

1. Renders a tabbed interface for different organization settings sections.
2. Manages general organization settings like name and slug.
3. Configures authentication methods (SAML SSO, OIDC, LDAP, SCIM).
4. Handles encryption settings and external Key Management System configurations.
5. Manages audit log stream settings.
6. Allows for incident contact management.
7. Provides functionality for organization deletion.

The code handles user interactions, form submissions, API calls, and state management for these various organization-related operations. It also implements permission checks to ensure users can only access features they're authorized to use, and applies plan restrictions based on the organization's subscription level.

## Entry points
The main entry point for this functionality is the `OrgSettingsPage` component, which is exported from the `index.tsx` file. This component renders a title and includes the `OrgTabGroup` component, which creates the tabbed interface for the different sections of the organization settings.

The `OrgTabGroup` component renders the following main tabs:

1. General
2. Security
3. Encryption
4. Audit Log Streams

Each tab corresponds to a specific component that handles its respective functionality.
