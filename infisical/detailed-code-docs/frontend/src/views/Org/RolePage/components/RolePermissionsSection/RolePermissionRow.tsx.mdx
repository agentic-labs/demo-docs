## High-level description
This code defines a React component called `RolePermissionRow` that renders a row in a table for managing role permissions. It allows users to set and modify permissions for different actions (read, create, edit, delete) on various resources within an organization's admin console.

## Code Structure
The main component `RolePermissionRow` uses several React hooks and custom components to manage the state and render the UI. It interacts with a form using react-hook-form and renders a collapsible row with a dropdown for selecting permission levels and checkboxes for individual permissions.

## Symbols

### `RolePermissionRow`
#### Description
A React component that renders a row for managing role permissions in a table format.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| isEditable | boolean | Determines if the permissions can be edited |
| title | string | The title of the permission row |
| formName | string | The key for the permission in the form data |
| setValue | UseFormSetValue&lt;TFormSchema&gt; | Function to set form values |
| control | Control&lt;TFormSchema&gt; | Form control object from react-hook-form |

#### Internal Logic
1. Uses `useToggle` hook to manage row expansion and custom permission states.
2. Calculates the selected permission category based on the current rule.
3. Handles permission changes through the dropdown and individual checkboxes.
4. Renders a collapsible row with a dropdown for permission levels and checkboxes for individual permissions.

### `getPermissionList`
#### Description
A function that returns the appropriate permission list based on the given option.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| option | string | The key representing the permission category |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| permissionList | Array | An array of permission objects with action and label properties |

### `Permission` (enum)
#### Description
An enum defining the possible permission levels for a role.

## Error Handling
The component includes basic error handling by showing an error notification when attempting to update a non-editable role.

## Performance Considerations
The component uses `useMemo` to calculate the selected permission category, which helps optimize performance by memoizing the result and preventing unnecessary recalculations.

This component is crucial for managing role permissions in the organization's admin console. It provides a user-friendly interface for setting and modifying permissions with different levels of access and granular control over individual actions.
