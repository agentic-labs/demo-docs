## High-level description
This TypeScript file defines types and enums used for filtering, sorting, and grouping secrets in the SecretMainPage component of the frontend application. It provides a structure for managing filter options, sorting directions, and grouping criteria for secrets.

## Symbols

### `Filter`
#### Description
A type definition that represents the filter options for secrets on the main page.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| tags | Record&lt;string, boolean&gt; | An object where keys are tag names and values are boolean flags indicating whether the tag is selected |
| searchFilter | string | A string used for searching or filtering secrets |
| groupBy | GroupBy \| null \| undefined | An optional parameter specifying how to group the secrets |

### `SortDir`
#### Description
An enum that defines the possible sorting directions for secrets.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ASC | string | Represents ascending order, with the value "asc" |
| DESC | string | Represents descending order, with the value "desc" |

### `GroupBy`
#### Description
An enum that defines the possible grouping criteria for secrets.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| PREFIX | string | Represents grouping by prefix, with the value "prefix" |

## Dependencies
This file relies on TypeScript's built-in types and does not import any external libraries or modules.

## Notes
1. The `Filter` type uses a `Record&lt;string, boolean&gt;` for tags, which allows for dynamic tag names as keys and boolean values to indicate selection status.
2. The `groupBy` property in the `Filter` type is optional and can be either of type `GroupBy`, `null`, or `undefined`.
3. Currently, the `GroupBy` enum only contains one option (`PREFIX`), but it's structured as an enum to allow for easy addition of more grouping options in the future.
4. The `SortDir` enum provides a type-safe way to specify sorting direction throughout the application.

These type definitions and enums are likely used in other components and services related to the SecretMainPage to ensure type safety and provide a consistent structure for handling secret filtering, sorting, and grouping operations.
