## High-level description
This code defines and exports configuration constants for a Next.js application. It retrieves environment variables for various third-party services and application settings, providing default values where applicable.

## Symbols

### `ENV`
#### Description
Defines the environment in which the application is running.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| process.env.NEXT_PUBLIC_ENV | string | Environment variable for the application environment |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ENV | string | The application environment, defaulting to "development" if not set |

#### Internal Logic
Uses the non-null assertion operator (`!`) to assert that the environment variable is not null or undefined. If it is not set, it defaults to "development".

## Dependencies
This code relies on the `process.env` object, which is typically available in Node.js environments and is extended by Next.js to include client-side environment variables prefixed with `NEXT_PUBLIC_`.
