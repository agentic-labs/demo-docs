## High-level description
This file defines a set of regular expressions used for password validation in a frontend application. These regexes cover various aspects of password strength, including character types, repeated characters, escape sequences, and potential personal information patterns.

#### Internal Logic
The `lowEntropyRegexes` array contains patterns for:
1. Email addresses
2. URLs (including subdomains, paths, top-level domains, and query parameters)
3. Dates in various formats
4. Phone numbers (generalized pattern)
5. Passport numbers (generalized pattern)
6. Driver's license numbers (generalized pattern)
7. US social security numbers
