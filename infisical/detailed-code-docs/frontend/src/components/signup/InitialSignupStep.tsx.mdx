## High-level description
This code defines a React component called `InitialSignupStep` that renders a signup page with various authentication options. It provides buttons for signing up with Google, GitHub, GitLab, or email, depending on the enabled login methods configured on the server.

## Code Structure
The main component `InitialSignupStep` uses several hooks and components to render the signup options. It relies on the server configuration to determine which signup methods to display.

## Symbols

### InitialSignupStep
#### Description
This is the main functional component that renders the initial signup step interface.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| setIsSignupWithEmail | (value: boolean) =&gt; void | A function to set whether the user is signing up with email |

#### Internal Logic
1. Uses the `useTranslation` hook for internationalization.
2. Retrieves the server configuration using the `useServerConfig` hook.
3. Defines a `shouldDisplaySignupMethod` function to check if a login method should be displayed.
4. Renders a title and buttons for each enabled login method (Google, GitHub, GitLab, Email).
5. Displays a policy text and a link to the login page.

## Configuration
The component uses the server configuration to determine which login methods to display:

| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| enabledLoginMethods | LoginMethod[] | undefined | Array of enabled login methods |

## API/Interface Reference
The component interacts with the following endpoints:

| Endpoint | Method | Description |
|:---------|:-------|:------------|
| /api/v1/sso/redirect/google | GET | Redirects to Google SSO |
| /api/v1/sso/redirect/github | GET | Redirects to GitHub SSO |
| /api/v1/sso/redirect/gitlab | GET | Redirects to GitLab SSO |

Note: These endpoints are opened in a new window and the current window is closed after redirection.
