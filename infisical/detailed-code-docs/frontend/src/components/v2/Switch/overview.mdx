## High-level description
This directory contains the implementation of a customizable Switch component for a React application. The Switch component is built using Radix UI's Switch primitive and is styled with Tailwind CSS. It's part of a version 2 ("v2") component library or design system.

## What does it do?
The Switch component provides a toggle switch functionality, similar to what you might see for enabling or disabling a setting in a user interface. It allows users to switch between two states (on/off) with a simple click or tap. The component is highly customizable, supporting various states such as disabled and required, and includes an optional label. It's designed to be accessible and easy to integrate into larger forms or UI layouts.

## Configuration
The Switch component is highly configurable through props:

- `children`: ReactNode - The label content for the switch.
- `id`: string - Unique identifier for the switch.
- `className`: string - Additional CSS classes for custom styling.
- `isChecked`: boolean - Determines if the switch is in the on state.
- `isDisabled`: boolean - Determines if the switch is disabled.
- `isRequired`: boolean - Determines if the switch is required.

Additional props from Radix UI's Switch primitive can also be passed through.

Here's an example of how the Switch component might be used:

```jsx
&lt;Switch
  isChecked={darkMode}
  onChange={toggleDarkMode}
  isRequired={true}
&gt;
  Dark mode
&lt;/Switch&gt;
```

This would render a required switch for toggling dark mode, with the label "Dark mode".

The Storybook stories provide examples of different configurations:

These stories demonstrate how the Switch component can be configured for different use cases.

The Switch component uses Tailwind CSS for styling, allowing for easy customization and consistency with the overall design system. The styling can be further customized by passing additional classes through the `className` prop.
