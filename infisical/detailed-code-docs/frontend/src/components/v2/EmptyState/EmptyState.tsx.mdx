## High-level description
The `EmptyState` component is a reusable React component that displays a message and an icon when there is no content to show. It is typically used in tables or lists to provide a visual indication and explanation when the data is empty.

## Code Structure
The `EmptyState` component is defined as a functional component that takes props for customization. It uses other components like `FontAwesomeIcon` for icons and `twMerge` for class name merging.

## Symbols

### EmptyState
#### Description
A React component that renders an empty state message with an icon.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| title | ReactNode | The main message to display |
| className | string (optional) | Additional CSS classes to apply |
| children | ReactNode (optional) | Additional content to render below the title |
| icon | IconDefinition (optional) | The icon to display (default: faCubesStacked) |
| iconSize | SizeProp (optional) | The size of the icon (default: "2x") |

#### Outputs
Returns a JSX element representing the empty state.

#### Internal Logic
1. The component renders a `div` container with merged CSS classes.
2. It displays a `FontAwesomeIcon` with the specified icon and size.
3. The title is rendered in a nested `div`.
4. If children are provided, they are rendered below the title.

## Usage in Related Files
The `EmptyState` component is used in various parts of the application to display empty states in tables and lists. Some examples include:

1. In `SecretOverviewPage.tsx`, it's used to show messages when there are no environments, no secrets, or when a search yields no results.
2. In `IPAllowlistTable.tsx`, it's used to display a message when no IP addresses are added.
3. In `SessionsTable.tsx`, it's shown when there are no sessions on file.
4. In `CertificateTemplatesTable.tsx`, it's used when no certificate templates have been created.
5. In `SecretScanningLogsTable.tsx`, it's displayed when no risks are detected.
6. In `AccessApprovalRequest.tsx`, it's shown when there are no more access requests pending.

In all these cases, the `EmptyState` component provides a consistent way to display messages when there's no data to show, improving the user experience by giving clear feedback about the current state of the application.
