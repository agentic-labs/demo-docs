## High-level description
This file defines constants and enums related to PKI (Public Key Infrastructure) items in the frontend of an application. It specifically defines types of PKI items and provides a mapping between these types and their human-readable names.

## Symbols

### `PkiItemType`
#### Description
An enumeration that defines the types of PKI items supported in the application.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| CERTIFICATE | string | Represents a certificate type PKI item |
| CA | string | Represents a Certificate Authority (CA) type PKI item |

#### Internal Logic
The enum uses string values for better readability and potential easier integration with backend APIs or data structures.

___

### `pkiItemTypeToNameMap`
#### Description
An object that maps `PkiItemType` enum values to their corresponding human-readable names.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| [PkiItemType.CA] | string | The human-readable name for the CA type ("CA") |
| [PkiItemType.CERTIFICATE] | string | The human-readable name for the Certificate type ("Certificate") |

#### Internal Logic
This constant uses computed property names to create a mapping between each `PkiItemType` enum value and its corresponding display name. The type annotation `{ [K in PkiItemType]: string }` ensures that the object contains a string value for each key in the `PkiItemType` enum.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| TypeScript | The code uses TypeScript features such as enums and mapped types |

This file serves as a central location for PKI-related constants and type definitions, which can be imported and used throughout the frontend application. It provides a clear and type-safe way to work with different PKI item types and their display names.
