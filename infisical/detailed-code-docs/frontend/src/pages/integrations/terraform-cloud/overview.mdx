## High-level description
This directory contains React components for integrating Terraform Cloud with the Infisical application. It provides functionality for authorizing and creating Terraform Cloud integrations, allowing users to connect their Terraform Cloud workspaces with Infisical for managing secrets and variables.

## What does it do?
The code in this directory implements two main workflows:

1. Authorization: Users can authorize Infisical to access their Terraform Cloud account by providing their Workspace ID and API Token. This process establishes a connection between Infisical and Terraform Cloud.

2. Integration Creation: After authorization, users can create a new integration by configuring various settings such as the source environment, secrets path, variable type, target Terraform Cloud project, and initial sync behavior. This integration allows Infisical to manage secrets and variables in the specified Terraform Cloud workspace.

These workflows enable users to securely manage their Terraform Cloud configurations and secrets through the Infisical platform, providing a streamlined process for integrating infrastructure-as-code practices with secret management.

## Entry points
The directory contains two main entry points:

1. `authorize.tsx`: This file contains the `TerraformCloudCreateIntegrationPage` component, which handles the authorization process. It provides a form for users to input their Terraform Cloud Workspace ID and API Token.

2. `create.tsx`: This file contains another `TerraformCloudCreateIntegrationPage` component (with a different implementation) that handles the integration creation process. It provides a form for users to configure various settings for the Terraform Cloud integration.

The data flow typically starts with the authorization page, where users provide their Terraform Cloud credentials. Once authorized, users are redirected to the creation page to set up the specific integration details.

## Configuration
Both components use a configuration option `requireAuth` set to `true`, indicating that authentication is required to access these pages.

The creation page also defines two configuration arrays:

1. `initialSyncBehaviors`: Defines the available options for initial sync behavior when creating an integration.
2. `variableTypes`: Defines the available variable type options for the integration.

These configurations allow for flexibility in setting up the Terraform Cloud integration, catering to different use cases and preferences.

Example code snippet from the creation page:

```javascript
const initialSyncBehaviors = [
  { value: "MERGE", label: "Merge" },
  { value: "OVERWRITE", label: "Overwrite" }
];

const variableTypes = [
  { value: "TERRAFORM", label: "Terraform" },
  { value: "ENV", label: "Environment" }
];
```

These configuration options are used in the form to provide dropdown selections for users when setting up their integration.

In summary, this directory provides a comprehensive solution for integrating Terraform Cloud with Infisical, handling both the authorization and creation processes with a user-friendly interface and flexible configuration options.
