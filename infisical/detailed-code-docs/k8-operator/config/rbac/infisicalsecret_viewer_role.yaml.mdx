## High-level description
This YAML file defines a Kubernetes ClusterRole named "infisicalsecret-viewer-role" that grants read-only permissions for viewing InfisicalSecrets resources. It is part of the RBAC (Role-Based Access Control) configuration for the Infisical Kubernetes operator.

## Symbols

### ClusterRole
#### Description
This ClusterRole defines the permissions for viewing InfisicalSecrets resources across all namespaces in the Kubernetes cluster.

#### Internal Logic
The ClusterRole is configured with two main rule sets:
1. Allows read-only access (get, list, watch) to InfisicalSecrets resources.
2. Allows read-only access (get) to the status of InfisicalSecrets resources.

The ClusterRole is configured with the following metadata labels:

| app.kubernetes.io/component | string | rbac | Indicates the component type |
| app.kubernetes.io/created-by | string | k8-operator | Specifies the creator |
| app.kubernetes.io/part-of | string | k8-operator | Indicates the larger system it belongs to |
| app.kubernetes.io/managed-by | string | kustomize | Specifies the management tool |

## API/Interface Reference
This ClusterRole defines the following permissions:
| API Group | Resources | Verbs | Description |
|:----------|:----------|:------|:------------|
| secrets.infisical.com | infisicalsecrets | get, list, watch | Allows viewing InfisicalSecrets |
| secrets.infisical.com | infisicalsecrets/status | get | Allows viewing InfisicalSecrets status |

Note: This ClusterRole is specifically designed for end-users who need read-only access to InfisicalSecrets resources. It does not grant any write or modify permissions.
