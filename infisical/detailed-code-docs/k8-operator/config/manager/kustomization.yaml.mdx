## High-level description
This file is a Kustomization configuration for a Kubernetes operator. It specifies the resources to be included and defines image customization for the controller, setting it to use the Infisical Kubernetes operator image.

## Symbols

### Kustomization
#### Description
This is the main configuration object for Kustomize, a tool used to customize Kubernetes manifests. It defines how to generate or transform Kubernetes resources.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| resources | list | Specifies the Kubernetes resource files to be included |
| images | list | Defines image customizations for the resources |

#### Internal Logic
The Kustomization configuration does the following:
1. Includes the `manager.yaml` file as a resource.
2. Specifies the API version and kind for the Kustomization.
3. Defines an image customization for the controller, setting its name and tag.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| Kustomize | Used to customize and patch Kubernetes manifests |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| apiVersion | string | kustomize.config.k8s.io/v1beta1 | Specifies the API version of the Kustomization configuration |
| kind | string | Kustomization | Defines the type of the configuration |

## API/Interface Reference
| Field | Type | Description |
|:------|:-----|:------------|
| resources | list | Specifies the Kubernetes resource files to be included |
| images | list | Defines image customizations for the resources |

In this case, the `resources` field includes `manager.yaml`, and the `images` field specifies the controller image to be used, which is `infisical/kubernetes-operator:latest`.
