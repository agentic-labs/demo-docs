## High-level description
This YAML file defines Kubernetes resources for configuring the Infisical operator system. It creates a dedicated namespace and a ConfigMap containing configuration data for the Infisical operator.

## Symbols

### Namespace
#### Description
Defines a Kubernetes Namespace resource named "infisical-operator-system".

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| apiVersion | string | Specifies the Kubernetes API version (v1) |
| kind | string | Specifies the resource type (Namespace) |
| metadata | object | Contains metadata for the Namespace |

#### Internal Logic
The Namespace resource is created with the name "infisical-operator-system", which will be used to isolate and organize resources related to the Infisical operator.

### ConfigMap
#### Description
Defines a Kubernetes ConfigMap resource named "infisical-config" within the "infisical-operator-system" namespace.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| apiVersion | string | Specifies the Kubernetes API version (v1) |
| kind | string | Specifies the resource type (ConfigMap) |
| metadata | object | Contains metadata for the ConfigMap |
| data | object | Contains key-value pairs of configuration data |

#### Internal Logic
The ConfigMap is created with the name "infisical-config" in the "infisical-operator-system" namespace. It contains a single key-value pair in the data section:
- Key: "hostAPI"
- Value: "https://example.com/api"

This configuration data can be used by the Infisical operator to specify the API host it should communicate with.

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| hostAPI | string | "https://example.com/api" | Specifies the host API URL for the Infisical operator |

Note: The configuration is defined within the ConfigMap and can be easily modified or extended as needed.
