## High-level description
This `render.yaml` file defines the configuration for deploying the Infisical application on the Render platform. It specifies a web service with Docker-based deployment, environment variables, and other deployment settings.

## Code Structure
The file contains a single `services` array with one service definition for the Infisical application.

## Symbols

### Service Definition
#### Description
Defines a web service for the Infisical application, specifying deployment settings and environment variables.

#### Internal Logic
- Sets up a web service named "infisical"
- Uses Docker for deployment with a custom Dockerfile
- Disables auto-deployment
- Specifies the GitHub repository for the application
- Defines several environment variables, some with generated values and others with specific settings

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| Docker | Used for containerizing and deploying the application |
| GitHub | Source code repository for the application |

### Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| type | string | web | Specifies the service type as a web application |
| name | string | infisical | Sets the name of the service |
| env | string | docker | Specifies Docker as the deployment environment |
| dockerfilePath | string | ./Dockerfile.standalone-infisical | Path to the custom Dockerfile for the service |
| autoDeploy | boolean | false | Disables automatic deployment |
| repo | string | https://github.com/Infisical/infisical.git | GitHub repository URL for the application |

## Environment Variables
The configuration defines several environment variables for the application:

| Variable | Generation |
|:---------|:-----------|
| ENCRYPTION_KEY | Generated |
| JWT_SIGNUP_SECRET | Generated |
| JWT_REFRESH_SECRET | Generated |
| JWT_SERVICE_SECRET | Generated |
| JWT_AUTH_SECRET | Generated |
| JWT_PROVIDER_AUTH_SECRET | Generated |
| MONGO_URL | Not synced |
| PORT | Set to 443 |

Note: The `generateValue: true` setting for most JWT-related variables and the encryption key indicates that Render will automatically generate secure values for these variables during deployment.
