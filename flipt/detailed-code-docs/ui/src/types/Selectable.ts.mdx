## High-level description
This code defines two interfaces, `ISelectable` and `IFilterable`, which are used for creating selectable and filterable items in user interfaces. These interfaces are fundamental for implementing dropdown menus, comboboxes, and other selection components with optional filtering capabilities.

## Symbols

### `ISelectable`
#### Description
`ISelectable` is an interface that defines the basic structure for selectable items in a user interface.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| key | string | A unique identifier for the selectable item |
| displayValue | string | The value to be displayed for the selectable item |

### `IFilterable`
#### Description
`IFilterable` is an interface that extends `ISelectable`, adding properties for filtering and status indication.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| key | string | (Inherited from ISelectable) A unique identifier for the filterable item |
| displayValue | string | (Inherited from ISelectable) The value to be displayed for the filterable item |
| status | 'active' \| 'inactive' \| undefined | Optional status of the filterable item |
| filterValue | string | The value used for filtering the item |

## Notes
1. The `IFilterable` interface is an extension of `ISelectable`, which means it inherits all properties from `ISelectable` and adds additional properties.
2. The `status` property in `IFilterable` is optional and can be used to indicate the active/inactive state of an item.
3. These interfaces are crucial for implementing reusable UI components that involve selection and filtering, such as dropdown menus, comboboxes, and list boxes.
4. The separation of `ISelectable` and `IFilterable` allows for flexibility in implementing UI components that may or may not require filtering capabilities.
