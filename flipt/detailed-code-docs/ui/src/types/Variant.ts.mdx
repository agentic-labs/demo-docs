## High-level description
This file defines TypeScript interfaces and types related to variants in a feature flag system. It also includes a utility function for converting a variant to a filterable format. The code is part of a UI component for managing feature flags and their variants.

## Code Structure
The file defines several interfaces and types that build upon each other:
- `IVariantBase`: Basic variant properties
- `IVariant`: Extends `IVariantBase` with additional fields
- `FilterableVariant`: A subset of `IVariant` with added `IFilterable` properties
- `toFilterableVariant`: A function to convert an `IVariant` to a `FilterableVariant`

## Symbols

### `IVariantBase`
#### Description
An interface defining the basic properties of a variant.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| key | string | Unique identifier for the variant |
| name | string | Display name of the variant |
| description | string | Detailed description of the variant |
| attachment | string (optional) | Additional data attached to the variant |

### `IVariant`
#### Description
An interface extending `IVariantBase` with additional metadata.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| id | string | Unique identifier for the variant instance |
| createdAt | string | Timestamp of when the variant was created |
| updatedAt | string | Timestamp of when the variant was last updated |

### `FilterableVariant`
#### Description
A type that combines selected properties of `IVariant` with `IFilterable` interface.

### `toFilterableVariant`
#### Description
A function that converts an `IVariant` to a `FilterableVariant` or returns null if the input is undefined.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| selected | IVariant \| undefined | The variant to be converted |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | FilterableVariant \| null | The converted filterable variant or null |

#### Internal Logic
1. Check if the input `selected` is defined
2. If defined, return an object with all properties of `selected`, plus `displayValue` set to `selected.name` and `filterValue` set to `selected.id`
3. If undefined, return null
