## High-level description
The `Pagination` component is a reusable React component that provides pagination functionality for tables or lists. It displays navigation controls to move between pages and allows users to change the number of items displayed per page.

## Code Structure
The component consists of two main parts: the `Page` component for rendering individual page links, and the main `Pagination` component that orchestrates the overall pagination functionality.

## Symbols

### `Page`
#### Description
A sub-component that renders individual page links or ellipsis for skipped pages.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| page | number \| string | The page number or ellipsis string |
| currentPage | number | The currently active page |
| onPageChange | (page: number) =&gt; void | Callback function for page change |

#### Internal Logic
- Renders an ellipsis if the `page` is a string
- Renders a clickable link for numeric pages, highlighting the current page

### `Pagination`
#### Description
The main component that renders the entire pagination control.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| className | string | Optional CSS class for styling |
| currentPage | number | The currently active page |
| totalCount | number | Total number of items |
| pageSize | number | Number of items per page |
| onPageChange | (page: number, size: number) =&gt; void | Callback for page or size change |

#### Internal Logic
- Uses the `usePagination` hook to calculate the pagination range
- Renders "Previous" and "Next" navigation buttons
- Renders individual page links using the `Page` component
- Provides a dropdown to change the number of items per page
- Hides the component if the total count is less than or equal to the smallest page size

## Performance Considerations
The component uses `useMemo` to optimize the calculation of whether a page is the current page, reducing unnecessary re-renders.

This Pagination component is designed to be flexible and reusable across different parts of the application where pagination is needed, such as in the various table components (FlagTable, SegmentTable, TokenTable, etc.) mentioned in the related files.
