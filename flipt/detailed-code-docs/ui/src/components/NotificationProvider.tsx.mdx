## High-level description
The `NotificationProvider.tsx` file defines a React context and provider component for managing error and success notifications in a React application. It provides a centralized way to handle and display notifications across different components.

## Code Structure
The code defines two main parts: the `NotificationContext` and the `NotificationProvider` component. The context holds the state and functions for managing errors and success messages, while the provider component wraps the application and provides the context values to its children.

## Symbols

### `NotificationContext`
#### Description
A React context created to hold the error and success notification states and their respective setter functions.

#### Inputs
N/A (Context creation)

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| NotificationContext | React.Context | A context object for notifications |

### `NotificationProvider`
#### Description
A React component that provides the `NotificationContext` to its children, managing the error and success states.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| children | React.ReactNode | Child components to be wrapped by the provider |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| JSX.Element | React component | The provider component wrapping its children |

#### Internal Logic
1. Uses `useState` to manage `error` and `success` states.
2. Defines a `setError` function that uses `getErrorMessage` utility to format error messages.
3. Provides context value with error and success states, along with setter and clear functions.

- The `getErrorMessage` function is imported from `~/utils/helpers`.
- This component is used in `ui/src/app/auth/Login.tsx` and `ui/src/app/Layout.tsx`.
- Custom hooks `useError` and `useSuccess` in `ui/src/data/hooks/error.ts` and `ui/src/data/hooks/success.ts` respectively use this context.

## Notes
- The component uses TypeScript for type safety.
- It provides a reusable way to manage notifications across the application.
- The error handling includes conversion of unknown error types to string messages.
