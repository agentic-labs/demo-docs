## High-level description
The `Footer` component is a React functional component that renders the footer section of the Flipt application. It displays version information, copyright notice, links to social media, and additional resources like the changelog and feedback page.

## Code Structure
The component imports necessary dependencies, defines helper functions for version information, and renders the footer content using JSX.

## References
This component references the `selectInfo` selector from the `metaSlice`, which is likely defined in the `ui/src/app/meta/metaSlice.ts` file.

## Symbols

### Footer
#### Description
The main functional component that renders the footer content.

#### Inputs
This component doesn't accept any props.

#### Outputs
Returns a JSX element representing the footer.

#### Internal Logic
1. Retrieves application info using the `useSelector` hook and `selectInfo` selector.
2. Defines helper functions `ref` and `refURL` to determine the version or commit information and corresponding URL.
3. Defines an array of social media links.
4. Renders the footer content, including:
   - Version/commit information (if available)
   - Copyright notice
   - Links to Changelog and Feedback
   - Social media icons

### ref
#### Description
A helper function that determines the version or commit reference to display.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| reference | string | The version or first 7 characters of the commit hash |

#### Internal Logic
- If it's a release version, return the version.
- If it's a commit, return the first 7 characters of the commit hash.
- Otherwise, return an empty string.

### refURL
#### Description
A helper function that determines the URL for the version or commit reference.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| url | string | The GitHub URL for the version or commit |

#### Internal Logic
- If it's a release version, return the GitHub release URL.
- If it's a commit, return the GitHub commit URL.
- Otherwise, return the main Flipt GitHub repository URL.

## Performance Considerations
The component uses `useSelector` to access the Redux store, which may cause re-renders when the selected state changes. However, this is unlikely to be a performance bottleneck in a footer component.
