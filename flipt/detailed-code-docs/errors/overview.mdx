## High-level description
This code defines a custom error package for a Go application. It provides utility functions for error handling, type assertions, and defines several custom error types. The package aims to simplify error creation, checking, and handling in the application.

## Code Structure
The code defines several utility functions (`As`, `AsMatch`, `New`, `NewErrorf`) and custom error types (`ErrNotFound`, `ErrInvalid`, `ErrValidation`, `ErrCanceled`, `ErrUnauthenticated`, `ErrUnauthorized`). These types and functions are designed to work together to provide a comprehensive error handling system.

## Symbols

### `As[E error](err error) (e E, _ bool)`
#### Description
A generic function that attempts to assert an error to a specific error type.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| err | error | The error to be asserted |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| e | E | The asserted error of type E |
| _ | bool | Whether the assertion was successful |

#### Internal Logic
Uses `errors.As` to perform the type assertion.

### `AsMatch[E error](err error) (match bool)`
#### Description
Similar to `As`, but only returns whether the error matches the specified type.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| err | error | The error to be checked |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| match | bool | Whether the error matches the specified type |

### Custom Error Types
The code defines several custom error types:
- `ErrNotFound`
- `ErrInvalid`
- `ErrValidation`
- `ErrCanceled`
- `ErrUnauthenticated`
- `ErrUnauthorized`

Each of these types implements the `error` interface and provides custom `Error()` methods.

### Utility Functions
- `InvalidFieldError(field, reason string) error`: Creates an `ErrValidation` for a specific field and reason.
- `EmptyFieldError(field string) error`: Creates an `ErrValidation` for an empty field.

## Error Handling
This package is primarily focused on error handling. It provides a set of custom error types and utility functions to create, check, and handle errors in a more structured way. The use of generics allows for type-safe error checking and creation.

The custom error types (`ErrNotFound`, `ErrInvalid`, etc.) allow for more specific error handling in the application. The `As` and `AsMatch` functions provide a convenient way to check for specific error types.
