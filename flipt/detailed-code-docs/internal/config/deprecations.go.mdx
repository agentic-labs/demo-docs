## High-level description
This code defines a package `config` that handles deprecated configuration fields in a software system. It provides a mechanism to manage and display deprecation messages for specific configuration options.

## Code Structure
The code defines a custom type `deprecated` and a map `deprecatedFields` to store deprecated fields and their corresponding messages. It also provides a method `Message()` for the `deprecated` type to retrieve the appropriate deprecation message.

### `Message()`
#### Description
A method of the `deprecated` type that returns the appropriate deprecation message for a given deprecated field.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| d | deprecated | The deprecated field |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| message | string | The formatted deprecation message |

#### Internal Logic
1. Check if the deprecated field exists in the `deprecatedFields` map.
2. If it exists, combine the default message with the specific message for the field.
3. If it doesn't exist, use only the default message.
4. Format the message with the deprecated field name and return it.
