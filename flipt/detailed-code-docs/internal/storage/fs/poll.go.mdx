## High-level description
The `poll.go` file defines a `Poller` struct and its associated methods. This `Poller` is designed to periodically check for updates using a provided `UpdateFunc` and trigger actions based on the update result.

## Code Structure
The `Poller` struct uses a `context.Context` for cancellation and a `time.Ticker` for scheduling updates. The `Poll` method runs the update loop, logging errors and debug information. The `Close` method gracefully stops the poller.

### `NewPoller`
#### Description
`NewPoller` creates a new `Poller` instance with the provided configuration.

#### Inputs
| Name   | Type                          | Description                                    |
|:-------|:------------------------------|:-----------------------------------------------|
| logger | `*zap.Logger`                  | The logger to use for the poller.              |
| ctx    | `context.Context`              | The context for the poller.                    |
| update | `UpdateFunc`                   | The function to call for updates.              |
| opts   | `...containers.Option[Poller]` | Optional configuration options for the poller. |

#### Outputs
| Type      | Description                   |
|:----------|:------------------------------|
| `*Poller` | The newly created Poller instance. |

#### Internal Logic
1. Creates a new `Poller` with default values.
2. Applies the provided options to the `Poller`.
3. Creates a new context with cancellation from the provided context.
4. Returns the configured `Poller`.

### `Close`
#### Description
`Close` stops the poller and waits for it to finish.

#### Inputs
N/A

#### Outputs
| Type  | Description                          |
|:------|:-------------------------------------|
| error | Any error that occurred during closing. |

#### Internal Logic
1. Calls the cancel function to stop the poller.
2. Waits for the `done` channel to be closed, indicating the poller has finished.
3. Returns nil as there are no errors to report.

### `Poll`
#### Description
`Poll` is the main polling loop that periodically calls the update function and handles the results.

#### Inputs
N/A

#### Outputs
N/A

#### Internal Logic
1. Defers closing the `done` channel to signal when polling has finished.
2. Creates a new ticker with the specified interval.
3. Enters a loop that:
   - Waits for either the context to be done or the ticker to tick.
   - If the context is done, returns from the function.
   - If the ticker ticks, calls the update function.
   - Handles any errors from the update function.
   - Calls the notify function if set.
   - Logs debug information about the update status.
