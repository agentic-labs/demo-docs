## High-level description
The code in `middleware_test.go` defines unit tests for the HTTP middleware used in the Flipt authentication system. It specifically tests the behavior of the `Handler` and `ErrorHandler` functions, ensuring they correctly handle authentication status and cookie management.

## References
- `middlewarecommon.TokenCookieKey`: This constant, defined in another package, represents the key used for storing the Flipt client token in HTTP cookies.

## Symbols
### `TestHandler`
#### Description
This test function verifies the behavior of the `Handler` function when handling a successful request. It checks if the handler returns a 200 OK status code and clears the authentication cookies.

#### Inputs
This function takes no inputs.

#### Outputs
This function has no explicit outputs, but it uses `assert` statements to verify the expected behavior of the code under test.

#### Internal Logic
1. **Arrange:**
   - Defines a simple HTTP handler that returns a 200 OK status code.
   - Creates a new `HTTPMiddleware` instance with a test configuration.
   - Wraps the test handler with the middleware using `middleware.Handler`.
   - Creates a new HTTP request for the `/auth/v1/self/expire` endpoint.
   - Creates a new `httptest.ResponseRecorder` to record the response.
2. **Act:**
   - Executes the request by calling `srv.ServeHTTP`.
3. **Assert:**
   - Asserts that the response status code is 200 OK.
   - Retrieves cookies from the response.
   - Calls `assertCookiesCleared` to verify that the authentication cookies are cleared.

### `TestErrorHandler`
#### Description
This test function verifies the behavior of the `ErrorHandler` function when an authentication error occurs. It checks if the error handler clears the authentication cookies and invokes the default error handler.

#### Inputs
This function takes no inputs.

#### Outputs
This function has no explicit outputs, but it uses `assert` statements to verify the expected behavior of the code under test.

#### Internal Logic
1. **Arrange:**
   - Defines a default error handler that writes a specific response body.
   - Creates a new `HTTPMiddleware` instance with a test configuration.
   - Sets the `defaultErrHandler` of the middleware to the defined default error handler.
   - Creates a new HTTP request for the `/auth/v1/self/expire` endpoint.
   - Adds a cookie with an "expired" token to the request header.
   - Creates a new `httptest.ResponseRecorder` to record the response.
   - Creates a new gRPC error with the `Unauthenticated` code and "token expired" message.
2. **Act:**
   - Invokes the `ErrorHandler` with the prepared context, request, and error.
3. **Assert:**
   - Retrieves the response body and asserts that it matches the expected output from the default error handler.
   - Retrieves cookies from the response.
   - Calls `assertCookiesCleared` to verify that the authentication cookies are cleared.

### `assertCookiesCleared`
#### Description
This helper function asserts that the provided cookies include the `flipt_client_token` and `flipt_state` cookies, both with empty values and appropriate attributes for clearing the cookies.

#### Outputs
This function has no explicit outputs, but it uses `assert` statements to verify the expected state of the cookies.

#### Internal Logic
1. Asserts that there are exactly 2 cookies.
2. Creates a map of cookies using their names as keys.
3. Checks for the presence of both `stateCookieKey` and `middlewarecommon.TokenCookieKey` cookies.
4. For each cookie, asserts:
   - The value is an empty string.
   - The domain is set to "localhost".
   - The path is set to "/".
   - The MaxAge is set to -1 (which instructs the browser to delete the cookie).
