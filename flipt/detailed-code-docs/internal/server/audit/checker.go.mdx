## High-level description
The code defines a mechanism for checking if certain event pairs (representing actions performed on Flipt resources) should be emitted to configured sinks. It uses a `Checker` struct that holds a set of allowed event pairs, and provides methods to check for the existence of a given pair and retrieve the list of all allowed events.

## Code Structure
The code defines two interfaces: `EventPairChecker` and `NoOpChecker`, and a struct `Checker`. `Checker` implements `EventPairChecker` and uses a map to store allowed event pairs. The `NewChecker` function initializes a `Checker` with a given list of event pairs, validating the input and ensuring no duplicates exist.

#### Internal Logic
1. Defines maps for valid nouns and verbs.
2. Initializes an empty map for event actions.
3. Iterates through the input event pairs:
   - Splits each pair into noun and verb.
   - Validates the noun and verb against the predefined maps.
   - Expands wildcards (*) if present.
   - Generates all valid combinations of nouns and verbs.
   - Checks for duplicates and adds valid pairs to the event actions map.
4. Returns an error if no valid event pairs exist.
5. Returns a new Checker with the populated event actions map.

### `Check`
#### Description
This method checks if a given event pair exists in the Checker's event actions map.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| eventPair | string | The event pair to check for existence |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| bool | bool | True if the event pair exists, false otherwise |

#### Internal Logic
1. Checks if the Checker or its event actions map is nil.
2. Looks up the event pair in the map.
3. Returns true if the pair exists, false otherwise.

### `Events`
#### Description
This method returns a list of all event pairs stored in the Checker's event actions map.

#### Inputs
None

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| []string | []string | A slice containing all stored event pairs |

#### Internal Logic
1. Initializes a slice with capacity equal to the number of event pairs.
2. Iterates through the keys of the event actions map.
3. Appends each key (event pair) to the slice.
4. Returns the populated slice.

### `NoOpChecker`
#### Description
This is a no-operation implementation of the EventPairChecker interface.

#### Inputs
None

#### Outputs
None

#### Internal Logic
This struct has no internal logic. Its methods always return false for `Check` and an empty slice for `Events`.

## Error Handling
The `NewChecker` function includes error handling for invalid input:
- Returns an error for invalid event pair format.
- Returns an error for invalid nouns or verbs.
- Returns an error for repeated event pairs.
- Returns an error if no valid event pairs exist after processing.

## Performance Considerations
The `Checker` uses a map for O(1) lookup time when checking for the existence of an event pair. The `Events` method has O(n) time complexity where n is the number of stored event pairs.
