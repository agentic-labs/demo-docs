## High-level description
The `webhook` package provides a client for sending audit events to a configured webhook URL. It signs the payload using a provided secret and sends it as a JSON payload in an HTTP POST request.

## Code Structure
The `webhookClient` struct implements the `Client` interface, which defines the `SendAudit` method for sending audit events. The `signPayload` method is used internally to sign the payload before sending.

#### Description
The `signPayload` method generates a SHA256 HMAC signature of the provided payload using the configured signing secret.

#### Inputs

#### Outputs
| Name | Type     | Description                                       |
|:-----|:---------|:---------------------------------------------------|
|      | []byte   | The SHA256 HMAC signature of the payload         |

#### Internal Logic

2. Writes the payload to the HMAC object.
3. Returns the HMAC sum.

### `SendAudit`
#### Description
The `SendAudit` method sends an audit event to the configured webhook URL.

#### Inputs
| Name | Type         | Description                        |
|:-----|:-------------|:-----------------------------------|
| ctx  | context.Context | The context for the request       |
| e    | audit.Event  | The audit event to be sent         |

#### Outputs
| Name | Type           | Description                                |
|:-----|:---------------|:-------------------------------------------|
| resp | *http.Response | The HTTP response from the webhook server  |
| err  | error          | Any error that occurred during the process |

#### Internal Logic
1. Marshals the audit event to JSON.
2. Creates a new HTTP POST request with the marshaled event as the body.
3. Sets the "Content-Type" header to "application/json".
4. If a signing secret is configured, signs the payload and adds the signature to the "x-flipt-webhook-signature" header.
5. Sends the request using the retryable HTTP client.
6. Returns the response and any error.

## Side Effects
- The `SendAudit` method makes an HTTP POST request to the configured webhook URL.

## Error Handling
The `SendAudit` method returns any errors encountered during the process of sending the audit event. This includes errors from JSON marshaling, creating the HTTP request, or sending the request.

## Logging
Logging is handled through the `zap.Logger` instance passed to the `NewWebhookClient` function. The actual logging calls are not visible in this file but are likely handled by the `retryablehttp.Client`.

## Performance Considerations
- The use of `retryablehttp.Client` provides built-in retry functionality, which can help with reliability in case of temporary network issues or server unavailability.
- The `maxBackoffDuration` parameter in `NewWebhookClient` allows for configuration of the maximum time to wait between retries, which can affect the overall performance and resource usage of the client.
