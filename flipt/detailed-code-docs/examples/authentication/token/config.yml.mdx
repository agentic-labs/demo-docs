## High-level description
This YAML file defines configuration options for a Flipt instance, specifically focusing on logging, database connection, and authentication settings. It enables token-based authentication with a bootstrapped token, cleanup settings, and logging verbosity.

## Configuration
This YAML file defines configuration options for a Flipt instance.

| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| `log.level` | string |  | Sets the logging level. Possible values are `DEBUG`, `INFO`, `WARN`, `ERROR`. |
| `db.url` | string |  | Specifies the database connection URL. In this case, it points to a SQLite database file. |
| `authentication.required` | boolean |  | Determines if authentication is required to access the Flipt instance. |
| `authentication.methods.token.enabled` | boolean |  | Enables or disables token-based authentication. |
| `authentication.methods.token.bootstrap.token` | string |  | Sets the initial bootstrap token value. |
| `authentication.methods.token.bootstrap.expiration` | string |  | Defines the expiration time for the bootstrap token. |
| `authentication.methods.token.cleanup.interval` | string |  | Specifies the interval for cleaning up expired tokens. |
| `authentication.methods.token.cleanup.grace_period` | string |  | Sets the grace period for expired tokens before they are deleted. |
