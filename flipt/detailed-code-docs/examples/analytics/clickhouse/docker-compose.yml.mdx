## High-level description
This `docker-compose.yml` file defines a multi-container Docker application for running Flipt with ClickHouse for analytics. It sets up two services: `clickhouse` for the ClickHouse database and `flipt` for the Flipt application, which connects to the ClickHouse database for storing analytics data.

## Code Structure
The `flipt` service depends on the `clickhouse` service, ensuring that the ClickHouse database is running before Flipt starts. Both services are connected to the same network `flipt_network`.

## Symbols
### `services`
#### Description
This section defines the services that make up the application.

#### Internal Logic
* **clickhouse:**
    * Uses the `clickhouse/clickhouse-server` image.
    * Exposes ports `8123` and `9000`.
    * Sets ulimits for open files.
    * Connects to the `flipt_network`.
    * Sets the database name to `flipt_analytics`.
* **flipt:**
    * Builds the image from the current directory.
    * Depends on the `clickhouse` service.
    * Exposes port `8080`.
    * Connects to the `flipt_network`.
    * Restarts always on failure.
    * Sets environment variables for logging, enabling ClickHouse analytics storage, disabling telemetry, and configuring the ClickHouse connection URL.
    * Defines a command to wait for the ClickHouse service to be ready before starting Flipt and running database migrations.

### `networks`
#### Description
This section defines the networks used by the services.

#### Internal Logic
* **flipt_network:**
    * Defines a bridge network named `flipt_network` used for communication between the services.

## Dependencies
This configuration file uses Docker Compose, which is assumed to be installed in the environment.

## Error Handling
This configuration does not explicitly define error handling mechanisms. However, the `restart: always` option for the `flipt` service ensures that the container will be restarted if it exits unexpectedly.

## Performance Considerations
The `clickhouse` service has `ulimits` set for the maximum number of open files, which can impact performance:

```yaml
ulimits:
  nofile:
    soft: 262144
    hard: 262144
```

This setting allows ClickHouse to handle a large number of concurrent connections and file operations, which is important for database performance.

Additionally, the use of a wait script (`./tmp/wait-for-it.sh`) ensures that the Flipt service doesn't start until the ClickHouse service is ready, preventing potential startup errors and improving overall system stability.
