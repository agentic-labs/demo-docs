## High-level description
This file is a configuration file for the Buf (Buffer) tool, which is used for managing Protocol Buffers. It defines various plugins and their configurations for generating code and documentation from Protocol Buffer definitions, specifically for the Flipt project.

## Symbols

### `version`
#### Description
Specifies the version of the Buf configuration file format being used.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| version | string | The version identifier |

### `plugins`
#### Description
Defines a list of plugins to be used by Buf for code generation and documentation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| plugins | array | List of plugin configurations |

Each plugin configuration in the `plugins` array has the following structure:

### Plugin Configuration
#### Description
Configures a specific plugin for code generation or documentation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| name | string | The name of the plugin |
| out | string | The output directory for generated files |
| opt | array | Optional configuration parameters for the plugin |
| strategy | string | The generation strategy (if applicable) |

## Plugins

1. **go**
   - Generates Go code for core APIs
   - Output: `rpc/flipt`
   - Options: 
     - `paths=source_relative`
     - Custom import path for gnostic annotations
   - Strategy: `all`

2. **go-grpc**
   - Generates Go gRPC code
   - Output: `rpc/flipt`
   - Options: `paths=source_relative`
   - Strategy: `all`

3. **grpc-gateway**
   - Generates gRPC-Gateway code
   - Output: `rpc/flipt`
   - Options:
     - `paths=source_relative`
     - Uses `rpc/flipt/flipt.yaml` for gRPC API configuration
   - Strategy: `all`

4. **go-flipt-sdk**
   - Generates Go SDK for Flipt
   - Output: `sdk/go`
   - Options:
     - `paths=source_relative`
     - Uses `rpc/flipt/flipt.yaml` for gRPC API configuration
   - Strategy: `all`

5. **buf.build/community/google-gnostic-openapi**
   - Generates OpenAPI documentation
   - Output: `.` (current directory)
   - Options:
     - Specifies OpenAPI version, schema naming, response handling, enum type, title, and description

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| buf | Protocol Buffer management and code generation tool |
| gnostic | Used for OpenAPI documentation generation |

This configuration file sets up a comprehensive code generation pipeline for the Flipt project, covering Go code generation, gRPC services, SDK creation, and API documentation. It ensures consistent output paths and uses specific configurations for each plugin to tailor the generated code and documentation to the project's needs.
