## High-level description
The `ViewAddOns` component displays a list of add-ons. It fetches add-on data from the API and renders an `AddOnsCard` for each add-on. If no add-ons exist, it displays a message prompting the user to create one.

## Symbols

### `ViewAddOns`
#### Description
This functional component fetches add-on data and renders the appropriate view based on the data. It also handles navigation to the add-on creation page.

#### Inputs
This is a functional component and does not accept any props.

#### Outputs
Renders a `PageLayout` component containing either a list of `AddOnsCard` components or a message prompting the user to create an add-on.

#### Internal Logic
1. **Fetches add-on data:** Uses the `useQuery` hook from `@tanstack/react-query` to fetch add-on data from the `getAddOns` API endpoint.
2. **Handles loading state:** Displays a `LoadingSpinner` while the add-on data is being fetched.
3. **Renders add-on list:** If add-ons exist, it iterates through the `addOns` array and renders an `AddOnsCard` component for each add-on.
4. **Renders empty state:** If no add-ons exist, it displays a message prompting the user to create one.
5. **Handles add-on creation navigation:** Provides a button to navigate to the `/create-addons` route, handled by the `navigateCreatePlan` function.
