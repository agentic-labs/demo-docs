## High-level description
This file defines types and interfaces related to payment processors in the Lotus application. It includes types for payment processor status, connection requests and responses, settings, and integration details. The file also defines a mapping of integration names to their display information.

## Code Structure
The file defines several interconnected types and interfaces that are used throughout the application to handle payment processor integrations. The `integrationsMap` object at the end of the file provides a centralized source of information for various integrations.

## Symbols

### `PaymentProcessors`
#### Description
A constant array of supported payment processor names.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| PaymentProcessors | readonly ["stripe", "braintree"] | Array of supported payment processor names |

### `PaymentProcessorType`
#### Description
A type representing the supported payment processors.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| PaymentProcessorType | "stripe" \| "braintree" | Union type of supported payment processors |

### `PaymentProcessorStatusType`
#### Description
An interface representing the status of a payment processor connection.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| payment_provider_name | PaymentProcessorType | Name of the payment processor |
| connected | boolean | Whether the processor is connected |
| redirect_url | string | URL for redirecting to the processor's connection page |
| self_hosted | boolean | Whether the processor is self-hosted |
| connection_id | string | Unique identifier for the connection |
| working | boolean | Whether the connection is working |
| account_id | string | Account ID for the processor |

### `PaymentProcessorConnectionResponseType`
#### Description
An interface representing the response from a payment processor connection attempt.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| payment_processor | string | Name of the payment processor |
| success | boolean | Whether the connection was successful |
| details | string | Additional details about the connection |

### `PaymentProcessorConnectionRequestType`
#### Description
An interface representing a request to connect a payment processor.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| payment_processor | PaymentProcessorType | The payment processor to connect |
| data | StripeConnectionRequestType \| BraintreeConnectionRequestType | Connection data specific to the processor |

### `StripeConnectionRequestType`
#### Description
An interface for Stripe-specific connection request data.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| authorization_code | string | Authorization code for Stripe connection |

### `BraintreeConnectionRequestType`
#### Description
An interface for Braintree-specific connection request data.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|

### `PaymentProcessorImportCustomerResponse`
#### Description
An interface representing the response from importing customers from a payment processor.

| Name | Type | Description |
|:-----|:-----|:------------|
| status | string | Status of the import operation |
| detail | string | Details about the import operation |

### `Source`
#### Description

#### Outputs
| Name | Type | Description |

#### Description
An interface for payment processor settings parameters.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| setting_group | PaymentProcessorType | The payment processor group |
| setting_name? | string | Optional setting name |

### `PaymentProcessorSetting`
#### Description
An interface representing a payment processor setting.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| setting_group | PaymentProcessorType | The payment processor group |
| setting_id | string | Unique identifier for the setting |
| setting_name | string | Name of the setting |
| setting_values | any | Values for the setting |

### `UpdatePaymentProcessorSettingParams`
#### Description
An interface for updating payment processor settings.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| setting_id | string | ID of the setting to update |
| setting_values | boolean | New value for the setting |

### `integrationsMap`
#### Description
An object mapping integration names to their display information.

#### Internal Logic
Defines display information (name, icon, description, account ID name) for various integrations including Stripe, Braintree, Snowflake, Salesforce, NetSuite, and Segment.
