## High-level description
This code defines types and interfaces for experiments and backtests in a pricing analysis system. It includes structures for representing backtest results, substitutions, and various metrics related to revenue and customer behavior.

## Code Structure
The code defines several interconnected interfaces and types that represent different aspects of experiments and backtests. These types are used across multiple components and pages to handle experiment data, display results, and manage the creation of new experiments.

## Symbols

### `BacktestType`
#### Description
Represents the basic information of a backtest experiment.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| backtest_name | string | Name of the backtest |
| backtest_id | string | Unique identifier for the backtest |
| status | string | Current status of the backtest |
| start_date | string | Start date of the backtest period |
| end_date | string | End date of the backtest period |
| time_created | string | Timestamp of when the backtest was created |
| kpis | string[] | Array of Key Performance Indicators for the backtest |

### `BacktestResultType`
#### Description
Extends `BacktestType` to include detailed results of the backtest.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| backtest_subsitutions | SubstitutionResults[] | Array of substitution results |
| backtest_results | ResultsOverview | Overview of the backtest results |

### `ResultsOverview`
#### Description
Provides an overview of the backtest results, including revenue comparisons.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| new_plans_revenue | number | Revenue from new plans |
| original_plans_revenue | number | Revenue from original plans |
| pct_revenue_change | number | Percentage change in revenue |
| substitution_results | SpecificResults[] | Detailed results for each substitution |

### `SpecificResults`
#### Description
Contains detailed results for a specific plan substitution.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| substitution_name | string | Name of the substitution |
| pct_revenue_change | number | Percentage change in revenue |
| new_plan | object | Details of the new plan |
| original_plan | object | Details of the original plan |
| results | object | Detailed results including revenue charts and customer data |

### `CreateBacktestType`
#### Description
Represents the data structure for creating a new backtest.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| backtest_name | string | Name of the new backtest |
| start_date | string | Start date for the backtest period |
| end_date | string | End date for the backtest period |
| kpis | string[] | Array of KPIs to be analyzed |
| substitutions | Substitution[] | Array of plan substitutions to be tested |

### `Substitution`
#### Description
Defines a plan substitution for the backtest.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| new_plan | string | Identifier for the new plan |
| original_plans | string[] | Array of identifiers for original plans |
| new_plan_name | string | Name of the new plan |
| original_plan_names | string[] | Array of names for original plans |
