## High-level description
This code defines TypeScript interfaces and types related to revenue, cost, and customer data for a frontend application. It includes structures for representing revenue periods, cost types, and customer cost data, which are likely used for financial reporting and analysis within the application.

## Symbols

### `RevenueType`
#### Description
An interface representing revenue data for two periods.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| total_revenue_period_1 | number | Total revenue for the first period |
| total_revenue_period_2 | number | Total revenue for the second period |
| earned_revenue_period_1 | number | Earned revenue for the first period |
| earned_revenue_period_2 | number | Earned revenue for the second period |
| daily_usage_revenue_period_1 | number | Daily usage revenue for the first period |

### `RevenueData`
#### Description
An interface representing a single data point of revenue for a specific date.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| date | string | The date of the revenue data point |
| metric_revenue | number | The revenue amount for the given date |

### `RevenuePeriod`
#### Description
An interface representing revenue data for a specific period and metric.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| total_revenue | number | The total revenue for the period |
| data | RevenueData[] | An array of revenue data points |
| metric | string | The metric associated with this revenue period |

### `CostType`
#### Description
An interface representing a cost associated with a specific metric.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| cost | number | The cost amount |
| metric | MetricType | The metric associated with this cost |

### `CustomerCostType`
#### Description
An interface representing detailed cost and revenue data for a customer.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| per_day | { date: string; cost_data: CostType[]; revenue: number }[] | Daily cost and revenue data |
| total_cost | number | Total cost for the customer |
| total_revenue | number | Total revenue for the customer |
| profit_margin | number | Profit margin as a decimal |
| markup | number | Markup as a decimal |

## Dependencies
This file depends on the `MetricType` imported from "./metric-type".

## Notes
- The `RevenueType` interface seems to be designed for comparing revenue across two periods.
- The `CustomerCostType` interface provides a comprehensive view of a customer's financial data, including daily breakdowns and overall metrics like profit margin and markup.
- These types are likely used in components and API calls related to financial reporting and customer analysis in the frontend application.
