## High-level description
This code defines a global store using Zustand for managing application-wide state in a React application. It includes state for user information, organization details, environment type, and quick start progress. The store provides methods to update these state values.

## Code Structure
The code defines a `GlobalStoreState` type and an `IQuickStartStoreType` interface. It then creates a Zustand store (`useGlobalStore`) with initial state and setter functions for each state property.

## Symbols

### `GlobalStoreState`
#### Description
An interface defining the structure of the global store state.

#### Internal Logic
Contains properties for username, organization, environment type, quick start progress, and setter functions for each.

### `IQuickStartStoreType`
#### Description
An interface defining the structure of the quick start progress state.

#### Internal Logic
Contains boolean flags for different steps in the quick start process.

### `useGlobalStore`
#### Description
A Zustand store that manages the global state of the application.

#### Internal Logic
- Initializes the state with default values.
- Provides setter functions for updating each state property.

### `setUsername`
#### Description
A function to update the username in the global store.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| username | string | The new username to set |

### `setOrgInfo`
#### Description
A function to update the organization information in the global store.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| org | components["schemas"]["Organization"] | The new organization information to set |

### `setQuickStartProgress`
#### Description
A function to update the quick start progress in the global store.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| quickStartProgress | IQuickStartStoreType | The new quick start progress to set |

### `setEnvironmentType`
#### Description
A function to update the environment type in the global store.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| environmentType | "Production" \| "Development" \| "Demo" \| "Internal Demo" | The new environment type to set |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| zustand | State management library used to create the global store |
| ../gen-types | Imports component types used in the store |

This global store provides a centralized way to manage and access application-wide state, making it easier to share data between components and manage complex state interactions.
