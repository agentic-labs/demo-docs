## High-level description
The `AddOnInfo` component displays detailed information about a specific add-on. It presents data like Add-on ID, price, type, and billing frequency in a structured format.

## Code Structure
The `AddOnInfo` component uses the `constructBillType` function to format the billing type string. It also utilizes helper functions `capitalize` and `createShortenedText` for text transformations. The `CopyText` component is used to provide a copy-to-clipboard functionality for the Add-on ID.

## Symbols

### `constructBillType`
#### Description
This function takes a string representing a billing type and formats it for display. If the string contains underscores, it replaces them with spaces and capitalizes each word. Otherwise, it returns the capitalized string.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| str | string | The billing type string to format. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| - | string | The formatted billing type string. |

### `AddOnInfo`
#### Description
This component displays detailed information about an add-on.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| addOnInfo | components["schemas"]["AddOnDetail"] | An object containing the add-on details. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| - | JSX.Element | The rendered add-on information. |

#### Internal Logic
The component uses a grid layout to display the add-on information in two columns on larger screens. It retrieves the add-on ID, price, type, and billing frequency from the `addOnInfo` prop and displays them in separate rows. The `createShortenedText` function is used to shorten the Add-on ID if it exceeds a certain length, and the `CopyText` component allows users to copy the full ID. The `constructBillType` function is used to format the billing type and frequency strings.
