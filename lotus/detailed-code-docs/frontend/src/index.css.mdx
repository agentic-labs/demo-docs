## High-level description
This CSS file defines the global styles for a web application. It imports external fonts, sets up custom font faces, and applies various styles to HTML elements and Ant Design components. The file also includes utility classes and custom styling for specific components and layouts.

## Symbols

### @import statements
#### Description
These statements import external font resources from Google Fonts.

### @font-face rules
#### Description
These rules define custom fonts to be used in the application, including Menlo, Alliance2, and Satoshi.

### Utility Classes
#### Description
Several utility classes are defined for font families and other common styles.

### Root Styles
#### Description
The `:root` selector sets global variables and default font family.

### Element Styles
#### Description
Styles for HTML elements like `h1`, `h2`, `h3`, `html`, `body`, and `a` are defined.

### Ant Design Component Overrides
#### Description
Custom styles for Ant Design components such as Modal, Descriptions, Input, Select, Tabs, Switch, and Table are defined to match the application's design requirements.

### Custom Component Styles
#### Description
Styles for custom components like customer detail cards and API key display are defined.

## Dependencies

| Tailwind CSS | Utility-first CSS framework |
| Ant Design | UI component library |

## Side Effects
This CSS file has global side effects on the styling of the entire application, affecting both custom and third-party components.

## Performance Considerations
1. The file imports multiple font files, which may impact initial page load time.
2. The use of `!important` in some rules may lead to specificity issues and make future style changes more difficult.

## Notes
1. The file uses a mix of custom styles and overrides for Ant Design components.
2. There's heavy use of the Alliance2 font family throughout the application.
3. Some styles are specifically tailored for responsive design and layout adjustments.
4. The file includes styles for a custom side menu with a unique selected item indicator.
5. There are custom styles for form inputs, including required field indicators and validation states.
