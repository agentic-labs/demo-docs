## High-level description
This file, `pyproject.toml`, is a configuration file for various Python development tools used in the backend of the Lotus project. It defines settings for code style checking (Ruff), dependency management (Poetry), and build system configuration.

## Code Structure
This file uses the TOML format to structure configuration settings for different tools. Each section (`[tool.name]`) corresponds to a specific tool, and nested key-value pairs define the tool's configuration options.

## Symbols
### `[tool.ruff]`
#### Description
This section configures the Ruff linter, which is used for enforcing code style and quality.

#### Inputs
N/A - This section defines configuration options, not inputs.

#### Outputs
N/A - This section defines configuration options, not outputs.

#### Internal Logic
- `line-length = 99`: Sets the maximum allowed line length to 99 characters.
- `select = ["E", "F"]`: Enables Pyflakes error codes `E` (style violations) and `F` (syntax errors) by default.
- `ignore = ["E501"]`: Ignores the Pyflakes error code `E501` (line too long).
- `exclude`: Lists directories to be excluded from linting.
- `per-file-ignores`: Allows specifying ignore rules for individual files (currently empty).
- `dummy-variable-rgx`: Defines a regular expression to identify dummy variables (prefixed with underscores) and allows their unused occurrences.
- `target-version = "py39"`: Sets the target Python version to 3.9 for compatibility checks.
