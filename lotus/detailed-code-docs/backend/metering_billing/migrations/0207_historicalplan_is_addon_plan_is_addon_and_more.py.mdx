## High-level description
This migration file adds new fields to the `HistoricalPlan`, `Plan`, and `PlanVersion` models in the `metering_billing` app. It introduces an `is_addon` boolean field for plans and a related `addon_spec` field for plan versions.

## Symbols

### Migration
#### Description
This Django migration adds new fields to existing models in the `metering_billing` app.

#### Internal Logic
The migration performs the following operations:
1. Adds an `is_addon` boolean field to the `HistoricalPlan` model with a default value of `False`.
2. Adds an `is_addon` boolean field to the `Plan` model with a default value of `False`.
3. Adds an `addon_spec` field to the `PlanVersion` model, which is a `OneToOneField` relationship to the `AddonSpecification` model.
