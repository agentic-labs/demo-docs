## High-level description
This Django migration script modifies the `default_currency` field of the `Customer` model in the `metering_billing` app. It changes the field to allow blank and null values and sets it to use `SET_NULL` on delete.

#### Internal Logic
The migration sets the `default_currency` field of the `Customer` model to:
- Allow blank values (`blank=True`)
- Allow null values (`null=True`)
- Use `models.SET_NULL` on delete, meaning if the related `PricingUnit` is deleted, the `default_currency` field will be set to null.
- Set `related_name` to '+', preventing the creation of a reverse relation from `PricingUnit` back to `Customer`.
