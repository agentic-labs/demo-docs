## High-level description
This directory contains documentation for the anomaly detection feature of Elementary Cloud Platform. The documentation focuses on automated monitors, which are pre-configured anomaly detection rules that run on metadata collected by the platform. 

## What does it do?
This documentation describes how Elementary Cloud Platform can be used to automatically detect anomalies in data. The platform collects metadata about data, such as the time of last update and the volume of data. This metadata is then used to train an anomaly detection model. The model can then be used to identify anomalies, such as unexpected delays in data updates or spikes in data volume. Users can also provide feedback on the detected anomalies, which helps to improve the accuracy of the model over time. 

## Entry points
The main entry point for this documentation is `docs/features/anomaly-detection/monitors-overview.mdx`. This file provides an overview of the anomaly detection feature and links to other relevant files in the directory. 

The documentation is organized conceptually into the following sections:

1. **Introduction to Anomaly Detection:** `monitors-overview.mdx`
2. **Automated Monitors:** `automated-monitors.mdx`, `automated-freshness.mdx`, `automated-volume.mdx`
3. **Opt-in Monitors:** `opt-in-monitors.mdx`
4. **Monitor Configuration and Feedback:** `monitors-configuration.mdx`, `monitors-feedback.mdx`, `disable-or-mute-monitors.mdx`

The data flow and control flow of the documentation follows this conceptual organization, guiding the user from a general understanding of anomaly detection to specific monitor types and their configuration.
