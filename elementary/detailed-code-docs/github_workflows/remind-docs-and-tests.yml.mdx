## High-level description
This GitHub Actions workflow, named "Remind docs and tests," is designed to automatically comment on newly opened pull requests to the master branch. The comment reminds the pull request author to add tests and document any user-facing changes.

## Code Structure
The workflow consists of a single job with one main step that uses the `wow-actions/auto-comment` action to post the reminder comment.

## Symbols

### Workflow
#### Description
This GitHub Actions workflow is triggered when a pull request is opened against the master branch. It runs on the latest Ubuntu environment and executes a single job.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| pull_request | GitHub event | The pull request that triggered the workflow |

#### Internal Logic
1. The workflow is triggered on pull request events targeting the master branch.
2. It runs a single job on the latest Ubuntu environment.
3. The job uses the `wow-actions/auto-comment@v1` action to post a comment on the pull request.
4. The comment mentions the pull request author and reminds them to add tests and document user-facing changes.

### wow-actions/auto-comment@v1
#### Description
This is a GitHub Action used to automatically post comments on pull requests.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| GITHUB_TOKEN | Secret | GitHub token for authentication |
| pullRequestOpened | String | The comment to be posted when a pull request is opened |

#### Outputs
This action doesn't produce any outputs. Its effect is posting a comment on the pull request.
