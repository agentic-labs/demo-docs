## High-level description
This GitHub Actions workflow is designed to automatically manage stale issues and pull requests in a repository. It runs on a daily schedule to identify and label inactive issues and PRs, and eventually close them if they remain inactive for an extended period.

## Code Structure
The workflow consists of a single job named "close-issues" that uses the `actions/stale@v5` action to handle the stale issue and PR management process.

## Symbols

### Workflow
#### Description
This GitHub Actions workflow, named "Close stale issues and PRs", is configured to run automatically on a schedule to manage stale issues and pull requests.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| schedule | cron | Defines when the workflow should run |

#### Internal Logic
1. Runs daily at 1:30 UTC (defined by the cron schedule)
2. Sets up a job named "close-issues" on an Ubuntu runner
3. Configures necessary permissions for the job
4. Uses the `actions/stale@v5` action with specific parameters to manage stale issues and PRs

### Job: close-issues
#### Description
This job is responsible for identifying and managing stale issues and pull requests using the `actions/stale` action.

#### Inputs
N/A (Uses default runner and permissions)

#### Internal Logic
1. Runs on an Ubuntu latest runner
2. Sets up permissions for contents, issues, and pull requests
3. Uses the `actions/stale@v5` action with specific configuration parameters

### Action: actions/stale@v5
#### Description
This action is used to identify and manage stale issues and pull requests based on inactivity.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| days-before-stale | number | Number of days before an issue/PR is considered stale |
| stale-issue-message | string | Message to post when an issue becomes stale |
| close-issue-message | string | Message to post when closing a stale issue |
| stale-pr-message | string | Message to post when a PR becomes stale |
| close-pr-message | string | Message to post when closing a stale PR |

#### Internal Logic
1. Identifies issues and PRs that have been inactive for 180 days
2. Labels them as stale and posts a stale message
3. If they remain inactive, closes them and posts a closing message

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| days-before-stale | number | 180 | Number of days before an issue/PR is considered stale |
