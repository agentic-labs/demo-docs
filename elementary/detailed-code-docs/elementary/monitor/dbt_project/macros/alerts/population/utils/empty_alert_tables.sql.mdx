## High-level description
This SQL file contains three macros that define empty table structures for different types of alerts in the Elementary monitoring system. These macros are used to create placeholder tables with predefined column structures for storing alert data related to models, source freshness, and a generic alerts table (v2).

## Symbols

### `empty_alerts_models()`
#### Description
This macro creates an empty table structure for storing alerts related to dbt models.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Empty table | Table | A table with columns for various model alert attributes |

#### Internal Logic
The macro uses the `elementary.empty_table()` function to create a table with specified columns and their data types. The columns include information such as alert ID, unique ID, detection time, database details, materialization type, file path, schema name, alert message, owners, tags, status, and other relevant metadata.

### `empty_alerts_source_freshness()`
#### Description
This macro creates an empty table structure for storing alerts related to source freshness checks.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Empty table | Table | A table with columns for various source freshness alert attributes |

#### Internal Logic
Similar to the previous macro, this one uses `elementary.empty_table()` to create a table with columns specific to source freshness alerts. The columns include information about the alert ID, maximum loaded timestamp, snapshot time, detection time, freshness status, error details, source identifiers, freshness thresholds, and other relevant metadata.

### `empty_alerts_v2()`
#### Description
This macro creates an empty table structure for a more generic alerts table, possibly a newer version of the alert system.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Empty table | Table | A table with columns for a generic alert structure |

#### Internal Logic
This macro also uses `elementary.empty_table()` to create a table with a more streamlined set of columns. It includes basic alert information such as alert ID, class ID, type, detection time, creation and update times, status, data (as a long string), and sent time.
