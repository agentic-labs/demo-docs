## High-level description
This code defines utility functions for retrieving the package version of the `elementary-data` package. It can get both the locally installed version and the latest version from PyPI.

## References
This code references the `requests` library for making HTTP requests.

## Symbols

### `get_package_version`
#### Description
This function retrieves the version of the locally installed `elementary-data` package.

#### Inputs
This function has no inputs.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| (return value) | str | The version string of the installed `elementary-data` package. |

#### Internal Logic
The function uses the `importlib.metadata.version` function to get the version of the installed package.

### `get_latest_package_version`
#### Description
This function retrieves the latest version of the `elementary-data` package from PyPI.

#### Inputs
This function has no inputs.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| (return value) | Optional[str] | The latest version string of the `elementary-data` package from PyPI, or `None` if there was an error fetching the information. |

#### Internal Logic
1. The function makes a GET request to the PyPI JSON API endpoint for the `elementary-data` package.
2. If the request is successful, it parses the JSON response to extract the version string.
3. If there is any error during the process, it returns `None`.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| requests | Used to make HTTP requests to the PyPI API. |
| importlib.metadata | Used to retrieve the locally installed package version. |
