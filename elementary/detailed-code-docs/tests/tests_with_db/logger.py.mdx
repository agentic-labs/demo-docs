## High-level description
This code defines a function `get_logger` that creates and configures a logger object. The logger is set up to output messages to the console (stdout) with a specific format, including timestamp, log level, and message.

## Symbols

### `get_logger`
#### Description
This function creates and configures a logger object with a specified name. It sets up a stream handler to output log messages to the console (stdout) and configures the logging format and level.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| logger_name | str | The name to be assigned to the logger |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| logger | logging.Logger | Configured logger object |

#### Internal Logic
1. Creates a `StreamHandler` that outputs to `sys.stdout`.
2. Sets a formatter for the handler with the format: "%(asctime)s — %(levelname)s — %(message)s".
3. Gets a logger instance with the provided `logger_name`.
4. Sets the logger's level to `DEBUG`.
5. Adds the configured handler to the logger.
6. Returns the configured logger.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| logging | Python's built-in logging module for creating and configuring loggers |
| sys | Used to access `sys.stdout` for console output |

## Logging
The code sets up a logging mechanism with the following characteristics:
- Output: Console (stdout)
- Log level: DEBUG
- Format: "%(asctime)s — %(levelname)s — %(message)s"
  - asctime: Timestamp of the log entry
  - levelname: Severity level of the log entry (e.g., DEBUG, INFO, WARNING, ERROR, CRITICAL)
  - message: The actual log message

This logging configuration allows for detailed debugging information to be output to the console, including the time of each log entry, its severity level, and the log message itself.
