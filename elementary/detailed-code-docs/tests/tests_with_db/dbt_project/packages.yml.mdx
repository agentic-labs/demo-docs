## High-level description
This YAML file defines the package dependencies for a dbt project. It specifies two packages: a local package and the dbt-labs/dbt_utils package with version constraints that depend on the dbt version being used.

## Symbols

### packages
#### Description
The `packages` key is the root-level element in this YAML file, containing a list of package dependencies for the dbt project.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| - | list | A list of package specifications |

### local package
#### Description
Specifies a local package dependency.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| local | string | The relative path to the local package |

### dbt-labs/dbt_utils package
#### Description
Specifies the dbt-labs/dbt_utils package dependency with a version constraint.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| package | string | The name of the package (dbt-labs/dbt_utils) |
| version | string | A Jinja template that determines the version constraint based on the dbt version |

#### Internal Logic
The version constraint for dbt-labs/dbt_utils is determined using a Jinja template:

1. A dictionary `minor_to_utils_range_map` is defined, mapping dbt minor versions to corresponding dbt_utils version ranges.
2. The dbt version is split, and the minor version number is extracted.
3. The appropriate version range is selected from the map based on the minor version.
4. If the minor version is not found in the map, a default range of `["&gt;=0.8.0", "&lt;2.0.0"]` is used.

This approach ensures compatibility between the dbt version and the dbt_utils package version.
