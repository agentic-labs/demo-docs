## High-level description
This code defines a pytest hook function `pytest_addoption` that adds a custom command-line option `--warehouse-type` to pytest. This allows users to specify a warehouse type when running pytest commands.

## Symbols

### `pytest_addoption`
#### Description
This function is a pytest hook that adds a custom command-line option to pytest. It allows users to specify a warehouse type when running tests.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| parser | `pytest.Parser` | The pytest command-line options parser |

#### Outputs
This function doesn't return any value. It modifies the parser in-place.

#### Internal Logic
The function uses the `parser.addoption()` method to add a new command-line option:
1. The option name is set to `"--warehouse-type"`.
2. The `action` is set to `"store"`, which means the option will store the provided value.
3. The `default` value is set to `None`, meaning if the option is not provided, it will default to `None`.

## Side Effects
This function modifies the pytest command-line parser, adding a new option that can be used when running tests.

| Dependency | Purpose |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| --warehouse-type | string | None | Allows specifying a warehouse type when running tests |

This custom option can be used when running pytest commands, like:
```
pytest --warehouse-type=some_type
```

The value of this option can then be accessed in test files or other pytest hooks using the `request.config.getoption("warehouse-type")` method.
