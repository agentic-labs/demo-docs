## High-level description
This code implements a Chainlit application that sets up starter messages and handles incoming messages. It demonstrates the use of Chainlit's decorators to define starter options and message handling functionality.

## Symbols

### `starters`
#### Description
This asynchronous function is decorated with `@cl.set_starters` and defines a list of starter options for the Chainlit application.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| return value | List[cl.Starter] | A list of three Starter objects with different labels and messages |

#### Internal Logic
The function creates and returns a list of three `cl.Starter` objects, each with a unique label and corresponding message.

___

### `on_message`
#### Description
This asynchronous function is decorated with `@cl.on_message` and handles incoming messages in the Chainlit application.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| msg | cl.Message | The incoming message object |

#### Internal Logic
The function simply echoes back the content of the received message by creating a new `cl.Message` object with the same content and sending it.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| chainlit | The main library used for building the conversational AI application |

## Code Structure
The code consists of two main functions: `starters` and `on_message`. The `starters` function sets up initial conversation options, while `on_message` handles the actual message processing. Both functions are decorated with Chainlit-specific decorators to integrate them into the Chainlit framework.
