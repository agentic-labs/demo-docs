## High-level description
This file defines Storybook stories for the `AppWrapper` component. It sets up two stories, "Primary" and "Secondary", each with different configurations for the `AppWrapper` component, allowing for visual testing and documentation of the component in different states.

## Symbols

### `meta`
#### Description
Defines the metadata for the Storybook stories of the `AppWrapper` component.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| title | string | The title of the story in Storybook |
| component | React.ComponentType | The component being showcased (AppWrapper) |
| parameters | object | Additional Storybook parameters |
| tags | string[] | Tags for the story, including 'autodocs' for automatic documentation |
| argTypes | object | Definitions for the component's props (empty in this case) |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| meta | Meta&lt;typeof AppWrapper&gt; | Storybook metadata object |

### `Primary`
#### Description
Defines the primary story for the `AppWrapper` component.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| args | object | Props passed to the AppWrapper component |
| args.widgetConfig | object | Configuration for the widget |
| args.widgetConfig.chainlitServer | string | URL of the Chainlit server |

### `Secondary`
#### Description
Defines the secondary story for the `AppWrapper` component with more detailed configuration.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| args | object | Props passed to the AppWrapper component |
| args.widgetConfig | object | Detailed configuration for the widget |
| args.widgetConfig.chainlitServer | string | URL of the Chainlit server |
| args.widgetConfig.theme | string | Theme of the widget ('dark') |
| args.widgetConfig.fontFamily | string | Font family for the widget |
| args.widgetConfig.button | object | Configuration for the widget button |

## Dependencies

## References
The stories reference the `AppWrapper` component, which is imported from 'appWrapper'. Based on the related code snippet provided, `AppWrapper` is a React component that sets up the necessary context providers and renders the main `App` component with the given `widgetConfig`.

## Notes
1. The stories use the `satisfies` operator with the `Meta` type to ensure type safety while allowing inference of the `Story` type.
2. Both stories provide different configurations for the `widgetConfig` prop of `AppWrapper`, showcasing how the component can be used with various settings.
3. The `Secondary` story includes more detailed configuration, including theme, font family, and button styling, demonstrating the flexibility of the `AppWrapper` component.
4. The stories are set up to use Storybook's autodocs feature, which will automatically generate documentation for the component.
5. The layout is set to 'centered' in the story parameters, which will center the component in Storybook's canvas.

This Storybook setup allows developers to visually test and document the `AppWrapper` component with different configurations, making it easier to develop and maintain the component's various states and appearances.
