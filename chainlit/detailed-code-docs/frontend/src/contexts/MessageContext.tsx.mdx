## High-level description
This code defines a React context for managing message-related state and functionality in a chat or messaging application. It provides a centralized way to share message-related data and functions across components.

## Code Structure
The `MessageContext` is created using React's `createContext` function. A `defaultMessageContext` object is defined with initial values for the context. The context and default values are then exported for use in other parts of the application.

## Symbols

### `defaultMessageContext`
#### Description
An object containing default values for the message context.

#### Internal Logic
Defines initial values for various properties used in the message context, such as avatars, loading state, and callback functions.

### `MessageContext`
#### Description
A React context created using `createContext` function, initialized with the `defaultMessageContext`.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| defaultValue | IMessageContext | The default context value |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| MessageContext | React.Context&lt;IMessageContext&gt; | The created context object |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| react | For using the `createContext` function |
| types/messageContext | For importing the `IMessageContext` type |

## References
This context is used in various components throughout the application, such as:
- `MessageContainer`
- `ActionButton`
- `ElementRef`
- `FeedbackButtons`
- `UserMessage`
- `Message`
- `AskUploadButton`

These components consume the `MessageContext` to access shared state and functionality related to messages.

The `IMessageContext` interface, defined in `frontend/src/types/messageContext.ts`, specifies the shape of the context object, including properties for file uploads, user interactions, UI state, and callback functions for various message-related actions.
