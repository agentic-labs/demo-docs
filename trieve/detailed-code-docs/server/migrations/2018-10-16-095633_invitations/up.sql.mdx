## High-level description
This SQL script creates a new table named `invitations` in the database. The table is designed to store information about invitations, including the recipient's email, expiration date, and timestamps for creation and updates.

#### Internal Logic
The table schema is defined with the following columns:

- **`id`**: A universally unique identifier (UUID) serving as the primary key for each invitation. It's marked as `NOT NULL` and `UNIQUE` to ensure data integrity.
- **`email`**: Stores the email address of the invitee. It's a `VARCHAR(100)` field, limited to 100 characters, and is marked as `NOT NULL`.
- **`expires_at`**: A `TIMESTAMP` field storing the date and time when the invitation expires. It's marked as `NOT NULL`.
- **`created_at`**: A `TIMESTAMP` field automatically populated with the date and time when the invitation record was created. It's marked as `NOT NULL`.
- **`updated_at`**: A `TIMESTAMP` field automatically updated with the date and time of the latest modification to the invitation record. It's marked as `NOT NULL`.
