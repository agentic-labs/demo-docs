## High-level description
This SQL script performs a database migration that reverts the changes introduced by the corresponding `up.sql` script. Specifically, it drops the `time_stamp` column from the `card_metadata` and `files` tables.

## Symbols
### `ALTER TABLE card_metadata DROP COLUMN IF EXISTS time_stamp;`
#### Description
This statement drops the `time_stamp` column from the `card_metadata` table if it exists. The `IF EXISTS` clause ensures that the statement executes successfully even if the column doesn't exist.

#### Internal Logic
- **ALTER TABLE `card_metadata`**: Specifies the table to be modified.
- **DROP COLUMN**: Indicates that a column should be removed.
- **IF EXISTS `time_stamp`**: Conditionally drops the column only if it exists.

### `ALTER TABLE files DROP COLUMN IF EXISTS time_stamp;`
#### Description
This statement is identical to the previous one, but it operates on the `files` table instead of `card_metadata`. It drops the `time_stamp` column from the `files` table if it exists.

#### Internal Logic
- **ALTER TABLE `files`**: Specifies the table to be modified.
- **DROP COLUMN**: Indicates that a column should be removed.
- **IF EXISTS `time_stamp`**: Conditionally drops the column only if it exists.
