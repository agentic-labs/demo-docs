## High-level description
This SQL script creates a new table named `file_upload_completed_notifications` in the database. This table is designed to store information about notifications sent to users upon the successful completion of file uploads. 

## Symbols

### `file_upload_completed_notifications` table
#### Description
This table stores data related to notifications sent to users when a file upload is completed. Each row represents a single notification.

#### Columns
| Name | Type | Description |
|:-----|:-----|:------------|
| `id` | UUID | Unique identifier for the notification (primary key). |
| `user_uuid` | UUID |  The UUID of the user who should receive the notification. |
| `collection_uuid` | UUID | The UUID of the collection the uploaded file belongs to. |
| `user_read` | boolean | Flag indicating whether the user has marked the notification as read (`true`) or not (`false`). Defaults to `false`. |
| `created_at` | TIMESTAMP | Timestamp indicating when the notification was created. Defaults to the current timestamp. |
| `updated_at` | TIMESTAMP | Timestamp indicating when the notification was last updated. Defaults to the current timestamp. |

## Side Effects
- Creates a new table named `file_upload_completed_notifications` in the database. 
- Impacts database schema by adding a new table.
