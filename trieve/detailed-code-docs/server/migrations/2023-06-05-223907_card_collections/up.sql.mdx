## High-level description
This SQL script creates a new table named `card_collection` in the database. This table is designed to store information about collections of cards, including their author, name, description, privacy setting, and timestamps for creation and last update. 

## Symbols

### `card_collection` table
#### Description
Defines the structure of the `card_collection` table, which stores information about collections of cards.

#### Columns
| Name | Type | Description |
|:-----|:-----|:------------|
| `id` | UUID | Unique identifier for the card collection (primary key) |
| `author_id` | UUID | Foreign key referencing the `users` table, indicating the creator of the collection |
| `name` | TEXT | Name of the card collection (not nullable) |
| `is_public` | BOOLEAN | Flag indicating if the collection is publicly viewable (defaults to false) |
| `description` | TEXT | Description of the card collection (not nullable) |
| `created_at` | TIMESTAMP | Timestamp indicating when the collection was created (defaults to current timestamp) |
| `updated_at` | TIMESTAMP | Timestamp indicating when the collection was last updated (defaults to current timestamp) |

## References
- **`users` table:** The `author_id` column references the `id` column of the `users` table, establishing a foreign key relationship. This ensures that every card collection is associated with a valid user.
