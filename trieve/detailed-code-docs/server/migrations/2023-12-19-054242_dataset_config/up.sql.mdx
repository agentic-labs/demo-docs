## High-level description
This SQL script adds a new column named `configuration` to the `datasets` table. The column is of type JSONB, cannot be null, and defaults to an empty JSON object.

## Symbols

### `ALTER TABLE datasets ADD COLUMN configuration JSONB NOT NULL DEFAULT '{}'::JSONB;`
#### Description
This SQL statement modifies the existing `datasets` table by adding a new column named `configuration`.

#### Internal Logic
* **`ALTER TABLE datasets`**: This clause specifies that the table named `datasets` is being altered.
* **`ADD COLUMN configuration`**: This clause adds a new column named `configuration` to the table.
* **`JSONB`**: This specifies the data type of the new column as JSONB, which is a specialized data type in PostgreSQL for storing JSON (JavaScript Object Notation) data.
* **`NOT NULL`**: This constraint ensures that the `configuration` column cannot contain null values.
* **`DEFAULT '{}'::JSONB`**: This clause sets the default value of the `configuration` column to an empty JSON object (`{}`). The `::JSONB` explicitly casts the empty object literal to the JSONB data type.

#### Side Effects
This script modifies the database schema by adding a new column to the `datasets` table.
