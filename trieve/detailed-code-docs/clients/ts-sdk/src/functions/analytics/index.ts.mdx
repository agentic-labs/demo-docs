## High-level description
This code defines a module for analytics functions in the Trieve SDK. It provides methods to interact with various analytics endpoints of the Trieve API, including CTR (Click-Through Rate), RAG (Retrieval-Augmented Generation), recommendation, search, and cluster analytics.

## Code Structure
The file exports several async functions that are methods of the TrieveSDK class. These functions make HTTP requests to different analytics endpoints using the `fetch` method of the TrieveSDK instance.

## Symbols

### `getCTRAnalytics`
#### Description
Retrieves CTR analytics for a dataset.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| data | CTRAnalytics | Analytics request parameters |
| signal | AbortSignal (optional) | Used to abort the request |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| response | Promise&lt;unknown&gt; | The API response containing CTR analytics data |

#### Internal Logic
Makes a POST request to `/api/analytics/ctr` with the provided data and dataset ID.

### `sendCTRAnalytics`
#### Description

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| response | Promise&lt;unknown&gt; | The API response after sending CTR data |
