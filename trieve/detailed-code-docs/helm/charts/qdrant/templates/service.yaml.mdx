## High-level description
This YAML file defines a Kubernetes Service resource for Qdrant, a vector database. It exposes the Qdrant application to other services within the cluster or externally, depending on the configuration. The Service is customizable through Helm values, allowing for flexible deployment options.

## Symbols

### Service Resource
#### Description
This Kubernetes Service resource is used to expose the Qdrant application, making it accessible to other components within the cluster or externally. It uses Helm templating to allow for customization of various aspects of the Service.

#### Internal Logic
1. Defines the Service name using the `qdrant.fullname` template.
2. Applies labels using the `qdrant.labels` template.
3. Adds additional labels and annotations if specified in the Helm values.
4. Sets the Service type, defaulting to "ClusterIP" if not specified.
5. Configures a load balancer IP if provided.
6. Defines ports based on the values provided in the Helm chart.
7. Sets the selector to match the Qdrant pods using the `qdrant.selectorLabels` template.

### Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| service.type | string | "ClusterIP" | Specifies the type of Kubernetes Service |
| service.loadBalancerIP | string | - | Sets a specific IP for LoadBalancer type services |
| service.additionalLabels | map | - | Additional labels to apply to the Service |
| service.annotations | map | - | Annotations to apply to the Service |
| service.ports | list | - | List of ports to expose on the Service |
