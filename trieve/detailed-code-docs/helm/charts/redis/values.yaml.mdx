## High-level description
The `values.yaml` file defines the configurable parameters for the Bitnami Redis Helm chart. It provides a comprehensive set of options for customizing the deployment of a Redis master-replica cluster on a Kubernetes cluster, including configuration for Redis master and replica nodes, Sentinel, persistence, networking, security, and resource allocation.

## Code Structure
The file is structured as a YAML document with nested key-value pairs. The top-level keys represent different sections of the Redis deployment, such as `global`, `common`, `image`, `master`, `replica`, `sentinel`, `metrics`, etc. Each section contains further nested keys that define specific configuration options for that aspect of the deployment.

## Symbols
### `global`
#### Description
Defines global parameters that apply to the entire Redis deployment.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| `global.imageRegistry` | string | Global Docker image registry. |
| `global.imagePullSecrets` | list | Global Docker registry secret names as an array. |
| `global.storageClass` | string | Global StorageClass for Persistent Volume(s). |
| `global.redis.password` | string | Global Redis password (overrides `auth.password`). |

#### Outputs
None

| `commonAnnotations` | map | Annotations to add to all deployed objects. |
| `secretAnnotations` | map | Annotations to add to secret. |

None
