## High-level description
This file defines a Kubernetes Deployment for Apache Tika, a content analysis toolkit. It sets up a single container running the Apache Tika server, exposing port 9998 for incoming requests.

## Code Structure
The YAML file defines a Kubernetes Deployment resource with metadata, selector, and template specifications. The template section describes the pod that will be created, including container details.

## Symbols

### Deployment
#### Description
Defines a Kubernetes Deployment resource for Apache Tika.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| .Release.Name | String | The name of the Helm release |
| .Values.containers.tika.tag | String | The tag of the Apache Tika container image |

#### Internal Logic
1. Sets up metadata with labels for the deployment.
2. Defines a selector to match pods.
3. Specifies a template for creating pods, including:
   - Labels for the pod
   - Container specification for Apache Tika

### Container Specification
#### Description
Defines the container running Apache Tika within the pod.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| .Values.containers.tika.tag | String | The tag of the Apache Tika container image |

#### Internal Logic
1. Sets the container name to "tika".
2. Constructs the image name using "apache/tika" and the provided tag.
3. Exposes port 9998 for the container.
