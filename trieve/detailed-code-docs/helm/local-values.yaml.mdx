## High-level description
This YAML file contains configuration settings for a local deployment of the Trieve application. It defines various environment variables, container versions, database configurations, and service-specific settings for components like Redis, Qdrant, analytics, authentication, and embedding models.

## Code Structure
The YAML file is structured into several main sections: environment settings, container versions, database configurations, and application-specific configurations for various services and components of the Trieve application.

## Symbols

### environment
#### Description
Specifies the deployment environment.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| environment | string | The deployment environment, set to "local" |

### domain
#### Description
Specifies the domain name for local deployment.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| domain | string | The domain name, set to "localtrieve.com" |

### useGpu
#### Description
Indicates whether to use GPU resources.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| useGpu | boolean | Flag to enable/disable GPU usage, set to false |

### containers
#### Description
Defines the container versions for various services.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| containers | object | Contains key-value pairs of service names and their corresponding container tags |

### postgres
#### Description
Configures PostgreSQL database settings.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| useSubchart | boolean | Flag to use a subchart for PostgreSQL |
| dbURI | string | PostgreSQL connection URI (used if useSubchart is false) |

### config
#### Description
Contains configuration settings for various components of the application.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| vite | object | Frontend configuration settings |
| redis | object | Redis configuration settings |
| qdrant | object | Qdrant vector database configuration |
| analytics | object | Analytics configuration |

| oidc | object | OpenID Connect configuration |
| smtp | object | Email configuration |
| llm | object | Language Model configuration |
| openai | object | OpenAI API configuration |
| s3 | object | S3 storage configuration |
| stripe | object | Stripe payment configuration |

### embeddings
#### Description
Defines the embedding models used in the application.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| embeddings | array | List of embedding model configurations |
