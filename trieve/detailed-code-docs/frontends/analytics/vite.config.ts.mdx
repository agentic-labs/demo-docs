## High-level description
This code defines the Vite configuration for an analytics frontend project built with SolidJS. It sets up two plugins: `vite-plugin-solid` for SolidJS support and `vite-plugin-runtime-env` for injecting environment variables at runtime.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| vite | Provides the build tooling and development server |
| vite-plugin-solid | Enables SolidJS support in Vite |
| vite-plugin-runtime-env | Allows injecting environment variables at runtime |
| solid | The SolidJS library itself |

## Symbols

### `defineConfig`
#### Description
This function, imported from `vite`, defines the Vite configuration object. It takes an object with various configuration options as an argument.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| config | object | An object containing Vite configuration options |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| config | object | The resolved Vite configuration object |

#### Internal Logic
The function merges the provided configuration options with default Vite settings and returns the final configuration object used by Vite.

### `plugins`
#### Description
This property within the `defineConfig` object specifies an array of Vite plugins to be used in the project.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| plugins | array | An array of Vite plugin objects |

#### Internal Logic
Vite iterates through the provided plugins and applies their transformations to the build process.

####  Plugins used:
*  `solid()`: This plugin, imported from `vite-plugin-solid`, enables SolidJS support in Vite. It handles SolidJS-specific transformations and optimizations during the build process.
* `runtimeEnv()`: This plugin, imported from `vite-plugin-runtime-env`, allows injecting environment variables at runtime. It makes environment variables accessible within the SolidJS application.
