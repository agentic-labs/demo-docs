## High-level description
The `useBetterNav` hook is a custom navigation utility in a SolidJS application. It enhances the default navigation functionality by preserving query parameters when navigating to a new pathname.

## Symbols

### `useBetterNav`
#### Description
This is a custom hook that combines the `useSearchParams` and `useNavigate` hooks from SolidJS router to create a navigation function that preserves query parameters.

#### Inputs
This hook doesn't take any inputs.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| betterNav | function | A function that takes a pathname and navigates to it while preserving existing query parameters |

#### Internal Logic
1. It uses `useSearchParams` to get the current query parameters.
2. It uses `useNavigate` to get the navigation function.
3. It returns a new function that:
   - Takes a pathname as an argument
   - Creates a new `URLSearchParams` object from the current query parameters
   - Constructs a new URL by combining the given pathname with the existing search parameters
   - Uses the `navigate` function to go to the constructed URL

## References
- `useNavigate` from "@solidjs/router"
- `useSearchParams` from "@solidjs/router"

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| @solidjs/router | Provides routing functionality, including `useNavigate` and `useSearchParams` hooks |

## Usage Examples
The `useBetterNav` hook is used in several components across the application:

1. In `Sidebar.tsx`, it's used to navigate between different routes while preserving query parameters:

```typescript
const navigate = useBetterNav();
// ...
onClick={() =&gt; {
  navigate(link.href);
}}
```

2. In `SearchTablePage.tsx`, it's used to navigate to a specific query details page:

```typescript
const navigate = useBetterNav();
// ...
onClick={() =&gt; {
  navigate(`/query/${props.event.id}`);
}}
```

3. In `DataExplorerTabs.tsx`, it's used to handle navigation between tabs:

```typescript
const betterNav = useBetterNav();
// ...
const handleLinkClick = (e: MouseEvent, path: string) =&gt; {
  e.preventDefault();
  betterNav(path);
};
```

These examples demonstrate how `useBetterNav` is used to maintain consistent navigation behavior across the application, ensuring that query parameters are preserved when moving between different routes or pages.
