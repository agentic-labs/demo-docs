## High-level description
The code defines a user interface for a CTR (Click-Through Rate) analytics page. Currently, it only includes a filter bar and a placeholder message indicating the page is unused.

## Code Structure
The `CTRAnalyticsPage` component renders a simple div with a title and the `SearchCTRAnalytics` component. The `SearchCTRAnalytics` component manages the filter state using `createStore` and renders a `FilterBar` component.

## References
- `AnalyticsParams` type from "shared/types"
- `FilterBar` component from "../components/FilterBar"

## Symbols

### `CTRAnalyticsPage`
#### Description
This functional component renders the basic structure of the CTR Analytics page. It includes a heading "Search" and the `SearchCTRAnalytics` component.

#### Inputs
*None*

#### Outputs
- JSX Element: A div element containing the page structure.

### `SearchCTRAnalytics`
#### Description
This functional component manages the filter state for the CTR analytics and renders the `FilterBar` component. It initializes the filter state with default values for date range, search method, and search type.

#### Inputs
*None*

#### Outputs
- JSX Element: A div element containing the `FilterBar` component and a placeholder message.

#### Internal Logic
1. **State Initialization:**
   - Uses `createStore` to create a state variable `searchCtrFilter` with an initial value of type `AnalyticsParams`.
   - The initial filter values include a date range starting from 7 days ago, a search method of "hybrid", and a search type of "search".
2. **Rendering:**
   - Renders a `FilterBar` component, passing the `searchCtrFilter` state and its setter function as props.
   - Displays a placeholder message "Unused page!!!".
