## High-level description
The code defines a user interface for a CTR (Click-Through Rate) analytics page. Currently, it only includes a filter bar (`FilterBar`) and a placeholder message indicating the page is unused.

## Code Structure
The `CTRAnalyticsPage` component renders a simple div with a title and the `SearchCTRAnalytics` component. The `SearchCTRAnalytics` component initializes the filter state using `createStore` and renders the `FilterBar` component, passing the filter state and update function as props.

## References
- `AnalyticsParams` type from "shared/types"
- `FilterBar` component from "../components/FilterBar"

## Symbols

### `CTRAnalyticsPage`
#### Description
This functional component renders the basic structure of the CTR Analytics page. It includes a heading "Search" and the `SearchCTRAnalytics` component.

#### Inputs
None

#### Outputs
JSX Element representing the CTR Analytics page.

### `SearchCTRAnalytics`
#### Description
This functional component sets up the filter state for the CTR Analytics page and renders the `FilterBar` component.

#### Inputs
None

#### Outputs
JSX Element containing the `FilterBar` component and a placeholder message.

#### Internal Logic
1. **State Initialization:**
   - Uses `createStore` to create a state variable `searchCtrFilter` of type `AnalyticsParams`.
   - Initializes the state with default filter values:
     - `date_range`: Starts from 7 days before the current date.
     - `search_method`: Set to "hybrid".
     - `search_type`: Set to "search".
     - `granularity`: Set to "minute" (not currently used).
2. **Rendering:**
   - Renders a `div` element.
   - Renders the `FilterBar` component, passing the `searchCtrFilter` state and `setSearchCtrFilter` update function as props.
   - Displays a placeholder message "Unused page!!!".
