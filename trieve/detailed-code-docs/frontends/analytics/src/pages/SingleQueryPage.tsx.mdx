## High-level description
The `SingleQueryPage` component displays detailed information about a single search query. It retrieves the query ID from the URL parameters and renders the `SingleQuery` component, passing the query ID as a prop.

## References
- `SingleQuery`: This component is responsible for fetching and displaying the details of a single search query. It is defined in the `frontends/analytics/src/components/SingleQueryInfo.tsx` file.

## Symbols

### `SingleQueryPage`
#### Description
This functional component renders a page dedicated to displaying information about a single search query.

#### Inputs
This component takes no direct inputs.

#### Outputs
This component renders a `div` element containing the `SingleQuery` component.

#### Internal Logic
1. **Retrieves Query ID:** It uses the `useParams` hook from `@solidjs/router` to extract the `id` parameter from the URL. This `id` represents the unique identifier of the search query.
2. **Renders `SingleQuery`:** It renders the `SingleQuery` component, passing the extracted `queryId` as a prop. This component is responsible for fetching and displaying the query details.

## Dependencies
- `@solidjs/router`: Used for routing and accessing URL parameters.

## Side Effects
This component has no direct side effects. However, the `SingleQuery` component it renders might have side effects related to data fetching and UI updates.
