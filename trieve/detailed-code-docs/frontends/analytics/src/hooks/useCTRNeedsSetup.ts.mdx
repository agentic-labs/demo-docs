## High-level description
The `useCTRNeedsSetup` function is a SolidJS hook that determines if click-through rate (CTR) analytics have been set up for a given dataset. It does this by checking if key CTR metrics are available from the backend API.

## Code Structure
The `useCTRNeedsSetup` hook utilizes the `createQuery` function from `@tanstack/solid-query` to fetch CTR summary data. It then uses a `createMemo` to derive whether CTR needs setup based on the presence of CTR metrics in the fetched data.

## References
- `DatasetContext`: Provides access to the selected dataset's information.
- `getSearchCTRSummary`: API function to fetch CTR summary data.

## Symbols
### `useCTRNeedsSetup`
#### Description
This hook determines if CTR analytics need to be set up for the current dataset. It fetches CTR summary data and checks if key metrics are present. If the metrics are missing, it implies that CTR data hasn't been sent for the dataset yet.

#### Inputs
This function has no inputs.

#### Outputs
Returns a SolidJS accessor (`ctrNeedsSetup`) which is a boolean value indicating whether CTR needs setup.

#### Internal Logic
1. Retrieves the current dataset from the `DatasetContext`.
2. Defines a `searchSummaryQuery` using `createQuery` to fetch CTR summary data using `getSearchCTRSummary`.
3. Creates a memoized value `ctrNeedsSetup` that:
    - Depends on the `searchSummaryQuery` success status and data.
    - Returns `true` if the query is successful and all of the following CTR metrics are missing:
        - `avg_position_of_click`
        - `percent_searches_with_clicks`
        - `searches_with_clicks`
    - Otherwise, returns `false`.
4. Returns the `ctrNeedsSetup` accessor.
