## High-level description
The `OrganizationSelectBox` component is a dropdown menu that allows users to select and switch between different organizations. It displays the currently selected organization and provides a list of available organizations for the user to choose from.

## Code Structure
The component uses the `UserContext` to access and update the current organization. It utilizes various components from the `terracotta` library to create the dropdown menu structure.

## Symbols

### OrganizationSelectBox
#### Description
This is the main functional component that renders the organization selection dropdown.

#### Internal Logic
1. Retrieves the user context using `useContext(UserContext)`.
2. Renders a `Popover` component that contains the current organization name and a dropdown arrow.
3. When the popover is open, it displays a `PopoverPanel` with a list of organizations.
4. Each organization is rendered as a `MenuItem` with a click handler to update the current organization.
5. The currently selected organization is visually highlighted and marked with a checkmark icon.

#### Performance Considerations
The component uses the `Show` component to conditionally render the `PopoverPanel`, which helps optimize performance by only rendering the panel when it's open.

## Side Effects
- When an organization is selected, it updates the current organization in the `UserContext`.

## Error Handling
There is no explicit error handling in this component. Error handling for API calls and state updates is likely managed in the `UserContext` or parent components.

## API/Interface Reference
This component doesn't expose an API but relies on the `UserContext` for data and state management. The context is expected to provide:

- `currentOrganization`: The currently selected organization
- `organizations`: An array of available organizations
- `setCurrentOrganization`: A function to update the current organization

The component assumes that each organization object has `id` and `name` properties.
