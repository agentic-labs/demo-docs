## High-level description
The Sidebar component is a crucial part of the chat interface, providing navigation, topic management, and user settings. It displays a list of topics, allows creation and editing of topics, and provides access to various application settings and external links.

## Code Structure
The Sidebar component is composed of several sub-components and uses various SolidJS primitives for state management and reactivity. It interacts with the UserContext for user-related data and operations, and uses the FullScreenModal for displaying settings.

## Symbols

### Sidebar
#### Description
The main component that renders the sidebar interface, including topic list, creation button, and various settings and links.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| props | SidebarProps | Contains various props for managing topics and sidebar state |

#### Internal Logic
1. Initializes state variables for editing topics, settings modal, and GitHub star count.
2. Implements functions for editing and deleting topics.
3. Renders the sidebar UI, including topic list, creation button, dataset and organization selectors, and various links.
4. Handles the settings modal display.

### submitEditText
#### Description
Asynchronous function to submit edited topic text to the server.

#### Internal Logic
1. Retrieves the current topic and dataset.
2. Sends a PUT request to update the topic name.
3. Refetches topics after successful update.

### deleteSelected
#### Description
Asynchronous function to delete the currently selected topic.

#### Internal Logic
1. Sends a DELETE request to remove the current topic.
2. Clears the current topic and refetches topics on success.

## Side Effects
- Fetches GitHub star count for the Trieve repository on component mount.
- Updates the checkmark icon visibility based on the edited topic name.

The component uses the `createToast` function to display error messages when topic editing or deletion fails.
