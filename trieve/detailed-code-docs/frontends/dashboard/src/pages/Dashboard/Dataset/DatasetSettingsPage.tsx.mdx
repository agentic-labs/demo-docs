## High-level description
This code defines the DatasetSettingsPage component, which allows users to configure various settings for a dataset. It includes forms for LLM settings, embedding settings, and additional options, as well as a danger zone for dataset management. The component uses the SolidJS framework and interacts with a DatasetContext to manage and update dataset configurations.

## Code Structure
The code is organized into several main components:
1. ServerSettingsForm: Handles LLM, embedding, and additional settings
2. DangerZoneForm: Manages dataset deletion and clearing
3. SaveButton: Triggers the save action for changes
4. DatasetSettingsSidebar: Provides navigation within the settings page
5. DatasetSettingsPage: The main component that combines all the above elements

These components work together to create a comprehensive settings interface for dataset management.

## Symbols

### ServerSettingsForm
#### Description
This component renders a form for configuring various server settings related to the dataset, including LLM settings, embedding settings, and additional options.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| serverConfig | Accessor&lt;ServerEnvsConfiguration&gt; | Current server configuration |
| setServerConfig | Function | Function to update server configuration |

#### Internal Logic
- Renders input fields for various configuration options
- Uses conditional rendering (Show component) for certain settings
- Updates the serverConfig state when input values change

### DangerZoneForm
#### Description
This component provides options for clearing or deleting the dataset.

#### Internal Logic
- Uses DatasetContext to access dataset information
- Implements confirmation steps for clearing and deleting actions
- Sends API requests to clear or delete the dataset

### SaveButton
#### Description
A button component that triggers the save action for changes made to the dataset settings.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| onSave | Function | Callback function to execute when the save button is clicked |

### DatasetSettingsSidebar
#### Description
Renders a sidebar navigation for the dataset settings page.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| onSave | Function | Callback function to execute when the save button is clicked |

#### Internal Logic
- Manages the selected navigation item state
- Renders links to different sections of the settings page
- Includes the SaveButton component

### DatasetSettingsPage
#### Description
The main component that combines all the above components to create the complete dataset settings page.

#### Internal Logic
- Uses DatasetContext to access and update dataset information
- Manages the server configuration state
- Implements the save functionality, including API calls to update the dataset configuration
- Renders the ServerSettingsForm, DangerZoneForm, and DatasetSettingsSidebar components

## Dependencies
- SolidJS framework and its related imports
- DatasetContext for accessing dataset information
- ServerEnvsConfiguration and other types from shared/types
- Various UI components and utilities

## Configuration
The component uses environment variables (import.meta.env) for API host configuration.

## Error Handling
The component uses createToast to display success and error messages to the user when performing actions like saving settings or deleting the dataset.

## Performance Considerations
The component uses createMemo and createEffect for optimizing reactivity and managing side effects efficiently.
