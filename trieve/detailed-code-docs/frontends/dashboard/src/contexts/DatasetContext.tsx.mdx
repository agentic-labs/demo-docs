## High-level description
This code defines a React component called `DatasetContext` that provides a context for managing dataset-related information in a dashboard application. It fetches and stores dataset information, making it available to child components through the context API.

## Code Structure
The main symbols in this code are:
1. `DatasetContext`: The context object created to store and provide dataset information.
2. `DatasetContextWrapper`: A component that wraps child components and provides the dataset context.
3. `createEffect`: A function used to perform side effects, in this case, fetching dataset information.

## Symbols

### DatasetContext
#### Description
A context object created using `createContext` to store and provide dataset information.

#### Inputs
None

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| dataset | Accessor&lt;Dataset \| null&gt; \| null | An accessor function that returns the current dataset or null |

### DatasetContextWrapper
#### Description
A component that wraps child components and provides the dataset context.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| props | DatasetStoreContextProps | Props containing child elements to be wrapped |

#### Outputs
Returns a JSX element that wraps the children with the DatasetContext.Provider.

#### Internal Logic
1. Creates signals for storing the dataset and fetching status.
2. Uses `createEffect` to fetch dataset information when the user context changes.
3. Fetches dataset information from the API using the dataset ID from URL parameters.
4. Updates the dataset state with the fetched information.
5. Handles errors by displaying a toast notification.

## Side Effects
- Fetches dataset information from an API endpoint.
- Displays toast notifications for errors.

The code implements error handling in the API fetch request:
- Checks for invalid dataset IDs.
- Catches fetch errors and displays an error toast notification.
