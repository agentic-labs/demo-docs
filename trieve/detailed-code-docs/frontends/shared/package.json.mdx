## High-level description
This package.json file defines the configuration for a shared frontend module in a larger project. It specifies dependencies, scripts, and export configurations for various submodules within the shared package.

## Code Structure
The file is structured as a standard package.json, containing metadata, scripts, exports, and dependencies for the shared module.

## Symbols

### name
#### Description
Specifies the name of the package.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| name | string | The name of the package |

### version
#### Description
Specifies the version of the package.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| version | string | The version number of the package |

### scripts
#### Description
Defines scripts that can be run using npm or yarn.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| genclient | string | Command to generate client code |

### exports
#### Description
Configures how the package's modules are exported for use in other projects.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| . | object | Main module export configuration |
| ./types | object | Types module export configuration |
| ./ui | object | UI module export configuration |
| ./utils | object | Utils module export configuration |

### dependencies
#### Description
Lists the production dependencies required by the package.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| dependency name | string | Version or source of the dependency |

### devDependencies
#### Description
Lists the development dependencies required by the package.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| dependency name | string | Version or source of the dependency |
