## High-level description
This code defines a profiling utility for MindsDB. It provides functions and a decorator to measure and record the execution time of different parts of the code, creating a hierarchical structure of profiling data.

## Code Structure
The `__init__.py` file imports and exposes key functions and classes from the `profiler.py` module, making them easily accessible when importing from the `mindsdb.utilities.profiler` package.

## Symbols

### `start`
#### Description
Starts a new profiling node with the given tag.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| tag | str | Name of the new profiling node |

### `stop`
#### Description
Finalizes the current profiling node and moves the pointer up in the profiling tree.

### `Context`
#### Description
A context manager class for profiling code blocks.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| tag | str | Name of the profiling node |

### `profile`
#### Description
A decorator function for profiling entire functions.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| tag | str | Optional tag for the profiling node. If not provided, it uses the function name and module |

### `enable`
#### Description
Enables profiling.

### `disable`
#### Description
Disables profiling.

### `set_meta`
#### Description
Adds additional metadata to the profiling data.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| **kwargs | dict | Metadata to add to the profiling data |

## Dependencies
The code relies on the `mindsdb.utilities.profiler.profiler` module, which contains the actual implementation of these functions and classes.

## Notes
This `__init__.py` file serves as an interface to the profiling functionality, exposing only the necessary functions and classes from the `profiler.py` module. It simplifies the import process for users of the profiling utility, allowing them to import these symbols directly from `mindsdb.utilities.profiler`.
