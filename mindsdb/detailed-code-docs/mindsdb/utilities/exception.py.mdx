## High-level description

This code defines the `EntityExistsError` and `EntityNotExistsError` exception classes, which are used to handle errors related to the existence or non-existence of entities in the MindsDB system. These custom exceptions inherit from a base `BaseEntityException` class and provide specific error messages for entity-related issues.

## Code Structure

The code defines three exception classes:
1. `BaseEntityException`: A base exception class for entity-related errors.
2. `EntityExistsError`: An exception raised when an entity exists but should not.
3. `EntityNotExistsError`: An exception raised when an entity does not exist but should.

## Usage

These exception classes are likely used throughout the MindsDB system to handle errors related to entity existence. They provide a standardized way to raise and catch exceptions when dealing with entities that may or may not exist in the system.

For example:
```python
try:
    # Some operation that might raise an EntityExistsError
    raise EntityExistsError("Custom message", "example_entity")
except EntityExistsError as e:
    print(str(e))  # Output: Custom message: example_entity

try:
    # Some operation that might raise an EntityNotExistsError
    raise EntityNotExistsError(entity_name="missing_entity")
except EntityNotExistsError as e:
    print(str(e))  # Output: Entity does not exists error: missing_entity
```

These exceptions help in providing clear and specific error messages when dealing with entity-related operations in the MindsDB system.
