## High-level description
This code defines the connection arguments required for connecting to a PostgreSQL database with pgvector support. It specifies the structure and properties of each connection parameter, including their types, descriptions, and whether they are required or optional.

## Code Structure
The code consists of two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary defines the structure and properties of each connection parameter, while `connection_args_example` provides an example of how these parameters should be filled.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the structure and properties of connection arguments required for connecting to a PostgreSQL database with pgvector support.

#### Internal Logic
The dictionary uses keys as parameter names and values as dictionaries describing the properties of each parameter. The properties include:
- `type`: The data type of the parameter (from `ARG_TYPE`)
- `description`: A brief explanation of the parameter
- `required`: A boolean indicating if the parameter is mandatory
- `label`: A user-friendly label for the parameter
- `secret`: (Only for password) A boolean indicating if the parameter should be treated as sensitive information

The parameters defined are:
1. `user`: PostgreSQL username
2. `password`: PostgreSQL password
3. `database`: Name of the database to connect to
4. `host`: Hostname or IP address of the PostgreSQL server
5. `port`: TCP/IP port of the PostgreSQL server
6. `schema`: (Optional) Schema to search objects in
7. `sslmode`: (Optional) SSL mode for the connection

### `connection_args_example`
#### Description
An OrderedDict providing an example of how the connection arguments should be filled.

#### Internal Logic
This dictionary contains sample values for the main required connection parameters:
- `host`: "127.0.0.1"
- `port`: 5432
- `user`: "root"
- `password`: "password"
- `database`: "database"

## Notes
- The code emphasizes using '127.0.0.1' instead of 'localhost' for connecting to a local PostgreSQL server.
- The `password` parameter is marked as `secret: True`, indicating it should be handled securely.
- Optional parameters (`schema` and `sslmode`) are included but marked as not required.

This code serves as a configuration file for defining the structure of connection arguments needed to connect to a PostgreSQL database with pgvector support in the MindsDB system.
