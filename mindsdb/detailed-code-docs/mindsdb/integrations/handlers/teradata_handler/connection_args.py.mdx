## High-level description
This code defines the connection arguments for a Teradata database handler in the MindsDB integration library. It specifies the required and optional parameters for establishing a connection to a Teradata database, including authentication details, connection settings, and SSL/TLS configurations.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments for the Teradata database handler. Each key-value pair represents a connection parameter with its type and description.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Various connection parameters | OrderedDict | Key-value pairs for each connection parameter |

#### Internal Logic
- Uses `ARG_TYPE` from `mindsdb.integrations.libs.const` to specify the type of each argument
- Includes parameters for host, user, password, database, ports, encryption settings, SSL/TLS configurations, and authentication methods

### `connection_args_example`
#### Description
An OrderedDict providing an example of how to use the connection arguments with sample values.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| host | str | Example IP address |
| user | str | Example username |
| password | str | Example password |
| database | str | Example database name |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to maintain the order of connection arguments |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Imported as ARG_TYPE, used to specify the type of each connection argument |

## Configuration
The `connection_args` OrderedDict defines the following configuration options:

| Option | Type | Description |
|:-------|:-----|:------------|
| host | ARG_TYPE.STR | The IP address/host name of the Teradata instance host |
| user | ARG_TYPE.STR | Specifies the user name |
| password | ARG_TYPE.PWD | Specifies the password for the user (marked as secret) |
| database | ARG_TYPE.STR | Specifies the initial database to use after logon |
| dbs_port | ARG_TYPE.STR | The port number of the Teradata instance |
| encryptdata | ARG_TYPE.STR | Controls encryption of data exchanged between the driver and the database |
| https_port | ARG_TYPE.STR | Specifies the database port number for HTTPS/TLS connections |
| sslca | ARG_TYPE.STR | Specifies the file name of a PEM file that contains CA certificates |
| sslcapath | ARG_TYPE.STR | Specifies a directory of PEM files that contain CA certificates |
| sslcipher | ARG_TYPE.STR | Specifies the TLS cipher for HTTPS/TLS connections |
| sslmode | ARG_TYPE.STR | Specifies the SSL mode for HTTPS/TLS connections |
| sslprotocol | ARG_TYPE.STR | Specifies the TLS protocol for HTTPS/TLS connections |
| tmode | ARG_TYPE.STR | Specifies the Teradata transaction mode |
| logmech | ARG_TYPE.STR | Specifies the login authentication method |
| logdata | ARG_TYPE.STR | Specifies extra data for the chosen logon authentication method |
| browser | ARG_TYPE.STR | Specifies the command to open the browser for Browser Authentication |

Note: The `password` option is marked as `secret: True`, indicating that it should be handled securely and not displayed in plain text.
