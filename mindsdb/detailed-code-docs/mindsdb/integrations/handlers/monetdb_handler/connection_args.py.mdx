## High-level description
This code defines the connection arguments required for establishing a connection to a MonetDB database in the MindsDB framework. It specifies the structure and types of connection parameters, along with their descriptions and example values.

## Code Structure
The code consists of two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary defines the structure and metadata for each connection parameter, while `connection_args_example` provides sample values for these parameters.

## References
This code references the `HANDLER_CONNECTION_ARG_TYPE` constant from `mindsdb.integrations.libs.const`, which is used to specify the data types of the connection arguments.

## Configuration
The `connection_args` OrderedDict defines the following configuration options:

| Option | Type | Description |
|:-------|:-----|:------------|
| host | STR | The host name or IP address of the MonetDB server/database |
| database | STR | The database name to use when connecting with the MonetDB server |
| user | STR | The user name used to authenticate with the MonetDB server |
| password | PWD | The password to authenticate the user with the MonetDB server |
| port | INT | Specify port to connect MonetDB through TCP/IP |
| schema_name | STR | Specify the schema name for Listing Table |

Note: The `password` option is marked as `secret: True`, indicating it should be handled securely.
