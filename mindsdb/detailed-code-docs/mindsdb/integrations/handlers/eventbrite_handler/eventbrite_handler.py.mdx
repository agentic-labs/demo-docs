## High-level description
This code defines the `EventbriteHandler` class, which is responsible for handling connections and interactions with the Eventbrite API. It sets up the necessary configuration, initializes various Eventbrite-related tables, and provides methods for connecting to and checking the connection with the Eventbrite API.

## Code Structure
The `EventbriteHandler` class inherits from `APIHandler` and initializes several Eventbrite-specific table classes. It manages the connection to the Eventbrite API and provides methods for connecting and checking the connection status.

## Symbols

### EventbriteHandler
#### Description
This class handles connections and interactions with the Eventbrite API. It initializes various Eventbrite-related tables and manages the API connection.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| name | str | Optional name for the handler |
| **kwargs | dict | Additional keyword arguments, including connection_data |

#### Internal Logic
1. Initializes the superclass and sets up connection arguments.
2. Registers various Eventbrite-related tables.
3. Provides methods for connecting to and checking the connection with the Eventbrite API.

### EventbriteHandler.connect
#### Description
Initializes the Eventbrite API Client.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| api | Client | The initialized Eventbrite API client |

#### Internal Logic
1. Checks if already connected.
2. Creates a new Client instance using the access token.
3. Sets the connection status to True.

### EventbriteHandler.check_connection
#### Description
Checks the connection to the Eventbrite API.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| response | StatusResponse | The status of the connection attempt |

#### Internal Logic
1. Attempts to connect to the API and retrieve the current user.
2. If successful, logs the connected user's name.
3. If an error occurs, logs the error message and updates the connection status.

| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| access_token | str | None | The Eventbrite API access token |

## Error Handling
The `check_connection` method catches exceptions when attempting to connect to the Eventbrite API and includes the error message in the `StatusResponse`.

## Logging
The code uses the `log` module from `mindsdb.utilities` to log information about successful connections and errors.
