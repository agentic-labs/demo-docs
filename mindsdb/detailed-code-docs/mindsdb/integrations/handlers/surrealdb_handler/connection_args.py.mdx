## High-level description
This code defines the connection arguments required for establishing a connection to a SurrealDB server in the MindsDB integration handler. It specifies the necessary parameters and their types, descriptions, and other attributes for authentication and connection.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the required connection arguments for connecting to a SurrealDB server.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| user | ARG_TYPE.STR | The username for authentication |
| password | ARG_TYPE.PWD | The password for authentication |
| database | ARG_TYPE.STR | The database name to connect to |
| host | ARG_TYPE.STR | The hostname or IP address of the SurrealDB server |
| port | ARG_TYPE.INT | The TCP/IP port of the SurrealDB server |
| namespace | ARG_TYPE.STR | The namespace name to connect to |

#### Internal Logic
Each connection argument is defined with the following attributes:
- `type`: The data type of the argument (string, password, or integer)
- `description`: A brief explanation of the argument's purpose
- `required`: Boolean indicating if the argument is mandatory
- `label`: A user-friendly label for the argument
- `secret`: Boolean indicating if the argument should be treated as sensitive information (only for password)

### `connection_args_example`
#### Description
An OrderedDict providing example values for the connection arguments.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| host | str | Example hostname |
| port | int | Example port number |
| user | str | Example username |
| password | str | Example password |
| database | str | Example database name |
| namespace | str | Example namespace |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to create ordered dictionaries for connection arguments |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Imported as ARG_TYPE, used to specify argument types |

## Notes
- The `connection_args` dictionary uses `ARG_TYPE` constants to define the type of each argument, ensuring type safety and consistency.
- The `password` argument is marked as `secret: True`, indicating that it should be handled securely and not displayed in plain text.
- The `connection_args_example` provides a set of sample values that can be used for testing or as a reference for users setting up the connection.

This code is crucial for the SurrealDB handler in MindsDB, as it defines the interface for connecting to SurrealDB servers and provides the necessary information for users to configure their connections correctly.
