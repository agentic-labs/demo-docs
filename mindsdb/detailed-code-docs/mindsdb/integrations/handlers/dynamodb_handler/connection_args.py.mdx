## High-level description
This code defines the connection arguments required for connecting to Amazon DynamoDB within the MindsDB framework. It specifies the necessary AWS credentials and region information, along with their types, descriptions, and other properties.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments required for connecting to Amazon DynamoDB. It includes AWS access key ID, secret access key, region name, and an optional session token.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| aws_access_key_id | ARG_TYPE.STR | The AWS access key identifier |
| aws_secret_access_key | ARG_TYPE.STR | The AWS secret access key |
| region_name | ARG_TYPE.STR | The AWS region to connect to |
| aws_session_token | ARG_TYPE.STR | Optional AWS session token for temporary credentials |

#### Internal Logic
- Each argument is defined with properties such as type, description, required status, and label.
- The `secret` property is set to `True` for sensitive information like `aws_secret_access_key` and `aws_session_token`.
- All arguments except `aws_session_token` are marked as required.

### `connection_args_example`

An OrderedDict providing an example of the connection arguments with sample values.

#### Inputs

## Notes
- The code uses `OrderedDict` to ensure that the connection arguments are maintained in a specific order, which can be important for certain applications or user interfaces.
- The `ARG_TYPE.STR` suggests that all arguments are expected to be strings.
- The `aws_session_token` is included as an optional argument, which is useful for scenarios involving temporary security credentials.
- The example provided in `connection_args_example` does not include the optional `aws_session_token`.

This code serves as a configuration template for establishing connections to Amazon DynamoDB within the MindsDB ecosystem, ensuring that all necessary credentials are properly defined and documented.
