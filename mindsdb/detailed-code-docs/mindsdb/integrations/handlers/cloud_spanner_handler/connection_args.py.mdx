## High-level description
This code defines the connection arguments required for connecting to a Google Cloud Spanner database in MindsDB. It specifies the structure and types of these arguments, as well as providing an example of how they should be used.

## Code Structure
The code defines two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary specifies the required and optional parameters for connecting to a Cloud Spanner database, while `connection_args_example` provides a sample of how these arguments should be populated.

## References
This code references the `HANDLER_CONNECTION_ARG_TYPE` enum from `mindsdb.integrations.libs.const`, which is used to specify the types of the connection arguments.

## Symbols

### connection_args
#### Description
An OrderedDict that defines the connection arguments required for connecting to a Google Cloud Spanner database.

#### Internal Logic
The dictionary contains key-value pairs where each key is a connection parameter, and the value is another dictionary describing the parameter's type, description, and other attributes.

| Key | Type | Description | Required | Secret |
|:----|:-----|:------------|:---------|:-------|
| instance_id | STR | The Cloud Spanner instance identifier | Yes | No |
| database_id | STR | The Cloud Spanner database identifier | Yes | No |
| project | STR | The Cloud Spanner project identifier | Yes | No |
| dialect | STR | Dialect of the database | No | No |
| credentials | STR | The Google Cloud Platform service account key in JSON format | Yes | Yes |

### connection_args_example
#### Description
An OrderedDict providing an example of how the connection arguments should be populated.

#### Internal Logic
This dictionary contains sample values for the `instance_id`, `database_id`, and `project` parameters. Note that there's a typo in the key `datbase_id` which should be `database_id`.

## Notes
1. The `credentials` argument is marked as `secret: True`, indicating that it should be handled securely and not displayed in plain text.
2. There's a typo in the `connection_args_example` dictionary: `datbase_id` should be `database_id`.
3. The `dialect` parameter is optional, as indicated by `"required": False`.

This code serves as a configuration file for the Cloud Spanner handler in MindsDB, defining the necessary parameters for establishing a connection to a Cloud Spanner database.
