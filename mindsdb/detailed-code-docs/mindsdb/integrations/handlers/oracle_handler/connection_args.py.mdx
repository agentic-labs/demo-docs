## High-level description
This code defines the connection arguments for an Oracle database handler in the MindsDB framework. It specifies the required and optional parameters needed to establish a connection to an Oracle database, along with their types, descriptions, and labels.

## Code Structure
The code consists of two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary defines the structure and metadata for each connection parameter, while `connection_args_example` provides a sample configuration.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to create ordered dictionaries for maintaining the order of connection arguments |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Imported as ARG_TYPE, used to specify the data types of connection arguments |

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments for the Oracle database handler. Each key-value pair represents a connection parameter and its metadata.

#### Internal Logic
The dictionary contains the following connection parameters:
1. dsn (Data Source Name)
2. host
3. port
4. sid (System Identifier)
5. service_name
6. user
7. password
8. disable_oob (Disable Out-of-Band)
9. auth_mode

Each parameter is defined with the following attributes:
- type: The data type of the parameter (from ARG_TYPE)
- description: A brief explanation of the parameter
- required: Boolean indicating if the parameter is mandatory
- label: A user-friendly label for the parameter
- secret: Boolean indicating if the parameter should be treated as sensitive (only for password)

### `connection_args_example`
#### Description
An OrderedDict providing an example configuration for connecting to an Oracle database.

#### Internal Logic
Contains sample values for the following parameters:
- host: "127.0.0.1"
- port: 1521
- user: "admin"
- password: "password"
- sid: "ORCL"

## Configuration
The `connection_args` dictionary defines the following configuration options:

| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| dsn | STR | None | The data source name (DSN) for the Oracle database |
| host | STR | None | The hostname, IP address, or URL of the Oracle server |
| port | INT | None | The port number for connecting to the Oracle database (Default is 1521) |
| sid | STR | None | The system identifier (SID) of the Oracle database |
| service_name | STR | None | The service name of the Oracle database |
| user | STR | None | The username for the Oracle database (Required) |
| password | PWD | None | The password for the Oracle database (Required, Secret) |
| disable_oob | BOOL | None | Boolean parameter to disable out-of-band breaks (Default is false) |
| auth_mode | STR | None | The authorization mode to use |

Note: The `user` and `password` options are marked as required, while others are optional.
