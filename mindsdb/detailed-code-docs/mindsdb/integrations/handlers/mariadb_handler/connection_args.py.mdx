## High-level description
This code defines the connection arguments required for establishing a connection to a MariaDB server in the MindsDB integration handler. It specifies the structure and properties of each connection parameter, including their types, descriptions, and whether they are required or optional.

## Code Structure
The code consists of two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary defines the structure and properties of each connection parameter, while `connection_args_example` provides an example of how these parameters should be populated.

## References
This code references the `HANDLER_CONNECTION_ARG_TYPE` enum from the `mindsdb.integrations.libs.const` module, which is used to specify the types of the connection arguments.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the structure and properties of each connection parameter for the MariaDB handler.

#### Internal Logic
Each key in the `connection_args` OrderedDict represents a connection parameter, and its value is another dictionary containing the following properties:
- `type`: The data type of the parameter (using `ARG_TYPE` constants)
- `description`: A brief explanation of the parameter
- `required`: A boolean indicating if the parameter is mandatory
- `label`: A user-friendly label for the parameter
- `secret`: (Optional) A boolean indicating if the parameter should be treated as sensitive information

The parameters defined include:
- `url`: An optional URI-like connection string
- `user`: The username for authentication
- `password`: The password for authentication
- `database`: The name of the database to connect to
- `host`: The hostname or IP address of the MariaDB server
- `port`: The TCP/IP port of the MariaDB server
- `ssl`: A boolean to enable SSL
- `ssl_ca`: Path or URL of the CA certificate file
- `ssl_cert`: Path or URL of the server public key certificate file
- `ssl_key`: Path or URL of the server private key file

### `connection_args_example`
#### Description
An OrderedDict providing an example of how the connection parameters should be populated.

#### Internal Logic
This dictionary contains sample values for the most common connection parameters:
- `host`: Set to '127.0.0.1'
- `port`: Set to 3306
- `user`: Set to 'root'
- `password`: Set to 'password'
- `database`: Set to 'database'

## Notes
- The `url` parameter, if provided, will override other connection arguments.
- The code suggests using '127.0.0.1' instead of 'localhost' when connecting to a local MariaDB server.
- SSL-related parameters are optional and only required if SSL connection is enabled.
