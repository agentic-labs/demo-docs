## High-level description
This SQL script sets up a test schema and table in BigQuery for the MindsDB BigQuery handler. It creates a schema named `MDB_DB_HANDLER_TEST`, a table named `TEST` within that schema, and inserts three rows of sample data into the table.

## Symbols

### DROP SCHEMA IF EXISTS MDB_DB_HANDLER_TEST
#### Description
This statement drops the `MDB_DB_HANDLER_TEST` schema if it already exists. This ensures a clean slate for the test setup.

### CREATE SCHEMA IF NOT EXISTS MDB_DB_HANDLER_TEST
#### Description
This statement creates the `MDB_DB_HANDLER_TEST` schema if it doesn't already exist. This schema will contain the test table.

### CREATE TABLE IF NOT EXISTS MDB_DB_HANDLER_TEST.TEST
#### Description
This statement creates a table named `TEST` in the `MDB_DB_HANDLER_TEST` schema if it doesn't already exist. The table has four columns:

| Column Name | Data Type | Description |
|:------------|:----------|:------------|
| COL_ONE     | INT64     | Integer column |
| COL_TWO     | INT64     | Integer column |
| COL_THREE   | FLOAT64   | Floating-point column |
| COL_FOUR    | STRING    | String column |

### INSERT INTO MDB_DB_HANDLER_TEST.TEST
#### Description
These statements insert three rows of sample data into the `TEST` table. Each row contains values for all four columns.

#### Internal Logic
The insert statements populate the table with the following data:

| COL_ONE | COL_TWO | COL_THREE | COL_FOUR |
|:--------|:--------|:----------|:---------|
| 1       | -1      | 0.1       | 'A'      |
| 2       | -2      | 0.2       | 'B'      |
| 3       | -3      | 0.3       | 'C'      |

This data provides a small but diverse set of values for testing purposes, including positive and negative integers, floating-point numbers, and strings.

## Side Effects
- Drops the `MDB_DB_HANDLER_TEST` schema if it exists.
- Creates a new schema named `MDB_DB_HANDLER_TEST`.
- Creates a new table named `TEST` in the `MDB_DB_HANDLER_TEST` schema.
- Inserts three rows of data into the `TEST` table.

These side effects are intentional and necessary for setting up the test environment for the BigQuery handler in MindsDB.
