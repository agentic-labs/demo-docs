## High-level description
This code defines the connection arguments for the DuckDB handler in MindsDB. It specifies the required parameters and their types for establishing a connection to a DuckDB database.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments for the DuckDB handler.

#### Internal Logic
The dictionary contains two keys:
1. `database`: Specifies the database file to read and write from.
2. `read_only`: A flag to determine if the connection should be in read-only mode.

Each key in the dictionary is associated with a nested dictionary containing the argument's type and description.

### `connection_args_example`
#### Description
An OrderedDict providing an example of how to use the connection arguments.

#### Internal Logic
It contains sample values for the `database` and `read_only` parameters.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to create ordered dictionaries for `connection_args` and `connection_args_example` |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Imported as ARG_TYPE, used to specify the types of connection arguments |

## References
This code is referenced by the DuckDB handler's `__init__.py` file, which imports `connection_args` and `connection_args_example` from this module.

## Notes
The `database` argument accepts a special value `:memory:` to create an in-memory database, which is the default behavior if not specified.
