## High-level description
This file defines custom exception classes for the RAG (Retrieval-Augmented Generation) handler in the MindsDB project. These exceptions are used to handle specific error scenarios related to model validation, prompt templates, vector stores, and LLM (Language Model) support.

## Symbols

### `InvalidOpenAIModel`
#### Description
An exception raised when an invalid OpenAI model is specified.

### `InvalidWriterModel`
#### Description
An exception raised when an invalid Writer model is specified.

### `MissingPromptTemplate`
#### Description
An exception raised when a required prompt template is missing.

### `UnsupportedVectorStore`
#### Description
An exception raised when an unsupported vector store is specified.

### `MissingUseIndex`
#### Description
An exception raised when a required index is missing.

### `UnsupportedLLM`
#### Description
An exception raised when an unsupported Language Model (LLM) is specified.

### `InvalidPromptTemplate`
#### Description
An exception raised when an invalid prompt template is provided.

## References
These exception classes are used in the related file `mindsdb/integrations/handlers/rag_handler/settings.py`. Specifically:

1. `InvalidOpenAIModel` is used in the `openai_model_must_be_supported` validator of the `OpenAIParameters` class.
2. `InvalidWriterModel` is used in the `writer_model_must_be_supported` validator of the `WriterLLMParameters` class.
3. `UnsupportedVectorStore` is used in the `vector_store_must_be_supported` validator of the `RAGBaseParameters` class.
4. `UnsupportedLLM` is used in the `llm_type_must_be_supported` validator of the `RAGHandlerParameters` class and in the `build_llm_params` function.
5. `InvalidPromptTemplate` is used in the `prompt_format_must_be_valid` validator of the `RAGBaseParameters` class.

These exceptions help in providing specific error messages when validating inputs and configurations in the RAG handler, improving error handling and debugging in the MindsDB project.
