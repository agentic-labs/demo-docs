## High-level description
This code defines the connection arguments required for the Luma API integration in MindsDB. It specifies the structure and properties of the API key needed to authenticate with the Luma service.

## Code Structure
The code defines two OrderedDict objects: `connection_args` and `connection_args_example`. These dictionaries specify the required connection arguments and provide an example of how to use them.

## References
This code references the `HANDLER_CONNECTION_ARG_TYPE` from the `mindsdb.integrations.libs.const` module.

#### Internal Logic
The `connection_args` OrderedDict contains a single key-value pair:
- Key: `api_key`
- Value: A dictionary specifying the properties of the API key:
  - `type`: Set to `ARG_TYPE.STR`, indicating it's a string
  - `description`: A brief description of the argument
  - `required`: Set to `True`, indicating it's a mandatory field
  - `label`: The label for the API key field
  - `secret`: Set to `True`, indicating it should be treated as sensitive information

## Notes
This code is part of the Luma handler integration in MindsDB. It defines the structure for the connection arguments required to authenticate with the Luma API. The use of OrderedDict ensures that the order of the arguments is preserved, which can be important for certain integrations or user interfaces.

The `connection_args` dictionary specifies that the Luma integration requires an API key, which is a string, mandatory, and should be treated as sensitive information. This structure allows MindsDB to properly handle the API key when establishing a connection to the Luma service.

The `connection_args_example` provides a simple example of how the connection arguments should be structured, which can be useful for documentation or testing purposes.
