## High-level description
This code defines the connection arguments required for establishing a connection to a Databricks cluster or SQL warehouse. It specifies the structure and properties of these arguments, including their types, descriptions, and whether they are required or optional.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments for a Databricks connection. Each key-value pair represents a connection parameter with its properties.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| server_hostname | dict | Properties for the server hostname parameter |
| http_path | dict | Properties for the HTTP path parameter |
| access_token | dict | Properties for the access token parameter |
| session_configuration | dict | Properties for additional session configuration |
| http_headers | dict | Properties for additional HTTP headers |
| catalog | dict | Properties for the catalog parameter |
| schema | dict | Properties for the schema parameter |

#### Internal Logic
- Each parameter is defined with properties such as type, description, required status, and label.
- The `access_token` parameter is marked as secret.
- Optional parameters include `session_configuration`, `http_headers`, `catalog`, and `schema`.
