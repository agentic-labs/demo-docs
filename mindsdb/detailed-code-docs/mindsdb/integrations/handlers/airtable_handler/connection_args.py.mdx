## High-level description
This code defines the connection arguments required for the Airtable handler in MindsDB. It specifies the structure and types of the connection parameters needed to establish a connection with an Airtable base and table, including the base ID, table name, and API key.

## Code Structure
The code defines two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary specifies the structure and metadata for each connection argument, while `connection_args_example` provides an example of how these arguments should be populated.

## References
This code references the `HANDLER_CONNECTION_ARG_TYPE` constant from `mindsdb.integrations.libs.const`, which is used to specify the data types of the connection arguments.

## Symbols

#### Description
An OrderedDict that defines the structure and metadata for the connection arguments required to connect to an Airtable base and table.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| base_id | ARG_TYPE.STR | The Airtable base ID |
| table_name | ARG_TYPE.STR | The Airtable table name |
| api_key | ARG_TYPE.STR | The API key for the Airtable API |

#### Internal Logic
- Each key in the OrderedDict represents a connection argument.
- Each value is a dictionary containing metadata about the argument, including its type and description.
- The `api_key` argument is marked as `secret: True`, indicating that it should be handled securely.
