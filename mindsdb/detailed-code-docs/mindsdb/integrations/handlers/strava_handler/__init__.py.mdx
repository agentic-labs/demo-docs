## High-level description
This code defines the Strava integration handler for MindsDB. It sets up the necessary components for connecting to and interacting with the Strava API, including handler metadata, connection arguments, and table definitions.

## Code Structure
The code imports the StravaHandler class and sets up various attributes such as version, name, type, and description. It also handles potential import errors and defines connection arguments for the Strava API.

## Symbols

### Handler
#### Description
The main handler class for the Strava integration, imported from `strava_handler.py`.

### version
#### Description
The version of the Strava handler, imported from `__about__.py`.

### description
#### Description
A brief description of the Strava handler, imported from `__about__.py`.

### title
#### Description
The display title for the Strava integration.

### name
#### Description
The internal name used to identify the Strava integration.

### type
#### Description
The type of the handler, set to `HANDLER_TYPE.DATA`.

### icon_path
#### Description
The path to the icon file for the Strava integration.

### import_error
#### Description
Stores any import errors that occur when trying to import the StravaHandler class.

### connection_args
#### Description
An OrderedDict containing the connection arguments required for the Strava API.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| strava_client_id | ARG_TYPE.STR | Client ID for accessing Strava Application API |
| strava_access_token | ARG_TYPE.STR | Access Token for accessing Strava Application API |

### connection_args_example
#### Description
An OrderedDict containing example values for the connection arguments.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| mindsdb.integrations.libs.const | Importing HANDLER_TYPE constant |
| collections | Using OrderedDict for connection arguments |

## Error Handling
The code implements basic error handling for the import of the StravaHandler class. If an exception occurs during import, the Handler is set to None, and the error is stored in the import_error variable.
