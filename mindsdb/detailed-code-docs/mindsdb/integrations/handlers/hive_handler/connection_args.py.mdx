## High-level description
This code defines the connection arguments required for establishing a connection to a Hive server in the MindsDB integration handler. It specifies the necessary parameters and their types, along with examples of how these arguments should be provided.

## Code Structure
The code consists of two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary defines the structure and properties of each connection argument, while `connection_args_example` provides an example of how these arguments should be filled.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments required for connecting to a Hive server. Each key in the dictionary represents a connection parameter, and its value is another dictionary describing the parameter's properties.

#### Internal Logic
The dictionary uses the following structure for each argument:
- `type`: Specifies the data type of the argument using `ARG_TYPE` constants.
- `description`: Provides a brief explanation of the argument's purpose.
- `secret`: (Optional) Indicates if the argument should be treated as sensitive information.

### `connection_args_example`
#### Description
An OrderedDict that provides an example of how the connection arguments should be filled when connecting to a Hive server.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to create ordered dictionaries for storing connection arguments |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Imported as ARG_TYPE, used to specify the data types of connection arguments |

## Configuration
The code defines the following configuration options for Hive server connection:

| Option | Type | Description |
|:-------|:-----|:------------|
| user | STR | The user name used to authenticate with the Hive server |
| password | PWD | The password to authenticate the user with the Hive server |
| database | STR | The database name to use when connecting with the Hive server |
| host | STR | The host name or IP address of the Hive server |
| port | INT | The TCP/IP port of the Hive server |
| auth | STR | The Auth type of the Hive server |

Note: The `password` argument is marked as `secret: True`, indicating that it should be handled securely.

## Example Usage
The `connection_args_example` provides a sample of how these arguments should be filled:

```python
connection_args_example = OrderedDict(
    host='127.0.0.1',
    port='10000',
    auth='CUSTOM',
    user='root',
    password='password',
    database='database'
)
```

This example shows typical values for connecting to a Hive server running on the local machine (127.0.0.1) on port 10000, using custom authentication with a root user.
