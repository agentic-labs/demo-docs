## High-level description
This code defines the connection arguments required for connecting to the Coinbase API. It specifies the structure and properties of these arguments, including their types, descriptions, and whether they are required or secret.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments needed for the Coinbase API. It includes three required arguments: API Key, API Secret, and API Passphrase.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| api_key | ARG_TYPE.STR | API Key for connecting to Coinbase API |
| api_secret | ARG_TYPE.PWD | API Secret for connecting to Coinbase API |
| api_passphrase | ARG_TYPE.PWD | API Passphrase for Coinbase API |

#### Internal Logic
Each argument is defined with the following properties:
- `type`: The data type of the argument (string or password)
- `description`: A brief explanation of the argument's purpose
- `required`: Boolean indicating if the argument is mandatory
- `label`: A user-friendly label for the argument
- `secret`: Boolean indicating if the argument should be treated as sensitive information

### `connection_args_example`
#### Description
An OrderedDict providing example values for the connection arguments defined in `connection_args`.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| api_key | str | Example value for the API Key |
| api_secret | str | Example value for the API Secret |
| api_passphrase | str | Example value for the API Passphrase |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to create ordered dictionaries for connection arguments |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Imported as ARG_TYPE, used to specify argument types |

## Notes
1. All three connection arguments (api_key, api_secret, and api_passphrase) are marked as required and secret.
2. The `api_key` is of type STR (string), while `api_secret` and `api_passphrase` are of type PWD (password), indicating they should be handled with additional security measures.
3. The example values provided in `connection_args_example` are placeholder strings and should be replaced with actual credentials when used in a real application.
