## High-level description
This code defines the connection arguments required for establishing a connection to a TiDB server in the MindsDB integration handler. It specifies the structure and types of connection parameters, including user credentials, database name, host, and port.

## Code Structure
The code defines two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary specifies the structure and types of connection parameters, while `connection_args_example` provides an example of how these parameters should be filled.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to create ordered dictionaries for storing connection arguments |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Imported as ARG_TYPE, used to specify the types of connection arguments |

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the structure and types of connection arguments required for connecting to a TiDB server.

#### Internal Logic
The dictionary contains five key-value pairs, each representing a connection parameter:
1. `user`: The username for authentication
2. `password`: The password for authentication (marked as secret)
3. `database`: The name of the database to connect to
4. `host`: The hostname or IP address of the TiDB server
5. `port`: The TCP/IP port of the TiDB server

Each parameter is defined with a type (from ARG_TYPE) and a description.

### `connection_args_example`
#### Description
An OrderedDict that provides an example of how the connection arguments should be filled for a TiDB connection.

#### Internal Logic
Contains sample values for each of the connection parameters defined in `connection_args`:
- `host`: '127.0.0.1'
- `port`: 4000
- `user`: 'root'
- `password`: 'password'
- `database`: 'database'

## Configuration
The connection arguments are configured as follows:
| Option | Type | Description |
|:-------|:-----|:------------|
| user | ARG_TYPE.STR | The user name used to authenticate with the TiDB server |
| password | ARG_TYPE.PWD | The password to authenticate the user with the TiDB server (marked as secret) |
| database | ARG_TYPE.STR | The database name to use when connecting with the TiDB server |
| host | ARG_TYPE.STR | The host name or IP address of the TiDB server |
| port | ARG_TYPE.INT | The TCP/IP port of the TiDB server |

## Notes
- The code emphasizes using '127.0.0.1' instead of 'localhost' when connecting to a local TiDB server.
- The `password` field is marked as `secret: True`, indicating that it should be handled securely and not displayed in plain text.
- The `port` for TiDB is set to 4000 in the example, which is different from the default MySQL port (3306).
