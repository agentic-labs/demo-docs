## High-level description
This code defines an `InformixHandler` class, which is a database handler for connecting to and interacting with Informix databases. It provides methods for connecting, disconnecting, executing queries, and retrieving table and column information from an Informix database.

## Code Structure
The `InformixHandler` class inherits from `DatabaseHandler` and implements various methods for database operations. It uses the `IfxPyDbi` library for Informix connections and `sqlalchemy_informix` for SQL dialect support.

## Symbols

### InformixHandler
#### Description
A class that handles connections and operations for Informix databases.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| name | str | Name of the handler instance |
| connection_data | Optional[dict] | Parameters for connecting to the database |

#### Internal Logic
1. Initializes connection parameters from the `connection_data` dictionary.
2. Constructs a connection string using the provided parameters.
3. Implements methods for connecting, disconnecting, checking connection, executing queries, and retrieving database metadata.

## Configuration
The `connection_args` dictionary defines the configuration options for the Informix connection:

| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| server | str | - | The server name to connect to |
| database | str | - | The database name to use |
| user | str | - | The username for authentication |
| password | str | - | The password for authentication |
| host | str | - | The host name or IP address of the server |
| port | int | - | The port number for the connection |
| schema_name | str | - | The schema name for showing tables |
| logging_enabled | bool | True | Enable or disable logging for COMMIT and ROLLBACK operations |

## Error Handling
The code implements basic error handling using try-except blocks. Errors are logged using the `logger` object and returned as `Response` objects with error messages.

## Logging
Logging is implemented using the `log` module from `mindsdb.utilities`. Error messages are logged using `logger.error()`.
