## High-level description
This code defines configuration settings for the Sentence Transformers handler in MindsDB. It specifies default parameters and constraints for the embedding model, text columns, and GPU usage.

## Symbols

### `DEFAULT_EMBEDDING_MODEL`
#### Description
A constant string that defines the default embedding model to be used.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| DEFAULT_EMBEDDING_MODEL | str | The name of the default embedding model: "all-MiniLM-L6-v2" |

### `Parameters`
#### Description
A Pydantic BaseModel class that defines the configuration parameters for the Sentence Transformers handler.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| embeddings_model_name | str | The name of the embedding model to use (default: DEFAULT_EMBEDDING_MODEL) |
| text_columns | list | A list of text columns to process (default: None) |
| use_gpu | bool | Whether to use GPU for processing (default: False) |

#### Internal Logic
The `Parameters` class uses Pydantic's `BaseModel` for data validation and settings management. It defines three fields:
1. `embeddings_model_name`: A string field with a default value set to `DEFAULT_EMBEDDING_MODEL`.
2. `text_columns`: A list field initialized as None, likely to be populated with column names for text processing.
3. `use_gpu`: A boolean field defaulting to False, determining whether GPU should be used for processing.

The nested `Config` class sets `extra = Extra.forbid`, which prevents any additional fields from being added to the model beyond those explicitly defined.
