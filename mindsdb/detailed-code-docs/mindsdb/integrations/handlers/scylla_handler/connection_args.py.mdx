## High-level description
This code defines the connection arguments required for establishing a connection to a ScyllaDB database in the MindsDB integration handler. It specifies the necessary parameters, their types, descriptions, and whether they are required or optional.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments for the ScyllaDB handler. Each key in the dictionary represents a connection parameter, and its value is another dictionary describing the parameter's properties.

#### Internal Logic
The `connection_args` OrderedDict contains the following keys, each representing a connection parameter:

1. `user`: The username for authentication.
2. `password`: The password for authentication.
3. `protocol_version`: The protocol version for the connection (optional).
4. `host`: The hostname or IP address of the ScyllaDB server.
5. `port`: The port number for the ScyllaDB server.
6. `keyspace`: The keyspace to connect to in ScyllaDB.
7. `secure_connect_bundle`: The path or URL to the secure connect bundle.

Each parameter is defined with the following properties:
- `type`: The data type of the parameter (using `ARG_TYPE` constants).
- `description`: A brief description of the parameter.
- `required`: A boolean indicating whether the parameter is mandatory.
- `label`: A user-friendly label for the parameter.
- `secret`: (Only for password) A boolean indicating if the parameter should be treated as sensitive information.
