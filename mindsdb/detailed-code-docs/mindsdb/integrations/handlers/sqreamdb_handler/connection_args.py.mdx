## High-level description
This code defines the connection arguments required for establishing a connection to a SQreamDB server in the MindsDB integration handler. It specifies the structure and types of connection parameters, along with their descriptions and example values.

## Code Structure
The code consists of two main OrderedDict objects: `connection_args` and `connection_args_example`. The `connection_args` dictionary defines the structure and metadata for each connection parameter, while `connection_args_example` provides sample values for these parameters.

## References
This code references the `HANDLER_CONNECTION_ARG_TYPE` enum from `mindsdb.integrations.libs.const`, which is used to specify the data types of the connection arguments.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the structure and metadata for SQreamDB connection parameters.

#### Internal Logic
The dictionary uses parameter names as keys, with each value being another dictionary containing:
- `type`: The data type of the parameter (from `ARG_TYPE`)
- `description`: A brief explanation of the parameter
- `secret`: (Optional) Boolean indicating if the parameter should be treated as sensitive information

#### Performance Considerations
Using OrderedDict ensures that the parameters are always presented in a consistent order, which can be beneficial for documentation and user interface purposes.

### `connection_args_example`
#### Description
An OrderedDict providing example values for the SQreamDB connection parameters.

#### Internal Logic
This dictionary uses the same keys as `connection_args` but provides sample values that could be used to establish a connection to a SQreamDB server.

## Configuration
The code defines the following configuration options for SQreamDB connections:

| Option | Type | Description |
|:-------|:-----|:------------|
| host | STR | The host name or IP address of the SQreamDB server/database |
| user | STR | The user name used to authenticate with the SQreamDB server |
| password | PWD | The password to authenticate the user with the SQreamDB server |
| port | INT | Specify port to connect SQreamDB server |
| database | STR | Specify database name to connect SQreamDB server |
| service | STR | Optional: service queue (default: "sqream") |
| use_ssl | BOOL | use SSL connection (default: False) |
| clustered | BOOL | Optional: Connect through load balancer, or direct to worker (Default: false - direct to worker) |

Note: The `password` option is marked as `secret: True`, indicating that it should be handled securely and not displayed in plain text.
