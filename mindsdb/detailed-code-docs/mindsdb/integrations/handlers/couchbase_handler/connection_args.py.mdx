## High-level description
This code defines the connection arguments required for establishing a connection to a Couchbase server in the MindsDB integration handler. It specifies the structure and types of connection parameters, along with examples, to facilitate proper authentication and connection to a Couchbase database.

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the connection arguments required for connecting to a Couchbase server. Each argument is represented by a key-value pair, where the key is the argument name and the value is a dictionary containing the argument's type and description.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| user | ARG_TYPE.STR | The username for authentication |
| password | ARG_TYPE.PWD | The password for authentication |
| bucket | ARG_TYPE.STR | The database/bucket name |
| host | ARG_TYPE.STR | The Couchbase server host |
| scope | ARG_TYPE.STR | The scope for query context |

#### Internal Logic
- Uses `OrderedDict` to maintain the order of arguments.
- Each argument is defined with its type (from `ARG_TYPE`) and a description.
- The `password` argument is marked as `secret: True` for security purposes.
- The `scope` argument is optional, with a default value of "_default" if not provided.

### `connection_args_example`
#### Description
An OrderedDict providing an example of how to fill in the connection arguments for a Couchbase server connection.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| host | str | Example host address |
| user | str | Example username |
| password | str | Example password |
| bucket | str | Example bucket name |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to create ordered dictionaries for maintaining the order of connection arguments |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Imported as ARG_TYPE, used to specify the types of connection arguments |

## Notes
- The `scope` argument is optional and will default to "_default" if not provided.
- The `password` argument is marked as secret, indicating that it should be handled securely and not displayed in plain text.
- The example connection arguments do not include the optional `scope` parameter.

This code serves as a configuration file for the Couchbase integration in MindsDB, defining the necessary parameters for establishing a connection to a Couchbase server and providing an example of how these parameters should be filled.
