## High-level description
This code defines the connection arguments required for establishing a connection to a DocumentDB server. It specifies the structure and types of these arguments, providing a standardized way to handle connection parameters for the DocumentDB integration in MindsDB.

## Code Structure
The code consists of two main parts: `connection_args`, which defines the structure and requirements for connection arguments, and `connection_args_example`, which provides an example of how these arguments should be used.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| collections.OrderedDict | Used to maintain the order of connection arguments |
| mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE | Defines constants for argument types |

## Symbols

### `connection_args`
#### Description
An OrderedDict that defines the structure and requirements for DocumentDB connection arguments.

#### Internal Logic
Each key in the OrderedDict represents a connection parameter, and its value is a dictionary describing the parameter's properties:
- `type`: The data type of the argument (from `ARG_TYPE`)
- `description`: A brief explanation of the argument
- `required`: Boolean indicating if the argument is mandatory
- `label`: A user-friendly label for the argument
- `secret`: (Optional) Boolean indicating if the argument should be treated as sensitive information

The defined parameters are:
1. `user`: Username for authentication
2. `password`: Password for authentication (marked as secret)
3. `database`: Name of the database to connect to
4. `host`: Hostname or IP address of the DocumentDB server
5. `port`: TCP/IP port of the DocumentDB server
6. `kwargs`: Additional parameters (optional)

### `connection_args_example`
#### Description
An OrderedDict providing an example of how to use the connection arguments.

#### Internal Logic
Contains sample values for each required connection argument:
- `host`: '127.0.0.1'
- `port`: 27017
- `username`: 'documentdb'
- `password`: 'password'
- `database`: 'database'

## Notes
1. The code uses `ARG_TYPE` from `mindsdb.integrations.libs.const` to define argument types consistently.
2. The `host` description specifically mentions using '127.0.0.1' instead of 'localhost' for local server connections.
3. The `kwargs` argument allows for additional MongoDB-compatible parameters, providing flexibility in the connection setup.

This code serves as a configuration template for DocumentDB connections in the MindsDB ecosystem, ensuring consistent and well-defined connection handling for this integration.
