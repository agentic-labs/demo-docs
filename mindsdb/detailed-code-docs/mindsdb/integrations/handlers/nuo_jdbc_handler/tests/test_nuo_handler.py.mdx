## High-level description
This file contains unit tests for the `NuoHandler` class, which is responsible for interacting with NuoDB databases. The tests cover connecting to the database, executing queries (create, insert, select), retrieving table and column information, and disconnecting.

## Code Structure
The `NuoHandlerTest` class contains multiple test methods, each testing a specific functionality of the `NuoHandler` class. Each test method follows the typical setup, execution, and assertion pattern. The `setUpClass` method initializes a `NuoHandler` instance and connection parameters used by all test methods.

#### Internal Logic
This class defines several test methods, each testing a specific aspect of the `NuoHandler`:
- `test_0_connect`: Tests the `connect` method of the handler.
- `test_1_check_connection`: Tests the `check_connection` method.
- `test_2_create`: Tests creating a table using the `query` method.
- `test_3_insert`: Tests inserting data into a table using the `query` method.
- `test_4_select`: Tests selecting data from a table using the `query` method.
- `test_5_get_tables`: Tests retrieving the list of tables using the `get_tables` method.
- `test_6_get_columns`: Tests retrieving column information for a table using the `get_columns` method.

Each test method follows the arrange, act, and assert pattern:
- **Arrange:** Sets up the necessary objects and data for the test.
- **Act:** Calls the method being tested.
- **Assert:** Verifies that the method under test behaves as expected by checking the response type or the returned data.
