## High-level description
This code defines connection arguments required to establish a connection with a NuoDB database. It specifies the type, description, and example values for each argument.

## Code Structure
The code defines two ordered dictionaries: `connection_args` and `connection_args_example`. 

- `connection_args` maps argument names to their properties (type, description, secret flag).
- `connection_args_example` provides example values for each argument.

#### Internal Logic
The dictionary uses `mindsdb.integrations.libs.const.HANDLER_CONNECTION_ARG_TYPE` enum to define the type of each argument. 

Here's a breakdown of the arguments:

| Argument | Type | Description |
|:---|:---|:---|
| `host` | `ARG_TYPE.STR` | The hostname or IP address of the NuoDB AP or TE. |
| `port` | `ARG_TYPE.INT` | The port number of the NuoDB AP or TE. |
| `database` | `ARG_TYPE.STR` | The name of the NuoDB database. |
| `schema` | `ARG_TYPE.STR` | The name of the schema within the NuoDB database. |
| `user` | `ARG_TYPE.STR` | The username for authentication. |
| `password` | `ARG_TYPE.PWD` | The password for authentication (marked as secret). |
| `is_direct` | `ARG_TYPE.STR` | Indicates whether to connect directly to the TE. |
| `jar_location` | `ARG_TYPE.STR` | The location of the NuoDB JDBC JAR file. |
| `driver_args` | `ARG_TYPE.STR` | Additional arguments to pass to the JDBC driver. |
