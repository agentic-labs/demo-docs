## High-level description
This code defines a responder for handling the 'getParameter' command in a MongoDB-compatible API for MindsDB. It specifically handles the 'featureCompatibilityVersion' parameter, returning a fixed version of 3.6.

## Symbols

### `Responce` class
#### Description
A class that inherits from `Responder` and handles the 'getParameter' command in the MongoDB API.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| query | dict | The query parameters for the 'getParameter' command |
| request_env | object | The request environment |
| mindsdb_env | object | The MindsDB environment |
| session | object | The current session |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| res | dict | The response containing the requested parameter value |

#### Internal Logic
1. Checks if the query is for 'featureCompatibilityVersion'.
2. If true, returns a dictionary with the version set to "3.6" and an "ok" status.
3. If the query is for an unknown parameter, raises a NotImplementedError.

### `responder` variable
#### Description
An instance of the `Responce` class, ready to be used as a responder for 'getParameter' commands.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| mindsdb.api.mongo.classes | Imports the Responder base class |

## Error Handling
The code implements basic error handling by raising a `NotImplementedError` when an unknown parameter is requested.

## Notes
1. The class name 'Responce' appears to be a typo and should likely be 'Response'.
2. The 'featureCompatibilityVersion' is hardcoded to "3.6", which may need to be updated or made configurable in the future.
3. The `when` class attribute uses `helpers.is_true` to determine when this responder should be used, but the exact behavior of `is_true` is not visible in this file.
