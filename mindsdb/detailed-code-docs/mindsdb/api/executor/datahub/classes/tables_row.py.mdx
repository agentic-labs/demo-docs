## High-level description
This code defines a `TablesRow` dataclass that represents a row in a database table, along with utility methods for converting between different data formats. It also defines a `TABLES_ROW_TYPE` class with constants for different table types.

## Code Structure
The main components of this code are:
1. `TABLES_ROW_TYPE`: A class with constants for different table types.
2. `TablesRow`: A dataclass representing a row in a database table, with methods for conversion between different formats.

## Symbols

### `TABLES_ROW_TYPE`
#### Description
A class that defines constants for different types of database tables.

#### Internal Logic
- Uses `__slots__ = ()` to optimize memory usage.
- Defines three constants: `BASE_TABLE`, `VIEW`, and `SYSTEM_VIEW`.

### `TablesRow`
#### Description
A dataclass that represents a row in a database table, with default values for various attributes and methods for data conversion.

#### Inputs
The class has numerous attributes, each representing a column in a database table. Some key attributes include:
- `TABLE_CATALOG`: str
- `TABLE_SCHEMA`: str
- `TABLE_NAME`: str
- `TABLE_TYPE`: str
- Various other attributes related to table properties and statistics

##### `from_dict(data: dict)`
#### Description
A static method that creates a `TablesRow` instance from a dictionary.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| data | dict | A dictionary containing key-value pairs for `TablesRow` attributes |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| TablesRow | TablesRow | A new instance of `TablesRow` created from the input dictionary |

#### Internal Logic
- Converts all keys to uppercase.
- Handles the case where 'TABLE_NAME' is not present but 'NAME' is.
- Removes any keys that are not valid `TablesRow` attributes.
- Creates and returns a new `TablesRow` instance using the processed dictionary.
