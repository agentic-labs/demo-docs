## High-level description
This code defines a function `init_datahub` that initializes and returns an `InformationSchemaDataNode` object. The purpose of this function is to create a data hub for accessing and managing information schema data within the MindsDB system.

## Symbols

### `init_datahub`
#### Description
This function initializes and returns a data hub for the MindsDB system. It creates an instance of `InformationSchemaDataNode` using the provided session object.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| session | object | A session object containing configuration and context information for the MindsDB system |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| datahub | InformationSchemaDataNode | An initialized data hub object for accessing information schema data |

#### Internal Logic
1. Create an instance of `InformationSchemaDataNode` using the provided session object.
2. Return the created `InformationSchemaDataNode` instance.

## Additional Context
The `InformationSchemaDataNode` class, as seen in the related code snippets, is a complex data structure that provides access to various system tables and information about the MindsDB system. It includes tables for schemas, databases, models, engines, and other system-related information. The `init_datahub` function serves as a simple factory method to create and return an instance of this class, which can then be used to query and manage system-level information within MindsDB.
