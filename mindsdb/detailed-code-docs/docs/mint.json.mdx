## High-level description

This JSON file configures the documentation website for MindsDB. It defines the structure, navigation, appearance, and content organization of the documentation, including sections for features, use cases, APIs, SDKs, integrations, and more.

## Code Structure

The JSON object contains several top-level keys that define different aspects of the documentation website:

1. `$schema`: Specifies the schema used for validation.
2. `name`, `logo`, `favicon`: Define branding elements.
3. `colors`: Sets the color scheme for the website.
4. `feedback`: Configures user feedback options.
5. `openapi`, `api`: Specify API-related configurations.
6. `topbarLinks`, `topbarCtaButton`, `anchors`: Define navigation elements.
7. `tabs`, `primaryTab`: Organize content into main sections.
8. `redirects`: List of URL redirections.
9. `navigation`: Defines the main content structure and hierarchy.
10. `footerSocials`: Links to social media profiles.
11. `analytics`: Configures analytics tracking.

## Symbols

### `navigation`
#### Description
An array that defines the hierarchical structure of the documentation content.

#### Internal Logic
- Organizes content into main groups (e.g., "Get Started", "Features", "Use Cases").
- Each group contains pages or nested groups.
- Pages are represented by strings (file paths) or objects with additional properties.

### `redirects`
#### Description
An array of objects defining URL redirections for the documentation website.

#### Internal Logic
- Each object contains a `source` (old URL) and a `destination` (new URL).
- Used to maintain backwards compatibility when reorganizing content.

### `tabs`
#### Description
An array of objects defining the main navigation tabs for the documentation.

#### Internal Logic
- Each object contains a `name` (display text) and a `url` (link destination).
- Represents the top-level organization of the documentation content.

### `colors`
#### Description
An object defining the color scheme for the documentation website.

#### Internal Logic
- Specifies `primary`, `light`, and `dark` colors used throughout the site.

### `feedback`
#### Description
An object configuring user feedback options for the documentation.

#### Internal Logic
- Enables or disables specific feedback features like suggesting edits, raising issues, and giving ratings.
