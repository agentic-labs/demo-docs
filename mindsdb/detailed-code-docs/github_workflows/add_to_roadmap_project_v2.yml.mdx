## High-level description
This GitHub Actions workflow automatically adds newly opened issues to a specific project board. It triggers when an issue is opened in the repository and uses the `actions/add-to-project` action to add the issue to the MindsDB roadmap project.

## Symbols

### Workflow: Add issue to roadmap project
#### Description
This workflow is designed to automate the process of adding new issues to a specific project board, specifically the MindsDB roadmap project.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| N/A | N/A | This workflow doesn't have explicit inputs. It's triggered by the 'issues' event. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| N/A | N/A | This workflow doesn't produce explicit outputs. Its effect is adding the issue to the project board. |

#### Internal Logic
1. The workflow is triggered when an issue is opened in the repository.
2. It runs on the latest Ubuntu runner.
3. It uses the `actions/add-to-project` action (version 0.4.0) to add the issue to the specified project.
4. The action is configured with the project URL and a GitHub token for authentication.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| actions/add-to-project@v0.4.0 | GitHub Action used to add issues to a project board |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| project-url | string | https://github.com/orgs/mindsdb/projects/53 | The URL of the project board where issues will be added |
| github-token | secret | ${{ secrets.ADD_TO_PROJECT_PAT }} | The GitHub token used for authentication when adding issues to the project |

## Error Handling
This workflow does not implement explicit error handling. If the action fails, the workflow will fail, and the issue may not be added to the project board.

## API/Interface Reference
| Trigger | Event | Types | Description |
|:--------|:------|:------|:------------|
| issues | opened | N/A | This workflow is triggered when a new issue is opened in the repository |

This workflow provides an automated interface for adding new issues to the MindsDB roadmap project board without manual intervention.
