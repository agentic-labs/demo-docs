## High-level description
This file, `apps/web/tsconfig.json`, is a TypeScript configuration file specific to the `web` application within a larger project. It defines how TypeScript should compile the code for this application, including compiler options, file inclusions, and paths.

## Symbols
### `tsconfig.json`
#### Description
This JSON file configures the TypeScript compiler for the `web` application. It extends a base configuration (`tsconfig/nextjs.json`) and customizes it for this specific application.

#### Internal Logic
- **`extends`**: Inherits settings from the `tsconfig/nextjs.json` file, providing a base configuration for Next.js projects.
- **`compilerOptions`**:  
    - **`target`**: Specifies the ECMAScript version to target (ES2018 in this case).
    - **`plugins`**: Includes the "next" plugin for Next.js specific TypeScript features.
    - **`paths`**: Defines path aliases for easier module imports. `@/*` maps to `./src/*`, allowing imports like `import { Component } from '@/components/Component'` instead of relative paths.
- **`include`**: Lists file patterns to include in the compilation process:
    - `next-env.d.ts`: Type definitions for Next.js environment variables.
    - `**/*.ts`: All TypeScript files.
    - `**/*.tsx`: All TypeScript JSX files.
    - `.next/types/**/*.ts`: Type definitions generated by Next.js.
    - `next.config.js`: Next.js configuration file.
- **`exclude`**: Excludes the `node_modules` directory from compilation.

## Dependencies
This configuration file implicitly depends on TypeScript and the Next.js framework as it uses their specific settings and plugins.
