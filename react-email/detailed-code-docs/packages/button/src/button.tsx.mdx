## High-level description
The `Button` component renders an anchor tag (`&lt;a&gt;`) that is styled to be a button. It handles padding for both web and email clients, specifically addressing the issue of padding in Outlook emails by using conditional comments and `mso` styles.

## Code Structure
The `Button` component uses several helper functions:
- `parsePadding`: Parses padding values from different CSS properties and converts them to pixel values.
- `pxToPt`: Converts pixel values to point values for use in `mso` styles.
- `computeFontWidthAndSpaceCount`: Calculates the font width and number of spaces needed to simulate padding in Outlook.
- `buttonStyle`: Returns the base styles for the button element.
- `buttonTextStyle`: Returns the styles for the text content of the button.

## Symbols

### `Button`
#### Description
This component renders an anchor tag (`&lt;a&gt;`) styled as a button. It handles padding for both web and email clients, specifically addressing the issue of padding in Outlook emails.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| children | React.ReactNode | The content of the button. |
| style | React.CSSProperties & { pt: number; pr: number; pb: number; pl: number; } | The styles to apply to the button. |
| target | string | The target attribute for the anchor tag. Defaults to "_blank". |
| ...props | ButtonProps | Any other props to pass to the anchor tag. |

#### Outputs
ReactNode: An anchor tag (`&lt;a&gt;`) rendered as a button.

#### Internal Logic
1. **Parses Padding:** It uses the `parsePadding` function to extract padding values from the `style` prop and convert them to pixel values.
2. **Calculates Text Raise:** Calculates the `textRaise` value for Outlook based on the top and bottom padding.
3. **Computes Font Width and Space Count:** For left and right padding, it calculates the `fontWidth` and `spaceCount` required to simulate padding using `mso-font-width` in Outlook.
4. **Renders Button:** Renders an anchor tag (`&lt;a&gt;`) with the provided props and calculated styles.
5. **Conditional Comments for Outlook:** Inside the anchor tag, it uses conditional comments (`&lt;!--[if mso]&gt;&lt;![endif]--&gt;`) to inject hidden `&lt;i/&gt;` tags with `mso` styles to simulate padding in Outlook.
6. **Renders Content:** Renders the button content (`children`) within spans styled with `buttonTextStyle`.

#### Performance Considerations
The component uses inline styles for performance optimization.
