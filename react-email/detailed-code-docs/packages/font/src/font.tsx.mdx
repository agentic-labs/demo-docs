## High-level description
The `Font` component allows users to embed custom fonts in their React email templates. It injects a `&lt;style&gt;` tag into the `&lt;head&gt;` of the HTML document, defining a `@font-face` rule for the custom font and setting the `font-family` for all elements. It also handles fallback fonts for better cross-client compatibility.

## Code Structure
The code defines several type aliases (`FallbackFont`, `FontFormat`, `FontWeight`, `FontStyle`) for better readability and type safety. The main component is `Font`, which takes `FontProps` as input and renders a `&lt;style&gt;` tag with the generated font styles.

## Symbols
### `Font`
#### Description
A React functional component that injects a `&lt;style&gt;` tag into the email's `&lt;head&gt;` to define a custom font and apply it to all elements.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| fontFamily | `string` | The name of the custom font family. |
| fallbackFontFamily | `FallbackFont \| FallbackFont[]` | A single or an array of fallback font families to use if the custom font is not available. |
| webFont | `{ url: string; format: FontFormat }` | (Optional) An object containing the URL and format of the custom font file. |
| fontStyle | `FontStyle` | (Optional) The font style to apply. Defaults to "normal". |
| fontWeight | `FontWeight` | (Optional) The font weight to apply. Defaults to 400. |

#### Outputs
Renders a `&lt;style&gt;` tag containing the generated `@font-face` rule and the `font-family` style for all elements.

#### Internal Logic
1. Constructs the `src` attribute for the `@font-face` rule based on the `webFont` prop.
2. Generates the CSS code for the `@font-face` rule, including the `font-family`, `font-style`, `font-weight`, `mso-font-alt` (for Outlook compatibility), and `src` attributes.
3. Generates the CSS code to set the `font-family` for all elements, including the custom font and fallback fonts.
4. Renders a `&lt;style&gt;` tag with the generated CSS code using `dangerouslySetInnerHTML`.

#### Performance Considerations
The component injects a `&lt;style&gt;` tag into the `&lt;head&gt;` of the HTML document, which is a relatively inexpensive operation. The performance impact should be negligible.
