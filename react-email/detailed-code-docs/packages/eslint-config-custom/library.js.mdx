## High-level description
This code defines a custom ESLint configuration for TypeScript packages, extending the Vercel Engineering Style Guide and integrating Prettier for code formatting. It sets up parser options, global variables, import resolution, and overrides specific rules for the project.

## References
This code references the following external libraries:
- `@vercel/style-guide/eslint/node`: Vercel's ESLint config for Node.js projects.
- `@vercel/style-guide/eslint/typescript`: Vercel's ESLint config for TypeScript projects.
- `eslint-config-prettier`: Disables ESLint rules that conflict with Prettier.

## Symbols
### `module.exports`
#### Description
This symbol defines and exports the ESLint configuration object.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| value | Object | An object containing the ESLint configuration. |

#### Internal Logic
The configuration object extends the Vercel style guide for Node.js and TypeScript, resolves the paths of the extended configurations, and includes the `eslint-config-prettier` configuration. It sets the `project` option for TypeScript parsing, defines `React` and `JSX` as global variables, configures the import resolver for TypeScript, disables the `@typescript-eslint/explicit-function-return-type` rule, and specifies patterns to ignore during linting.
