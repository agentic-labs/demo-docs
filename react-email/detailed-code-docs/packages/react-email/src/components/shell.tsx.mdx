## High-level description
This code defines a `Shell` component that serves as a layout wrapper for a React application. It provides a responsive layout with a sidebar, topbar, and main content area, along with functionality for toggling the sidebar and switching between different views (desktop, mobile, and source).

## Code Structure
The `Shell` component is the main container, which includes the `Sidebar` and `Topbar` components. It manages the state for sidebar toggling and view transitions. The `Sidebar` and `Topbar` components are imported and used within the `Shell`.

## Symbols

### Shell

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| currentEmailOpenSlug | string | The slug of the currently open email |
| children | React.ReactNode | The main content to be rendered |

| markup | string | The HTML markup of the email |

| setActiveView | function | Function to set the active view |

#### Internal Logic
1. Manages state for sidebar toggling and transition effects
2. Renders the `Sidebar` component with conditional styling based on the sidebar state
3. Renders the `Topbar` component if an email is open
4. Renders the main content area with responsive styling

### Sidebar
#### Description
The `Sidebar` component is imported and used within the `Shell`. It displays a list of email templates or directories.

### Topbar
#### Description
The `Topbar` component is imported and used within the `Shell`. It provides navigation and view switching options.

## Side Effects
The component uses the `useState` hook to manage local state for sidebar toggling and transition effects.
