## High-level description
This YAML file defines the permissions for a finance role within an application. It specifies which actions the finance role can perform across various modules of the system, such as analytics, billing, customer management, and organizational settings.

## Code Structure
The file is structured as a YAML document with nested key-value pairs. Each top-level key represents a module or area of the application, and its nested keys define specific actions with boolean values indicating whether the finance role has permission to perform that action.

## Configuration
This YAML file serves as a configuration for role-based access control (RBAC) specifically for the finance role. It defines a set of permissions across various modules of the application.

| Module | Permissions | Description |
|:-------|:------------|:------------|
| analytics | view | Can view analytics |
| invoices | create | Can create invoices |
| organization | view, update | Can view and update organization details |
| organization.invoices | view, update | Can view and update organization invoices |
| organization.integrations | view, create, update, delete | Full access to manage integrations |
| payment_requests | view, create | Can view and create payment requests |

Note: All other permissions not mentioned in the table above are set to `false`, indicating that the finance role does not have access to those actions.
