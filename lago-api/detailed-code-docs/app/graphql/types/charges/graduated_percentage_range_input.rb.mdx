## High-level description
The `GraduatedPercentageRangeInput` defines the structure of input arguments for a graduated percentage range within a graduated percentage charge model. This input type is used to create or update charges with a graduated percentage fee structure.

## References
- `Types::BaseInputObject`: This input type inherits from the `BaseInputObject` which provides basic input object functionality.

## Symbols
### `GraduatedPercentageRangeInput`
#### Description
This input object defines the structure for a single graduated percentage range within a graduated percentage charge model. It includes fields for defining the range boundaries (`from_value`, `to_value`), the flat amount charged within the range (`flat_amount`), and the percentage rate applied to the value within the range (`rate`).

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| from_value | GraphQL::Types::BigInt | The starting value of the graduated percentage range. This field is required. |
| to_value | GraphQL::Types::BigInt | The ending value of the graduated percentage range. This field is optional. |
| flat_amount | String | The flat amount charged within this range. This field is required. |
| rate | String | The percentage rate applied to the value within this range. This field is required. |

#### Outputs
This is an input object, it does not have direct outputs. It is used as an argument to mutations or queries.

#### Internal Logic
This input object doesn't contain any complex internal logic. It simply defines the structure of the input data.
