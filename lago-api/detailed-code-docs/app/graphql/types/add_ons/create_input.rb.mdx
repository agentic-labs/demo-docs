## High-level description
This code defines a GraphQL input type named `CreateAddOnInput` used for creating new add-ons. It specifies the required and optional arguments for the add-on creation mutation.

## References
- `Types::BaseInputObject`: This is the base class for the input type.
- `GraphQL::Types::BigInt`: Used for the `amount_cents` argument.
- `Types::CurrencyEnum`: Used for the `amount_currency` argument.
- `String`: Used for various string-based arguments.

## Symbols
### `CreateAddOnInput`
#### Description
This class defines the input type for creating a new add-on. It inherits from `Types::BaseInputObject` and defines several arguments.

#### Inputs
This class does not have explicit input parameters as it is a GraphQL input type. Instead, it defines arguments that are used when making a mutation request.

#### Outputs
This class does not have explicit output values. It is used as an input type for the `CreateAddOn` mutation.

#### Internal Logic
The class defines several arguments using the `argument` method:

- `amount_cents`: Represents the add-on amount in cents, required and must be a `BigInt`.
- `amount_currency`: Represents the currency of the add-on amount, required and must be a value from `Types::CurrencyEnum`.
- `code`: A unique code for the add-on, required and must be a `String`.
- `description`: An optional description for the add-on, can be a `String`.
- `invoice_display_name`: An optional display name for invoices, can be a `String`.
- `name`: The name of the add-on, required and must be a `String`.
- `tax_codes`: An optional array of tax codes associated with the add-on, each element must be a `String`.

These arguments define the data structure expected when creating a new add-on through the GraphQL API.
