## High-level description
The `CreateInput` class defines the input structure for a GraphQL mutation that creates a data export job for invoices. It specifies the filters, format, and resource type for the export.

## Code Structure
The `CreateInput` class inherits from `Types::BaseInputObject` and defines three arguments: `filters`, `format`, and `resource_type`. These arguments correspond to the input parameters required for creating an invoice data export job.

## References
- `Types::DataExports::Invoices::FiltersInput`: Defines the input object for filtering invoices.
- `Types::DataExports::FormatTypeEnum`: Defines the available export formats.
- `Types::DataExports::Invoices::ExportTypeEnum`: Defines the types of invoice data that can be exported.

## Symbols

### `CreateInput`
#### Description
This class defines the input object for the `CreateInvoicesDataExport` GraphQL mutation. It specifies the structure of the input data required to create a new data export job for invoices.

#### Inputs
This class is an input object and does not accept direct inputs. Instead, it defines arguments that represent the input data.

#### Outputs
This class is an input object and does not have output values.

#### Internal Logic
The class defines three arguments:
- `filters`: An instance of `Types::DataExports::Invoices::FiltersInput` that allows specifying criteria for filtering the invoices to be included in the export.
- `format`: An instance of `Types::DataExports::FormatTypeEnum` that specifies the desired format for the exported data.
- `resource_type`: An instance of `Types::DataExports::Invoices::ExportTypeEnum` that specifies the type of invoice data to export (e.g., 'invoices' or 'invoice_fees').
