## High-level description
The code defines a GraphQL input object type named `UpdateIntegrationMappingInput` used for updating an existing integration mapping. It specifies the arguments that can be provided in a mutation to update an integration mapping.

## Symbols
### `UpdateIntegrationMappingInput`
#### Description
This object defines the input arguments for the `UpdateIntegrationMapping` mutation. It inherits from `Types::BaseInputObject` and specifies the fields that can be updated for an integration mapping.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| external_account_code | String | (Optional) The external account code for the integration mapping. |
| external_id | String | (Optional) The external ID for the integration mapping. |
| external_name | String | (Optional) The external name for the integration mapping. |
| id | ID | (Required) The ID of the integration mapping to update. |
| integration_id | ID | (Optional) The ID of the integration associated with the mapping. |
| mappable_id | ID | (Optional) The ID of the mappable object. |
| mappable_type | Types::IntegrationMappings::MappableTypeEnum | (Optional) The type of the mappable object. |

#### Outputs
This is an input object type, it does not have direct outputs. It is used as an argument to the `UpdateIntegrationMapping` mutation.

#### Internal Logic
This object simply defines the structure and types of the input arguments. The actual update logic is handled by the `UpdateIntegrationMapping` mutation resolver.
