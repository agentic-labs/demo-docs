## High-level description
The code defines a GraphQL input type named `CreateNetsuiteIntegrationInput` used for creating a new Netsuite integration. It specifies the required and optional arguments for the `createNetsuiteIntegration` mutation.

## References
- This input type is used by the `Mutations::Integrations::Netsuite::Create` mutation.

## Symbols
### `CreateInput`
#### Description
This class defines the input object structure for creating a new Netsuite integration. It inherits from `Types::BaseInputObject` and specifies the required and optional arguments.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| code | String | The code identifier for the integration. (required) |
| name | String | The name of the integration. (required) |
| account_id | String | The Netsuite account ID. (required) |
| client_id | String | The Netsuite client ID. (required) |
| client_secret | String | The Netsuite client secret. (required) |
| connection_id | String | The ID of the connection to use for the integration. (required) |
| script_endpoint_url | String | The URL of the Netsuite script endpoint. (required) |
| token_id | String | The Netsuite token ID. (required) |
| token_secret | String | The Netsuite token secret. (required) |
| sync_credit_notes | Boolean | Whether to synchronize credit notes. (optional) |
| sync_invoices | Boolean | Whether to synchronize invoices. (optional) |
| sync_payments | Boolean | Whether to synchronize payments. (optional) |
| sync_sales_orders | Boolean | Whether to synchronize sales orders. (optional) |

#### Outputs
This class does not have explicit outputs. It is used as an input type for the `createNetsuiteIntegration` mutation.
