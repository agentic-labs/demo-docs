## High-level description
The code defines a GraphQL input type named `UpdateInput` within the `Types::Wallets::RecurringTransactionRules` module. This input type is used to specify the arguments for updating an existing recurring transaction rule.

## Symbols

### `UpdateInput`
#### Description
This class defines the input type `UpdateRecurringTransactionRuleInput` for GraphQL mutations. It inherits from `Types::BaseInputObject` and lists all the arguments that can be provided when updating a recurring transaction rule.

#### Inputs
This class does not define any inputs. It defines arguments for a GraphQL mutation.

#### Outputs
This class does not define any outputs. It is an input type for a GraphQL mutation.

#### Internal Logic
The class uses the `argument` method to define the following fields for updating a recurring transaction rule:

| Name | Type | Description |
|:-----|:-----|:------------|
| `granted_credits` | `String` | The amount of credits granted by the rule (optional) |
| `interval` | `Types::Wallets::RecurringTransactionRules::IntervalEnum` | The interval at which the rule recurs (optional) |
| `invoice_requires_successful_payment` | `Boolean` | Whether the invoice requires successful payment (optional) |
| `lago_id` | `ID` | The ID of the rule in Lago (optional) |
| `method` | `Types::Wallets::RecurringTransactionRules::MethodEnum` | The method used for the recurring transaction (optional) |
| `paid_credits` | `String` | The amount of credits paid by the rule (optional) |
| `started_at` | `GraphQL::Types::ISO8601DateTime` | The date and time when the rule starts (optional) |
| `target_ongoing_balance` | `String` | The target ongoing balance for the rule (optional) |
| `threshold_credits` | `String` | The threshold of credits for the rule (optional) |
| `trigger` | `Types::Wallets::RecurringTransactionRules::TriggerEnum` | The trigger for the recurring transaction (optional) |

These fields correspond to the attributes of a recurring transaction rule that can be modified.
