## High-level description
The `CustomerCheckoutSerializer` class is a custom serializer responsible for formatting data related to a customer's checkout process for API responses. It specifically includes information about the customer, their associated payment provider, and a checkout URL.

## References
This serializer is used by the `Webhooks::PaymentProviders::CustomerCheckoutService` and the `Api::V1::CustomersController`.

## Symbols

### `CustomerCheckoutSerializer`
#### Description
This class inherits from `ModelSerializer` and overrides the `serialize` method to define a custom JSON representation of a customer checkout object.

#### Inputs
The constructor of the `CustomerCheckoutSerializer` class takes the following inputs:

| Name | Type | Description |
|:-----|:-----|:------------|
| model | Object | An instance of a customer object. |
| options | Hash | A hash of options, including `:checkout_url`. |

#### Outputs
The `serialize` method returns a hash containing the serialized customer checkout data.

| Name | Type | Description |
|:-----|:-----|:------------|
| lago_customer_id | Integer | The Lago ID of the customer. |
| external_customer_id | String | The external ID of the customer. |
| payment_provider | String | The name of the payment provider. |
| payment_provider_code | String | The code of the payment provider. |
| checkout_url | String | The URL for the customer to complete the checkout process. |

#### Internal Logic
The `serialize` method constructs a hash containing specific attributes from the `model` object and the `checkout_url` from the `options` hash.

## Dependencies
This serializer depends on the `ModelSerializer` class.
