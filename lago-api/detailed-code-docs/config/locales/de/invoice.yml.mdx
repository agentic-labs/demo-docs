## High-level description
This YAML file contains German translations for invoice-related terms and phrases used in a billing or invoicing system. It provides localized text for various aspects of an invoice, including subscription details, usage-based fees, tax information, and payment terms.

## Symbols

### `de:`
#### Description
This is the root key for the German language translations in the YAML file.

### `invoice:`
#### Description
This key contains all the invoice-related translations nested under it.

### Notable sub-keys under `invoice:`

#### `all_subscriptions:`
Description: Translates to "All subscriptions" in German.

#### `amount_with_tax:` and `amount_without_tax:`
Description: Provides translations for amounts with and without taxes.

#### `breakdown:`
Description: Translates to "Breakdown" in German, used for itemizing invoice details.

#### `graduated:`
Description: Contains translations for graduated pricing tiers, including fees per unit and flat fees for different ranges.

#### `package:`
Description: Includes translations for package-based pricing, such as fee per package and free units.

#### `percentage:`
Description: Contains translations for percentage-based pricing, including adjustments and free units per transaction.

#### `prepaid_credits:`
Description: Translates to "Prepaid credits" in German.

#### `tax_name_only:`
Description: Provides translations for various tax scenarios, such as tax-exempt customers and reverse charge mechanisms.

#### `taxes:`
Description: Contains specific tax-related translations, including exemptions and reverse charge explanations.

## Dependencies
This file does not have any external dependencies. It is a standalone YAML file containing translations.

## Configuration
This file serves as a configuration for German language translations in the invoicing system. The structure of the YAML file defines how different invoice-related terms and phrases should be displayed in German.

## Notes
1. The file uses nested keys to organize translations logically, grouping related terms together.
2. Some translations include placeholders (e.g., `%{breakdown_duration}`) which are likely replaced with actual values when rendering the invoice.
3. The file includes translations for various pricing models, including subscriptions, usage-based fees, graduated pricing, and package-based pricing.
4. Tax-related translations are comprehensive, covering different scenarios like tax exemptions and reverse charge mechanisms.
5. The file includes translations for time-based terms (weekly, monthly, quarterly, yearly) which are likely used for different subscription intervals.

This translation file is crucial for providing a localized experience for German-speaking users of the invoicing system, ensuring that all invoice-related information is presented in clear and accurate German.
