## High-level description
This code defines a `Params` class within the `Lago::Adyen` module. Its primary purpose is to handle and format parameters for Adyen API requests, ensuring that default application information is included in every request.

## Symbols

### `Lago::Adyen::Params`
#### Description
This class is responsible for managing and formatting parameters for Adyen API requests. It allows custom parameters to be merged with default application information.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| params | Hash | Custom parameters to be included in the Adyen API request |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| formatted_params | Hash | A hash containing both default and custom parameters |

#### Internal Logic
1. The class initializes with an optional `params` hash.
2. The `to_h` method merges the default parameters with any custom parameters provided.
3. The `default_params` method defines a hash with application information, including the platform name (Lago) and integrator.

## Dependencies
This code doesn't explicitly import any external libraries, but it's part of the larger Lago application and specifically relates to Adyen integration.

## References
The `Lago::Adyen::Params` class is used in the related `CreditNotes::Refunds::AdyenService` class, specifically in the `create_adyen_refund` method. It's used to format the parameters for the Adyen refund API call.

## Performance Considerations
This class is lightweight and doesn't perform any heavy computations. The main operation is a simple hash merge, which is generally efficient in Ruby.
